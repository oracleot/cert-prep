Rex/Sage LangGraph Map

A quick map of how this repo turns Rex and Sage into an explicit LangGraph state machine.

Core idea

Rex and Sage are not free-floating agents. In this implementation, each is a node function inside a LangGraph state machine. The graph owns the control flow; each node reads AppState and returns a small state update.

Files to know

  • agents/graphs/session.py — graph structure, routing, compile step.
  • agents/state.py — shared state contract.
  • agents/nodes/rex_challenge.py — first Rex challenge node.
  • agents/nodes/rex_rechallenge.py — later Rex challenge node.
  • agents/nodes/evaluate_answer.py — grading node.
  • agents/nodes/sage_respond.py — Sage explanation/depth nodes.
  • agents/llm.py — LangChain ChatOpenAI wrapper for OpenRouter.

Flow

START → coach_open → rex_challenge → evaluate_answer → if correct: sage_depth else: sage_explain → if cycle < max_cycles: rex_rechallenge else: coach_close → END

Node pattern

def some_node(state: AppState) -> dict:
    # 1. read current state
    # 2. call tools / model / database if needed
    # 3. return only the state patch
    return {"field_to_update": value}

Memory hook

State is the backpack. Rex puts a challenge into it. The user puts an answer into it. The evaluator puts a judgment into it. Sage puts an exchange record into it. The graph decides where the backpack goes next.
