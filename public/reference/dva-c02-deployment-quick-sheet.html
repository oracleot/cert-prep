Deployment quick sheet

Domain 3 exam selectors for artifacts, dev/test environments, automated deployment testing, CI/CD, safe rollout, and rollback.

Domain 3 shape

  • Weight: 20%, roughly 13 exam questions.
  • Official tasks: prepare artifacts, test in development, automate deployment testing, and deploy with CI/CD services.
  • Fast triage: package, build, template, stage, branch, approval, canary, rollback, or pipeline usually means Deployment.

Task selector

Dependencies, package, layer, image → 3.1 Prepare artifacts Mock endpoint, stage, alias, dev stack → 3.2 Test in dev Automated test gate, buildspec, version → 3.3 Automate tests Commit, pipeline, approval, rollout → 3.4 Deploy with CI/CD

Artifact selector

Team package repository → CodeArtifact Packaged SAM/Lambda zip storage → Amazon S3 Native Lambda dependency build → sam build --use-container Zip and upload SAM artifacts → sam package Create or update serverless stack → sam deploy Shared zip dependency or runtime → Lambda layer Container image source → Amazon ECR Runtime feature flags/config rollout → AWS AppConfig

SAM command cuts

  • sam build: prepare local project; use containers to match the Lambda runtime.
  • sam package: upload artifacts to S3 and produce a packaged template.
  • sam deploy: create or update the CloudFormation stack.
  • Transform: AWS::Serverless-2016-10-31 enables SAM resources inside CloudFormation.

Dev/test environment selector

Backend missing but consumers must test → API Gateway MOCK Same API, different env settings → API Gateway stages/variables Stable trigger while Lambda changes → Lambda alias ARN EC2 packages/files from stack metadata → cfn-init Stack waits for EC2 bootstrap → cfn-signal Fastest deployment, high blast radius → all-at-once Major platform/runtime change → blue/green Same resources stop/install/start → in-place

Automated testing selector

Ordered source-build-test-deploy flow → CodePipeline Commands run unit tests/package app → CodeBuild buildspec Small slice, wait, then rest → CodeDeploy canary Equal slices every interval → CodeDeploy linear All traffic moves immediately → CodeDeploy all-at-once Only approved image proceeds → ECR tag or digest Cross-account test deployment → STS AssumeRole target role

CI/CD service selector

Release workflow orchestration → CodePipeline Build commands, artifacts, reports → CodeBuild Traffic shifting and app rollout → CodeDeploy EC2 file placement/lifecycle hooks → AppSpec file Same stack across accounts/Regions → CloudFormation StackSets Program IaC in familiar language → AWS CDK Managed app platform deployment modes → Elastic Beanstalk Stack rollback on alarm → CloudFormation rollback trigger

High-yield traps

  • CodePipeline orchestrates; CodeBuild runs commands; CodeDeploy shifts traffic or updates app fleets.
  • Buildspec belongs to CodeBuild. AppSpec belongs to CodeDeploy EC2/on-premises deployments.
  • Lambda versions are immutable; aliases are movable pointers for promotion and traffic shifting.
  • Canary is small-first-then-rest. Linear is equal increments over time.
  • CloudFormation ChangeSets preview changes; StackSets roll out stacks across accounts or Regions.
  • Rollback clues often mention failed hooks, failed health checks, or CloudWatch alarms.
