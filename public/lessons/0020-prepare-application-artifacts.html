Lesson 0020

Prepare the artifact before you deploy.

Task Statement 3.1: choose the build output, package command, repository, or configuration source the scenario is really testing.

The win

You will separate dependency artifacts, serverless packages, container images, and runtime configuration without over-selecting CI/CD services.

Source map: Domain 3 Task 3.1 in docs/aws dva-c02.md.

Artifact selector

Package dependencies for teams → AWS CodeArtifact Store SAM/Lambda zip artifacts → Amazon S3 Build Lambda-native dependencies → sam build --use-container Zip and upload SAM artifacts → sam package Create or update the stack → sam deploy Declare SAM inside CloudFormation → Transform Store container deployment images → Amazon ECR Point traffic without changing ARNs → Lambda alias

Exam shortcut

If the template includes SAM resources, look for Transform: AWS::Serverless-2016-10-31. If the question says deploy code plus dependencies, decide between zip, layer, or container image.

Retrieval drill: pick the deployment clue

1. A DVA-C02 scenario describes a development team that needs to store and share npm packages across multiple AWS accounts. You must choose the managed artifact repository. Which AWS service provides managed package hosting for npm, pip, and Maven?

CodeArtifact is a fully managed artifact repository that works with npm, pip, Maven, NuGet, and more—Task 3.1.
CloudFormation provisions infrastructure; it does not store package dependencies.
CodeDeploy automates application deployments; it does not host packages.
AppConfig manages runtime configurations; it does not store npm packages.
CodeArtifact is correct. It is the managed artifact repository for language package managers (Task 3.1).
Review CodeArtifact documentation and Task 3.4 CI/CD services.

2. A scenario involves AWS SAM code that must be zipped, uploaded to S3, and rewritten as a deployment-ready template. You must name the SAM CLI command. Which command performs the zip, S3 upload, and template rewrite?

sam package zips the application, uploads the package to S3, and writes a packaged CloudFormation template—Task 3.1.
sam deploy deploys a packaged template; it does not create the package or upload to S3.
sam validate checks template syntax; it does not upload artifacts.
sam publish publishes to the AWS Serverless Application Repository; it does not handle packaging.
sam package is correct. It creates the deployment package and uploads it to S3 (Task 3.1).
Study SAM CLI command reference and Task 3.4 SAM deployment.

3. A DVA-C02 scenario describes a Lambda function with native dependencies that must compile against the Amazon Linux runtime environment. You must choose the correct build command. Which SAM CLI flag builds inside a Lambda-compatible Docker container?

sam build --use-container builds inside a Docker container that matches the Lambda runtime environment—Task 3.1.
sam deploy --guided launches an interactive deployment wizard; it does not build native dependencies.
sam delete --stack deletes a CloudFormation stack; it does not build anything.
sam traces --tail follows X-Ray traces; it does not build native dependencies.
sam build --use-container is correct. Native Lambda dependencies require a compatible runtime environment, provided by the container flag.
Review Task 3.2 Lambda promotion and Lambda deployment packages.

4. A DVA-C02 scenario shows a CloudFormation template that contains AWS SAM Lambda and API Gateway resources. You must identify the required template declaration. Which CloudFormation top-level key processes SAM resources before creating real AWS resources?

Transform: AWS::Serverless-2016-10-31 instructs CloudFormation to process SAM resources before creating the actual stack—Task 3.1.
Mappings define conditional parameter lookups; they do not process SAM resources.
Outputs expose stack values; they do not process SAM resources.
Metadata provides附加 data about resources; it does not process SAM syntax.
Transform is correct. The Transform key triggers the SAM macro to convert serverless resources into CloudFormation resources.
Study Task 3.4 SAM deployment and CloudFormation SAM transform.

5. A scenario describes a Lambda zip package that exceeds the deployment size limit because the same dependencies are repeated in every function. You must choose the shared dependency solution. Which Lambda feature allows multiple functions to share the same dependencies without duplication?

Lambda layers package shared libraries and runtimes that all functions in the same account can reference—reducing package size duplication—Task 3.1.
Lambda aliases point to versions; they do not share dependencies across functions.
Lambda triggers invoke functions; they do not solve dependency duplication.
Lambda resource policies control invocation permissions; they do not package dependencies.
Lambda layer is correct. Layers share dependencies across functions, reducing zip size and avoiding duplication.
Review Task 3.2 Lambda promotion and Lambda layers.

6. A DVA-C02 scenario involves an ECS task definition that must reference an immutable Docker image stored in AWS. You must choose the container image registry. Which AWS managed service stores and manages Docker images for ECS and Lambda container functions?

Amazon ECR is the managed Docker container registry that stores, scans, and serves images to ECS tasks and Lambda container functions—Task 3.1.
EFS is a file system; it does not store Docker container images.
SQS is a message queue; it does not store container images.
VPC provides networking isolation; it does not store container images.
Amazon ECR is correct. ECR stores Docker images and provides them to ECS tasks and Lambda container-image functions.
Review Task 3.4 ECS deployment and Amazon ECR.

7. A scenario describes an EC2 instance that must install packages, create files, and start services from CloudFormation template metadata during launch. You must name the bootstrap helper. Which CloudFormation helper script reads metadata and configures the EC2 instance?

cfn-init reads the AWS::CloudFormation::Init metadata and installs packages, creates files, and starts services on EC2—Task 3.1.
cfn-lint validates CloudFormation template syntax; it does not configure EC2 instances.
cfn-flip reformats CloudFormation templates between JSON and YAML; it does not configure EC2.
cfn-diff compares local templates against deployed stacks; it does not configure EC2.
cfn-init is correct. It is the CloudFormation bootstrap helper that applies metadata-driven configuration to EC2 instances.
Study CloudFormation helper scripts and Task 3.2 EC2 bootstrapping.

8. A DVA-C02 scenario involves runtime feature flags and configuration values that must be validated, deployed gradually, and rolled back if errors occur. You must choose the configuration management service. Which AWS service provides validated, safe runtime configuration propagation without code redeployment?

AppConfig validates configuration data, supports gradual deployment, monitors with CloudWatch alarms, and rolls back on error—Task 3.1 configuration lifecycle.
CodeBuild runs build and test commands; it does not manage runtime feature flags.
CodeStar provides a unified project dashboard; it does not propagate runtime configurations.
CloudTrail records API calls; it does not manage application configuration.
AppConfig is correct. It provides validated, monitored, and rollback-capable configuration propagation without redeployment.
Review AWS AppConfig and Task 3.2 configuration.
