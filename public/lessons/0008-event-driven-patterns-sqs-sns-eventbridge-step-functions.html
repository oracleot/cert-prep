Lesson 0008

Pick the event service by the coupling problem.

DVA-C02 Domain 1 expects fast recognition of SQS, SNS, EventBridge, and Step Functions in loose-coupling scenarios.

The win

When a scenario says "decouple," do not stop at "serverless." Classify the communication shape: buffer, broadcast, route, or orchestrate.

Related review: Lambda event source mappings.

The four shapes

SQS producer → queue → consumer SNS fanout publisher → topic → many subscribers EventBridge event bus → rules → selected targets Step Funcs state machine → ordered workflow steps

SQS: buffer and absorb spikes

Choose SQS when producers and consumers should not depend on each other's speed or availability. Keywords: queue, backlog, polling, message retention, visibility timeout, dead-letter queue.

SNS: push one message to many

Choose SNS for pub/sub fanout when one publication must notify multiple subscribers such as SQS queues, Lambda functions, HTTPS endpoints, email, or SMS.

EventBridge: route application events

Choose EventBridge when events need content-based rules, SaaS/AWS/custom event buses, schema discovery, or many services reacting without a central workflow owner.

Step Functions: own the workflow state

Choose Step Functions when the scenario needs branching, retries, wait states, error handling, human-readable state history, or long-running workflow orchestration.

Orchestration vs choreography

  • Orchestration: one coordinator decides the next step. Exam clue: state machine, workflow, branch, retry policy.
  • Choreography: services react to events independently. Exam clue: event bus, loose coupling, rules, subscribers.
  • Fanout: duplicate one event to multiple consumers. Classic answer: SNS topic with SQS subscriptions.
  • Back pressure: smooth traffic spikes between producer and worker. Classic answer: SQS queue.

Exam traps

  • SNS is not a durable work queue. If workers need to pull and retry, put SQS behind SNS.
  • EventBridge is not Step Functions. EventBridge routes events; Step Functions tracks workflow state.
  • Lambda is not the default orchestrator. Complex coordination belongs in Step Functions.
  • Loose coupling still needs idempotency. Retries and duplicate events can happen.

Practice: choose the pattern

1. An e-commerce platform receives a surge of order events that arrive faster than fulfillment workers can process them. Producers and consumers must not depend on each other's speed or availability. Which AWS service absorbs the traffic spike?

SQS buffers messages between producers and consumers, decoupling their availability and processing rates.
SNS pushes one message to many subscribers—it does not buffer for slower consumers.
EventBridge routes events based on rules—it does not provide durable buffering.
Step Functions orchestrate workflow steps—they do not buffer message backlogs.
SQS is correct. SQS queues decouple producers from consumers by buffering messages until workers can process them.
Review Lesson 0004 on Lambda event sources. Amazon SQS documentation.

2. A SaaS platform fires one checkout event that must simultaneously notify the billing service, analytics pipeline, and email subsystem. Which AWS service fans out to multiple targets?

SQS queues deliver messages to one consumer at a time—they do not broadcast to multiple targets.
SNS pub/sub broadcasts one message to all topic subscribers: billing, analytics, email.
EventBridge routes events based on rules—SNS is the more direct answer for one-to-many fanout.
Step Functions orchestrate sequential or parallel workflow tasks—they do not broadcast events.
SNS is correct. SNS pub/sub fans out one message to multiple subscribers simultaneously.
Review Lesson 0008 for the broadcast pattern. Amazon SNS documentation.

3. An ISV's SaaS application emits events that must route to different Lambda functions or SQS queues based on event detail-type and source field values. Which AWS service filters and routes based on event content?

SQS delivers messages to consumers—it does not route based on event field values.
SNS broadcasts to all subscribers—it does not route selectively based on event content.
EventBridge rules route events to targets based on event content, patterns, or custom event buses.
Step Functions execute workflows—they do not route events based on field values.
EventBridge is correct. EventBridge rules filter and route events based on JSON patterns in event fields.
Review Lesson 0008 on EventBridge. Amazon EventBridge rules.

4. An order fulfillment microservice requires conditional branching, human-approval wait states, and per-task retry policies. The workflow must also provide a human-readable execution history. Which AWS service orchestrates this multi-step workflow?

SQS queues handle message buffering—they do not orchestrate multi-step workflows.
SNS broadcasts events—it does not track workflow state or implement branching logic.
EventBridge routes events—it does not maintain workflow state or implement wait states.
Step Functions manage state machines with branching, wait states, retries, and error handling.
Step Functions is correct. State machines provide branching, wait states, retries, and human-readable workflow history.
Review Lesson 0008 on Step Functions. AWS Step Functions documentation.

5. A microservices platform uses event-driven services that subscribe to an Amazon EventBridge event bus and react independently to order, payment, and shipping events without a central coordinator dictating the next step. Which architectural pattern describes this decentralized, event-driven approach?

EventBridge event bus enables choreography—services subscribe to events and react independently.
State machine implies orchestration—a central coordinator controls the workflow.
Wait state is an orchestration concept in Step Functions—it implies central control.
Retry policy is an operational detail—it does not indicate choreography or orchestration.
Event bus is correct. Event-driven services reacting to events on a bus is the definition of choreography.
