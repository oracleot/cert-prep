Lesson 0012

Choose the data pattern from the exam clue.

Domain 1 loves service-selection questions: cache the read path, stream events, transform data, or build the app flow.

The win

You will sort four common Domain 1 traps by keyword: lazy cache, write-through cache, streaming ingest, and ETL tool.

Related review: event-driven patterns and Domain 1 map.

1. Caching: pick the write/read behavior

Lazy loading app misses cache → reads database → fills cache Write-through app writes cache and database in the write path TTL cache entry expires after a chosen time window
  • Lazy loading: choose it when only requested data should be cached. First read is slower; later reads are faster.
  • Write-through: choose it when the cache must be warm immediately after writes. Write latency can increase.
  • TTL: choose it when stale data must expire automatically. TTL is a freshness control, not a database backup.
  • DAX / ElastiCache clue: DAX fits DynamoDB microsecond read caching; ElastiCache fits Redis/Memcached app caches.

2. Streaming: pick the ownership model

Kinesis Data Streams

Choose it for custom consumers, ordered records per shard, replay within retention, Lambda event source mappings, or multiple applications reading the same stream.

Amazon Data Firehose

Choose it when the clue says near-real-time delivery to S3, Redshift, OpenSearch, Splunk, or HTTP endpoints with minimal custom consumer code.

Amazon MSK

Choose it when the scenario already uses Apache Kafka, needs Kafka APIs, Kafka clients, topics, partitions, or wants AWS to manage Kafka brokers.

3. ETL: pick the transformation engine

Glue managed serverless ETL + Data Catalog + crawlers EMR managed Hadoop/Spark clusters for big data frameworks Lambda small event transform, short running, serverless glue code Kinesis streaming transform/analytics before storage or delivery
  • AWS Glue: default exam answer for managed ETL, schema discovery, crawlers, and the Glue Data Catalog.
  • Amazon EMR: choose for Spark/Hadoop ecosystem control, cluster tuning, or long/heavy big-data processing.
  • Lambda: choose for lightweight transforms triggered by S3, streams, EventBridge, API Gateway, or queues.
  • Kinesis: choose when transformation must happen continuously on streaming records.

Fast distractor cuts

  • If the question says replay records, Firehose is usually the distractor; think Kinesis Data Streams.
  • If it says deliver to S3 with no servers, custom Kinesis consumers are the distractor; think Data Firehose.
  • If it says Kafka clients, do not translate to Kinesis; think Amazon MSK.
  • If it says schema catalog/crawler, Lambda is too small; think AWS Glue.

Retrieval drill: select the service or pattern

1. A product catalog API serves infrequent and unpredictable read requests. The team wants to cache only items that users actually request, filling the cache on demand to minimize storage overhead. Which Amazon ElastiCache strategy matches this requirement?

Lazy loading fills the cache only on cache misses—ideal for on-demand caching of requested items.
Write-through writes to cache and database simultaneously—cache is warm but not on-demand.
Data Firehose is a streaming delivery service—not a caching strategy.
AWS Glue is an ETL service—not a caching strategy.
Lazy loading is correct. Cache misses trigger database reads, filling the cache on demand.
Review Lesson 0012 on caching strategies. Amazon ElastiCache strategies.

2. A stock price tracking application requires newly written price data to be immediately visible in the cache without a subsequent read to populate it. Write latency increase is acceptable. Which Amazon ElastiCache caching strategy keeps the cache warm on writes?

Lazy loading fills the cache only on reads—not on writes.
Write-through writes to both cache and database simultaneously, keeping the cache warm.
Data Firehose is a streaming delivery service—not a caching strategy.
AWS Glue is an ETL service—not a caching strategy.
Write-through is correct. Data is written to both cache and database, ensuring immediate cache freshness.
Review Lesson 0012 on write-through caching. Amazon ElastiCache strategies.

3. Several applications must replay ordered events from the same stream. Which streaming service supports replay?

Kinesis Data Streams retains records for replay within the retention period to multiple consumers.
Firehose delivers data in near-real-time—it does not support replay or multiple consumers.
MSK is Kafka-managed—it supports replay but is for existing Kafka apps, not a general answer.
Glue Data Catalog stores schema metadata—it does not retain streaming data.
Kinesis Data Streams is correct. It retains records and supports replay to multiple consumers within the retention window.
Lesson 0008. Kinesis Data Streams.

4. Stream records should land in S3 with little code. Which AWS streaming service delivers to S3 serverlessly?

Kinesis Data Streams requires custom consumer code to write to S3.
Data Firehose serverlessly delivers streaming data to S3, Redshift, OpenSearch, or HTTP endpoints.
MSK is Kafka-managed—it requires custom consumers to write to S3.
Glue Data Catalog stores schema metadata—it does not deliver data to S3.
Data Firehose is correct. It provides serverless delivery to S3 without writing custom consumer code.
Review Lesson 0012 on Data Firehose. Amazon Data Firehose.

5. Existing Kafka apps need managed Kafka brokers. Which AWS service manages Kafka infrastructure?

Kinesis Data Streams is an AWS-native streaming service—not Kafka-compatible.
Data Firehose is a streaming delivery service—not a Kafka service.
Amazon MSK provides fully managed Apache Kafka brokers on AWS.
Glue Data Catalog stores schema metadata—it does not run Kafka.
Amazon MSK is correct. MSK manages Kafka brokers, enabling existing Kafka apps to run on AWS.
Review Lesson 0012 on MSK. Amazon MSK.

6. You need to crawl S3 data and maintain a schema catalog. Which AWS service discovers schemas automatically?

AWS Glue crawlers automatically discover schemas and populate the Glue Data Catalog.
Lambda can process data but does not automatically discover schemas across S3.
EMR runs Spark/Hadoop jobs—it is not the default for automatic schema discovery.
TTL expires DynamoDB items—it does not crawl S3 or discover schemas.
AWS Glue is correct. Glue crawlers automatically discover data schemas and populate the Data Catalog.
Review Lesson 0012 on Glue. AWS Glue.

7. A short S3 event transform should run serverlessly. Which AWS compute service fits short-running transforms?

AWS Glue runs managed ETL jobs—it is overkill for a simple short-running transform.
Lambda is serverless and charges per invocation—it is ideal for short-running transforms triggered by S3 events.
EMR runs Spark/Hadoop clusters—it is designed for heavy, long-running processing.
TTL is a DynamoDB feature—it does not run transforms.
Lambda is correct. Lambda is serverless, event-driven, and charges per millisecond—ideal for lightweight S3 transforms.
Review Lesson 0012 on Lambda transforms. AWS Lambda use cases.

8. A serverless API needs to CRUD over DynamoDB. Which service combination provides the API and compute?

API Gateway provides the HTTP API; Lambda handles business logic for DynamoDB CRUD operations.
KMS encrypts data; Macie discovers sensitive data—they do not provide CRUD APIs.
CodeDeploy shifts traffic; SAM packages serverless apps—they do not provide CRUD APIs.
X-Ray traces requests; Alarms notify operators—they do not provide CRUD APIs.
API Gateway + Lambda is correct. This is the standard serverless pattern for DynamoDB CRUD operations.
