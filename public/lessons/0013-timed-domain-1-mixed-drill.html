Timed drill

1. A public HTTPS API invokes Lambda and returns JSON. What is the serverless API pattern?

API Gateway receives HTTP; Lambda runs logic and returns JSON.
S3 + Firehose handles streaming—not HTTP APIs.
DAX caches DynamoDB reads—it does not handle APIs.
SNS + EventBridge routes events—not an API pattern.
API Gateway + Lambda is the standard serverless API pattern.
Lesson 0007. API Gateway HTTP API.

2. A media processing application receives image uploads via Amazon S3 that must automatically trigger a Lambda resize job each time a new object is created. The resize function must run serverlessly with no dedicated infrastructure. Which event-driven pattern fits this workflow?

DAX caches DynamoDB reads—not for image processing.
S3 triggers Lambda on object creation; Lambda resizes the image.
Glue crawlers discover schemas—not for image processing.
MSK is Kafka-managed—overkill for single image processing.
S3 event to Lambda is correct. S3 object-created events trigger Lambda.
Lesson 0002. Lambda with S3.

3. An order processing microservice requires durable message buffering and automatic retry behavior between the order intake Lambda and the fulfillment service. The queue must survive Lambda failures and maintain message ordering. Which AWS queue service provides this durability?

DAX caches DynamoDB reads—not for message buffering.
EFS is a shared file system—not a message queue.
SQS provides durable message queuing with visibility timeout and retry support.
X-Ray traces requests—not message buffering.
Amazon SQS is correct. SQS buffers messages between services with built-in retry behavior.
Lesson 0008. SQS.

4. One event must fan out to email, Lambda, and queues. Which AWS service broadcasts?

SQS delivers to one consumer—not fan out to multiple.
DAX is a DynamoDB cache—not a fanout service.
EBS is a block volume—not a fanout service.
SNS pub/sub fans out one message to email, Lambda, SQS, HTTP endpoints, and more.
Amazon SNS is correct. SNS broadcasts to multiple subscribers simultaneously.
Lesson 0008. SNS.

5. An ISV's multi-tenant SaaS platform emits events that must route to different AWS Lambda functions based on the event source and detail-type field values. The routing rules must be configurable without changing application code. Which AWS service routes events by content?

DynamoDB Streams capture item changes—not route SaaS events.
EventBridge rules route events to targets based on content patterns.
Lambda layers share code—they do not route events.
S3 lifecycle expires objects—it does not route events.
EventBridge rule is correct. Routes based on content, source, and detail fields.
Lesson 0008. EventBridge rules.

6. An expense approval workflow requires conditional branching for manager vs. director approval, timed wait states for SLA enforcement, and per-task retry policies for transient failures. The team also needs a human-readable execution history for auditing. Which AWS service orchestrates this workflow?

API Gateway handles HTTP routing—not workflow orchestration.
Data Firehose delivers streaming data—not workflow orchestration.
Step Functions state machines support waits, branches, retries, and human-readable history.
Secrets Manager stores credentials—not workflow orchestration.
Step Functions is correct. State machines manage complex workflows with built-in control flow.
Lesson 0008. Step Functions.

7. DynamoDB reads need microsecond latency. Which DynamoDB feature provides this?

DLQ captures failed messages—it does not speed up reads.
EBS is an EC2 block volume—it is not for DynamoDB.
MSK is Kafka-managed—it does not cache DynamoDB reads.
DynamoDB Accelerator (DAX) is an in-memory cache providing microsecond read latency.
DAX is correct. DynamoDB Accelerator provides microsecond latency for read-heavy workloads.
Lesson 0010. DAX.

8. A single tenant creates a hot DynamoDB partition. How do you distribute write traffic?

DLQ handles failed messages—it does not fix hot partitions.
Sharding the partition key (e.g., random suffix) distributes writes across partitions.
Scan reads all items—it worsens performance.
X-Ray traces requests—it does not fix hot partition writes.
Shard the key is correct. Random suffix sharding distributes hot partition writes.
Lesson 0010. Partition key design.

9. Failed async Lambda events need later inspection. Which feature isolates poison events?

Provisioned concurrency pre-warms environments—it does not capture failures.
Stage variables pass per-stage values—they do not capture failures.
DLQ captures failed async events for later inspection and reprocessing.
Mapping templates transform requests—they do not capture failures.
Dead-letter queue is correct. DLQ isolates events that exceed retry limits.
Lesson 0002. Lambda DLQ.

10. Durable object storage with lifecycle transitions is needed. Which AWS storage supports lifecycle policies?

EBS volumes do not have lifecycle policies for tiering or expiration.
EFS has lifecycle management for storage classes—not object lifecycle transitions.
DAX is a DynamoDB cache—it does not store objects with lifecycle policies.
S3 supports lifecycle policies that transition objects to cheaper tiers or expire them.
Amazon S3 is correct. S3 lifecycle policies automate object transitions and expirations.
Lesson 0011. S3 lifecycle.
