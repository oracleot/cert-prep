Lesson 0001

DVA-C02 domain triage

Your first exam skill: read a scenario and identify what AWS is really testing before looking at the answer choices.

The win

In the local curriculum, DVA-C02 is split into four domains: Development, Security, Deployment, and Troubleshooting/Optimization. The fastest first move is to classify the question by intent.

Use the DVA-C02 curriculum map while reviewing the source notes in docs/aws dva-c02.md.

The triage algorithm

1. Ask: what is the scenario trying to change? 2. Match the verb to a domain. 3. Then choose the service family. Build / integrate / store data → Domain 1 Authenticate / authorize / encrypt → Domain 2 Package / test / release / rollback → Domain 3 Debug / trace / measure / optimize → Domain 4

Keyword map

  • Domain 1: Lambda invocation, API Gateway mapping, DynamoDB capacity, SQS/SNS/EventBridge, Step Functions, SDK retries.
  • Domain 2: IAM roles, STS, Cognito, permission boundaries, KMS, Secrets Manager, Parameter Store, WAF, Macie.
  • Domain 3: SAM package/deploy, Lambda layers, aliases, CodePipeline, CodeDeploy, CloudFormation, ECR/ECS, AppConfig.
  • Domain 4: CloudWatch metrics/logs, X-Ray traces, Logs Insights, CloudTrail, CodeGuru, Compute Optimizer, cache hits/misses.

Practice: classify first

Do not answer from memory of a service. First classify the domain.

1. A serverless application uses different DynamoDB tables for each API stage. Which domain classifies this integration work?

Domain 1 covers AWS service integration—DynamoDB tables for dev/prod stages is code-to-data-store work, not auth, deployment, or observability.
Domain 2 focuses on authentication, authorization, and encryption controls—not on data store selection.
Domain 3 covers packaging and deployment pipelines, not table selection per API stage.
Domain 4 addresses debugging and optimization; this scenario is about building the app flow.
Domain 1 (Development) is the correct answer. The scenario describes integrating Lambda with DynamoDB—application code connecting to AWS services.
Review the Lambda invocation lesson for the Domain 1 service cluster. See also Lambda supported services.

2. A mobile app needs temporary AWS credentials mapped to least-privilege roles. Which domain handles identity delegation?

Domain 1 covers service integration and data stores—not identity or credential management.
Domain 2 includes authentication, authorization, and temporary credential handling via IAM/STS.
Domain 3 focuses on CI/CD pipelines and deployment, not identity delegation.
Domain 4 addresses observability and optimization, not credential issuance.
Domain 2 (Security) is correct. Temporary credentials mapped to roles is an IAM/STS pattern covered under authentication and authorization.
Study Lesson 0014 for IAM role patterns. IAM documentation.

3. A SAM application must zip artifacts, upload to S3, and produce a deployable template. Which domain covers build automation?

Domain 1 is about service integration, not about packaging or deployment workflows.
Domain 2 covers security controls, not artifact management or SAM workflows.
Domain 3 covers SAM package/deploy, CloudFormation, and CI/CD artifact workflows.
Domain 4 is about debugging and performance—not about artifact packaging.
Domain 3 (Deployment) is correct. SAM package/upload is a core Task 3.1 artifact preparation workflow.
Review Lesson 0020 for SAM commands. SAM CLI reference.

4. An API Gateway and Lambda integration is slower than expected; you must compare responsiveness metrics. Which domain measures performance?

Domain 1 covers service integration and data stores, not performance measurement.
Domain 2 is about security controls, not latency analysis.
Domain 3 handles packaging and deployment, not runtime observability.
Domain 4 covers debugging, tracing, and performance measurement.
Domain 4 (Troubleshooting and Optimization) is correct. Comparing responsiveness metrics is a core observability task.
Study CloudWatch metrics and AWS X-Ray. Lesson 0019.

Practice: service family

5. A team needs to share libraries across multiple Lambda functions without duplicating code in each deployment package. Which Lambda feature enables this?

Aliases point to specific versions—they do not package shared code.
Lambda layers are mounted into the execution environment and shared across functions.
Triggers invoke functions—they do not package or share libraries.
Metrics are monitoring data—they have no role in code sharing.
Lambda layers is correct. Layers mount libraries, custom runtimes, or config files into function execution environments.
See Lesson 0005 for more Lambda configuration patterns. Lambda layers documentation.

6. A deployment strategy must shift Lambda traffic in equal increments every fixed interval. Which CodeDeploy configuration matches this pattern?

Canary shifts a small percentage first, then the remainder—not equal increments.
Linear shift moves equal traffic percentages at fixed intervals, matching the requirement.
Rolling batch replaces instances in groups, not traffic percentage shifts.
Immutable swap deploys to a new fleet entirely before cutting over.
Linear shift is correct. Equal increments at fixed intervals is the defining characteristic of CodeDeploy linear configuration.
Review Lesson 0023 for deployment strategies. CodeDeploy configurations.

7. A distributed system requires distributed traces and service maps to visualize request flow. Which AWS service provides this capability?

X-Ray provides distributed tracing, service maps, and request timing analysis.
Config records resource changes—it does not trace request paths.
Macie discovers sensitive data in S3—it does not provide traces.
KMS manages encryption keys—it does not trace requests.
AWS X-Ray is correct. It is the primary observability tool for distributed tracing and service maps.
Lesson 0019 covers X-Ray service maps. See also X-Ray documentation.

8. An application requires automatic rotation of database credentials that are stored securely. Which AWS service manages credential rotation natively? A team is evaluating AWS Secrets Manager for a PostgreSQL password that must rotate every 30 days without downtime.

Secrets Manager provides built-in automatic rotation for credentials like RDS passwords.
Parameter Store stores parameters but does not rotate credentials automatically.
Certificate Manager handles TLS certificates, not database credentials.
Identity Center provides SSO federation, not credential rotation.
Secrets Manager is correct. Automatic rotation is a key differentiator from Parameter Store for credentials.
Review Lesson 0016. Secrets Manager documentation.
