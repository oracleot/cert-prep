Lesson 0019

Classify Domain 3 deployment clues.

Domain 3 is Deployment: about 20% of DVA-C02, or roughly 13 questions. Your job is to pick the right lifecycle service cluster before the answer choices blur together.

The win

You will recognize whether a scenario is asking about artifacts, development testing, automated testing, or CI/CD deployment.

Related review: Domain 2 security patterns and the Domain 3 intro in docs/aws dva-c02.md.

Domain 3 deployment overview

Deployment questions cover the full application lifecycle: provision resources, manage configuration, package and release code, update software, monitor rollout health, control access, and recover safely.

Exam scenarios often combine services. The clue is not just "deploy"; it is what part of deployment needs control: artifact packaging, environment testing, IaC automation, traffic shifting, rollback, or release orchestration.

The four task statements

  1. 3.1 Prepare application artifacts: dependencies, package structure, repositories, resource requirements, and configuration data.
  2. 3.2 Test in development environments: deployed-code testing, mock API integrations, development endpoints, and stack updates.
  3. 3.3 Automate deployment testing: test events, API environments, approved versions, IaC templates, and service-managed environments.
  4. 3.4 Deploy with CI/CD services: pipelines, deployment strategies, rollbacks, branches, labels, and dynamic runtime configuration.

Deployment service cluster selector

Artifact repository or package storage → CodeArtifact, S3, ECR Serverless packaging and local build → SAM, Lambda layers, container images Infrastructure as code and stack updates → CloudFormation, SAM, CDK Application configuration lifecycle → AppConfig, Parameter Store, Secrets Manager Development endpoints and mock testing → API Gateway stages/mocks, Step Functions Local Version promotion without changing triggers→ Lambda versions and aliases Release orchestration across environments → CodePipeline, CodeBuild, CodeDeploy Traffic shifting, rollback, safe rollout → CodeDeploy, Elastic Beanstalk, ECS, CloudFormation

Fast distractor cuts

  • If the clue says package, dependency, layer, image, or repository, start in Task 3.1.
  • If the clue says mock endpoint, stage variable, dev endpoint, or stack update, start in Task 3.2.
  • If the clue says automated test event, approved version, IaC test environment, or API environment, start in Task 3.3.
  • If the clue says commit, branch, build, deploy stage, rollback, canary, linear, or blue/green, start in Task 3.4.

Retrieval drill: choose the best answer

Domain 3 Deployment is exactly 20 percent of DVA-C02, or roughly 13 questions out of 65.
12 percent understates the weight—Domain 1 alone is closer to that range.
26 percent overstates Domain 3; it is close to Domain 1's actual weight.
32 percent is far too high—Domain 1 (40%) and Domain 2 (30%) absorb most of the exam.
Domain 3 Deployment is 20 percent of DVA-C02, roughly 13 questions. This is the lightest of the four domains.

2. A DVA-C02 scenario describes a workload that stores versioned packages in an S3 bucket before deploying. You must name the artifact stage. Which answer labels the correct artifact operation?

Storing versioned packages before deployment is the Artifacts stage of the CI/CD pipeline, covered by Task 3.1.
Mock APIs are used in development testing (Task 3.2), not for storing packages.
Rollbacks are a deployment safety feature in Task 3.4, not an artifact storage operation.
Branches trigger pipelines (Task 3.4) but do not store or version packages.
Artifacts is correct. Versioned package storage is the prepare stage in Task 3.1, done with S3, CodeArtifact, or ECR before any deployment occurs.
Study Task 3.1 artifact preparation and CodePipeline artifacts.

3. A scenario describes an API Gateway integration that returns test responses without invoking a Lambda backend. You must select the correct integration type. Which answer names the right AWS integration?

MOCK integration makes API Gateway return configured responses with no backend call—perfect for consumer-driven contract testing.
Lambda proxy integration invokes a Lambda function; it requires a live backend.
Step Functions invokes state machines—not a direct API Gateway integration type.
DynamoDB streams capture table changes—they are not an API Gateway integration type.
MOCK is correct. API Gateway MOCK integrations return fixed responses without calling any backend, used in development testing (Task 3.2).
Review Task 3.2 dev testing and API Gateway integration types.

4. A scenario describes a pipeline that builds, tests, and deploys automatically on every code commit. You must name the orchestration service. Which AWS service manages the ordered stages?

CodePipeline orchestrates ordered stages (source, build, test, deploy) triggered by repository changes—Task 3.4.
CloudTrail records API calls; it does not orchestrate CI/CD pipelines.
CloudFront is a CDN; it does not manage deployment pipelines.
CodeGuru reviews code quality; it does not orchestrate build and deploy stages.
CodePipeline is correct. It is the release automation spine that chains source, build, test, and deploy actions on commit events (Task 3.4).
Study Task 3.3 testing automation and CodePipeline.

5. A DVA-C02 scenario involves a Lambda trigger that must keep pointing to the same ARN while function versions change. You must identify the stable pointer. Which Lambda feature provides the unchanging trigger target?

A Lambda alias is a stable, movable pointer to a specific version—event sources can target the alias ARN without updates after version promotion.
Lambda layers share dependencies across functions; they do not serve as stable trigger targets.
Container images package runtime and code but are referenced by image URIs, not used as stable aliases.
An SQS queue is an event source; it does not solve the stable-version-pointer problem.
Lambda alias is correct. Aliases (dev, beta, prod) allow event sources to remain unchanged while versions are promoted—Task 3.2.
Review Task 3.2 Lambda promotion and Lambda aliases.

6. A scenario describes a CodeDeploy Lambda deployment that shifts 10 percent of traffic every 15 minutes. You must name the traffic-shifting configuration. Which deployment configuration performs equal increments at fixed intervals?

Linear shifts equal traffic percentages at fixed intervals (e.g., 10 percent every 15 minutes)—Task 3.3.
Canary shifts a small initial percentage, then shifts the rest at once—fixed intervals are not canary behavior.
In-place replaces the function code on the same instances; it does not shift traffic incrementally.
AllAtATime shifts all traffic immediately—equal increments are not all-at-once behavior.
Linear is correct. CodeDeploy linear configuration shifts traffic in equal increments at fixed intervals (Task 3.3).
Study Task 3.3 traffic shifting and CodeDeploy configurations.

7. A scenario involves feature flags that must be validated before rollout and applied without redeploying code. You must choose the configuration management service. Which AWS service validates and propagates runtime configuration safely?

AppConfig validates configuration profiles and propagates them to applications without redeployment—Task 3.1 configuration lifecycle.
CodeBuild runs build commands; it does not manage runtime feature flags.
CodeArtifact manages package dependencies; it does not propagate runtime configuration.
CloudTrail records API activity; it does not manage application configuration.
AppConfig is correct. It validates configurations and rolls them out safely without redeploying code (Task 3.1).
Review Task 3.1 configuration management and AWS AppConfig.
