Lesson 0023

Deploy code by matching the workload to the CI/CD service.

Task Statement 3.4: select the pipeline, deployment configuration, IaC tool, or rollback control the scenario is really testing.

The win

You will map exam clues to CodePipeline, CodeBuild, CodeDeploy, CloudFormation StackSets, Elastic Beanstalk, SAM/CDK, and rollback triggers.

Source map: Domain 3 Task 3.4 in docs/aws dva-c02.md.

Selector map

Commit starts ordered release stages → CodePipeline Run build commands and test reports → CodeBuild buildspec Traffic shift for Lambda or ECS → CodeDeploy configuration EC2 app update with agent and AppSpec → CodeDeploy EC2/on-prem Same stack across accounts or Regions → CloudFormation StackSets Managed app platform with deployment modes → Elastic Beanstalk Serverless local build and deploy → AWS SAM Rollback on alarm during stack operation → CloudFormation rollback trigger

Traffic-shift shortcut

Lambda and ECS blue/green deployments use canary, linear, or all-at-once traffic shifting. EC2/on-premises deployments use in-place or blue/green, commonly with OneAtATime, HalfAtATime, or AllAtOnce.

Retrieval drill: pick the deployment control

1. A DVA-C02 scenario describes a Git commit that must trigger source, build, approval, and production stages automatically. You must name the orchestration service. Which AWS service chains source, build, test, approval, and deploy stages triggered by repository changes?

CodePipeline orchestrates source, build, approval, and deploy stages in sequence after a repository change—Task 3.4.
CodeArtifact stores package dependencies; it does not orchestrate release stages.
CodeCommit hosts the Git repository; it does not chain pipeline stages.
CodeGuru performs code reviews; it does not orchestrate CI/CD stages.
CodePipeline is correct. It is the release workflow engine that chains all stages, including manual approval gates.
Review Task 3.3 testing automation and CodePipeline.

2. A scenario involves a CodeBuild project where npm install, lint, test, and package commands must execute in sequence. You must name the configuration file. Which YAML file defines the build phases that CodeBuild executes during the project build?

buildspec.yml defines install, pre_build, build, and post_build commands that CodeBuild executes—Task 3.4.
AppSpec file configures CodeDeploy lifecycle hooks; it does not run in CodeBuild.
CloudFormation template defines infrastructure; it does not run build commands.
OpsWorks runbook documents operational steps; it is not a CodeBuild configuration.
buildspec.yml is correct. CodeBuild reads this file to run npm, lint, test, and package commands in defined phases.
Study Task 3.3 testing automation and CodeBuild buildspec.

3. A DVA-C02 scenario describes a CodeDeploy Lambda deployment where traffic shifts to the new version in equal 10 percent increments every 10 minutes. You must name the traffic configuration. Which CodeDeploy Lambda deployment configuration performs equal increments at fixed intervals?

Linear shifts equal traffic percentages at fixed intervals—Task 3.4.
Canary shifts an initial small percentage, then shifts the rest at once; it does not use equal fixed increments.
Blue/green uses a separate environment; it does not incrementally shift traffic.
Immutable deploys to a fresh fleet and terminates the old one; it does not shift traffic incrementally.
Linear is correct. Equal increments at fixed intervals describe the CodeDeploy linear configuration.
Review CodeDeploy configurations and Task 3.3.

4. A scenario involves an ECS production service that must shift traffic from one task set to another during a deployment. You must name the AWS service managing the traffic shift. Which CodeDeploy compute platform manages ECS task set routing during blue/green deployments?

CodeDeploy ECS blue/green deployments shift the production listener between the original and replacement task sets—Task 3.4.
CodeBuild runs build and test commands; it does not manage ECS traffic shifting.
CodeArtifact stores package dependencies; it does not manage ECS deployments.
CodeCommit hosts the source repository; it does not manage ECS task set routing.
CodeDeploy is correct. Its ECS platform shifts the ALB/NLB listener between original and replacement task sets during blue/green deployments.
Study Task 3.3 traffic shifting and CodeDeploy configurations.

5. A DVA-C02 scenario involves an EC2 application deployment where files must be copied and lifecycle hooks must run on instances during the deployment. You must name the configuration file. Which CodeDeploy file specifies hooks, file copy destinations, and application revision for EC2 deployments?

AppSpec file (AppSpec.yml) is required by the CodeDeploy agent for EC2/on-premises deployments—it defines hooks, file copies, and revisions—Task 3.4.
buildspec file runs in CodeBuild; it is not read by the CodeDeploy agent.
ECS task definition defines container settings; it does not configure CodeDeploy EC2 deployments.
CloudFormation template defines infrastructure resources; it does not replace the CodeDeploy AppSpec.
AppSpec file is correct. It is the CodeDeploy agent configuration for EC2/on-premises deployments.
Review Task 3.3 testing automation and AppSpec file.

6. A scenario describes a single CloudFormation template that must create and update the same infrastructure stack across multiple AWS accounts and Regions. You must name the CloudFormation feature. Which CloudFormation capability manages a stack template across accounts and Regions from one operation?

StackSets extend CloudFormation stacks across multiple accounts and Regions from a single template and operation—Task 3.4.
ChangeSets preview changes before applying them; they do not deploy across accounts and Regions.
NestedStacks nest stacks within a parent stack; they do not span accounts and Regions.
WaitHandles synchronize stack creation across resources; they do not span accounts.
CloudFormation StackSets is correct. StackSets enable one template to manage stacks in multiple accounts and Regions simultaneously.
Study CloudFormation StackSets and Task 3.3 cross-account.

7. A DVA-C02 scenario involves a serverless application that must be built, tested, debugged, and deployed locally before shipping to AWS. You must name the serverless IaC tool. Which AWS tool provides local build, test, debug, and deploy capabilities specifically for Lambda, API Gateway, and DynamoDB serverless applications?

AWS SAM provides local build, test, debug, and deploy for serverless applications (Lambda, API Gateway, DynamoDB)—Task 3.4.
AWS FIS (Fault Injection Simulator) runs chaos experiments; it does not build or deploy serverless applications.
AWS DMS (Database Migration Service) migrates databases; it does not build or deploy serverless applications.
AWS WAF (Web Application Firewall) enforces security rules; it does not build or deploy serverless applications.
AWS SAM is correct. The SAM CLI provides local equivalents of cloud operations for Lambda and API Gateway development.
Review Task 3.1 artifact preparation and AWS SAM.

8. A scenario involves a CloudFormation stack update where the operation must roll back automatically if a CloudWatch alarm enters ALARM state. You must name the CloudFormation feature. Which CloudFormation stack update feature monitors alarms and rolls back on alarm breach during create or update operations?

Rollback triggers attach CloudWatch alarms to a stack update and automatically roll back if the alarm enters ALARM state—Task 3.4.
API Gateway stage variables pass environment-specific values to Lambda; they do not trigger stack rollbacks.
Parameter Store labels mark versioned parameters; they do not trigger stack rollbacks.
CodeStar branches manage project timelines; they do not trigger CloudFormation rollbacks.
Rollback trigger is correct. It monitors CloudWatch alarms during stack operations and initiates automatic rollback on failure.
Review CloudFormation rollback triggers and Task 3.3.
