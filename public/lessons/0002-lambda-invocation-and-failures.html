Lesson 0002

Lambda: know how the event enters, then how failure is handled.

DVA-C02 Lambda questions usually hide the answer in the invocation path: synchronous, asynchronous, or event source mapping.

The exam move

Before picking a Lambda feature, ask: who invokes the function? The caller determines retry behavior, error handling, and whether Lambda reads from a source in batches.

Keep the Lambda quick sheet open while you practice.

Three invocation patterns

1. Synchronous

The caller waits. If Lambda succeeds, the caller gets the response. If it fails, the caller sees the error and must decide whether to retry. The AWS CLI command is aws lambda invoke.

2. Asynchronous

The caller hands off the event and does not wait. Lambda queues the event, runs the function, retries function errors twice, and can send invocation records to destinations or a dead-letter queue.

3. Event source mapping

Lambda polls SQS, Kinesis, or DynamoDB Streams and sends records to your function in batches. With Kinesis and DynamoDB Streams, shards are the concurrency unit because each shard is processed in order.

Failure routing mental model

Caller waits? → synchronous → caller handles retry Caller drops event? → asynchronous → Lambda retries, destinations or DLQ Lambda polls source? → event source mapping → source rules, batch behavior, redrive policy

Common DVA-C02 traps

  • Retries require idempotency. Your function may process the same event more than once.
  • Event source mapping is a Lambda resource. Use it for SQS, Kinesis, and DynamoDB Streams.
  • Memory affects CPU. Increasing memory can improve CPU-bound performance.
  • Reserved concurrency is a control. It can cap a function so it does not consume too much account concurrency.
  • Provisioned concurrency targets cold starts. It keeps environments initialized for low latency.

Practice: classify the Lambda path

1. An API Gateway request invokes Lambda and waits for a response.

2. S3 sends object-created events to Lambda without waiting for function output.

3. Lambda reads records from an SQS queue in batches.

Practice: choose the feature

4. A function must avoid cold-start latency for a production API.

5. A function should not consume all available account concurrency.

6. Failed asynchronous events need to be captured for later debugging.
