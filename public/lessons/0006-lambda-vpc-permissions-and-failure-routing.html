Lesson 0006

Lambda in a VPC needs network config first, permissions second.

DVA-C02 scenarios often mix VPC access, execution-role permissions, and failure-routing distractors. Classify the problem before choosing a feature.

The win

When a Lambda question mentions private resources, ask three questions in order: Where is it attached? What can its role do? Where do failed async events go?

Use this with the Lambda quick sheet and Lesson 0002.

The decision path

Needs RDS/ElastiCache/private subnet resource? → configure Lambda VPC subnets + security groups → execution role needs EC2 network-interface permissions → outbound internet needs NAT, not just a public subnet Async failure routing? → destinations for success/failure records → DLQ only for failed async events

VPC access keyword pattern

Choose Lambda VPC configuration when the function must reach resources inside a VPC, such as private RDS, ElastiCache, or internal services. You select subnets and security groups; Lambda creates and manages elastic network interfaces.

ENI permission pattern

The function execution role must allow Lambda to manage network interfaces. The exam-friendly answer is usually the AWS managed policy AWSLambdaVPCAccessExecutionRole, which includes EC2 network-interface permissions plus CloudWatch Logs permissions.

Subnet and security group pattern

For private database access, put Lambda in subnets that can route to the target and assign a security group that the target allows inbound. If the function also needs public internet access from private subnets, look for a NAT gateway.

Destinations vs DLQ pattern

For asynchronous invocation, destinations can route success or failure records to SQS, SNS, Lambda, or EventBridge. A DLQ captures failed async events only, usually to SQS or SNS, and gives less context than destinations.

Exam traps

  • Resource policy is not execution role. Invocation permission lets a service call Lambda; execution role lets Lambda call AWS APIs.
  • VPC attachment is not internet access. A Lambda in private subnets needs NAT for outbound public internet.
  • Security groups are still enforced. The database must allow the Lambda security group inbound.
  • DLQ is not for success. Choose destinations when the scenario needs success and failure routing.
  • Console test events are payload tests. They do not prove private subnet routing, NAT, or security group access.

Practice: classify the missing piece

1. A Lambda function must connect to an RDS database in private subnets. What Lambda configuration enables private network access?

VPC configuration attaches Lambda to private subnets, allowing ENI-based access to RDS in private subnets.
DLQ captures failed async events—it does not enable VPC access.
Alias swap changes which version is targeted—it does not enable VPC access.
Lambda layer packages shared code—it does not enable VPC access.
VPC config is correct. Lambda needs VPC configuration with subnets and security groups to connect to private RDS.
Lesson 0005. Lambda VPC config.

2. Lambda VPC setup fails because it cannot create network interfaces. Which IAM permission is missing from the execution role?

KMS decrypt permissions are for encryption operations—not for creating network interfaces.
Lambda needs EC2 CreateNetworkInterface, DescribeNetworkInterfaces, and DeleteNetworkInterface permissions for VPC access.
S3 read permissions enable object access—they do not affect ENI creation.
SNS send permissions are for publishing messages—they do not affect ENI creation.
EC2 ENI permissions are correct. The execution role needs network interface permissions for Lambda to connect to VPC resources.
Lesson 0005 covers Lambda VPC configuration. See Lambda VPC permissions.

3. A Lambda function in private subnets must call a public third-party API. Which AWS resource enables outbound internet from private subnets?

A NAT gateway in a public subnet routes outbound traffic from private subnet instances to the internet.
DLQ targets capture failed events—they do not enable internet access.
Lambda layers mount shared code—they do not enable internet routing.
Alias shift changes version targets—it does not enable internet access.
NAT gateway is correct. Private subnet resources use NAT gateway to reach the internet without direct public IPs.
Lesson 0005 covers VPC networking patterns. See NAT gateway.

4. Async Lambda results need different targets for success and failure. Which async feature routes both outcomes to separate destinations?

Lambda destinations route success and failure records to different targets (SQS, SNS, Lambda, EventBridge).
Redrive jobs move messages in queues—they do not route async invocation results.
Stage variables pass per-API-stage values—they do not route invocation results.
Mock tests validate payloads—they do not route invocation results.
Destination is correct. Destinations can independently route success and failure records to different targets.
Lesson 0002. Lambda destinations.

5. A service needs permission to invoke the Lambda function. Which policy controls who can call Lambda?

Lambda resource-based policies control which principals can invoke the function.
Execution role controls what Lambda can call—it does not control who can invoke Lambda.
Subnet settings control network access—they do not control invocation permissions.
Memory tuning affects execution resources—it does not control invocation permissions.
Resource policy is correct. Resource-based policies grant invocation permissions to services and accounts.
Lesson 0014. Lambda resource policies.
