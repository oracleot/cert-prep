Lesson 0017

Secure traffic before you secure code.

Task Statement 2.4: choose the AWS feature that protects traffic paths: public HTTPS, private AWS API calls, or subnet-level filtering.

The win

You will classify security clues into four exam moves: TLS with ACM, API Gateway custom domains, VPC endpoints, and security group vs NACL.

Use this after the Domain 2 source notes in docs/aws dva-c02.md.

1. Public HTTPS: ACM + the front door

Need HTTPS certificate → AWS Certificate Manager Need API vanity hostname → API Gateway custom domain Need DNS to the API → Route 53 alias / CNAME Need minimum TLS version → API Gateway security policy
  • ACM issues, imports, stores, and renews public/private TLS certificates for integrated AWS services.
  • API Gateway custom domains map names such as api.example.com to API stages; the custom domain uses an ACM certificate.
  • Regional API Gateway custom domains use an ACM certificate in the same Region; edge-optimized custom domains use certificates in us-east-1.
  • WAF is not TLS. Choose it for layer 7 filtering: SQL injection, XSS, IP sets, rate-based rules, and managed rule groups.

2. Private AWS traffic: endpoint first, NAT second

VPC endpoint clue

If private subnets must reach AWS services without internet gateways, NAT gateways, public IPs, or VPN paths, choose a VPC endpoint.

Gateway endpoint S3 and DynamoDB, route table target, no ENI Interface endpoint PrivateLink ENIs, security groups, most AWS APIs Endpoint policy Restrict which actions/resources use the endpoint Bucket policy Restrict S3 access with aws:sourceVpce

3. Network filters: stateful vs stateless

  • Security groups attach to ENIs, are stateful, and only allow traffic. Return traffic is automatically allowed.
  • NACLs attach to subnets, are stateless, can allow or deny, and evaluate rules by number.
  • Ephemeral port trap: NACLs need rules for both request and return traffic, including ephemeral ports. Security groups do not.
  • DVA clue: app-to-database access usually means security groups; subnet-wide explicit deny usually means NACL.

Retrieval drill: classify the control

1. An API Gateway REST API needs api.example.com with HTTPS. The team wants a clean hostname for mobile clients. Which combination of services is required?

ACM provides the TLS certificate and API Gateway custom domain maps api.example.com to the API stage.
Route 53 alias routes DNS but does not provide TLS certificates or custom domain mapping.
NACL and security groups filter traffic—they do not provide HTTPS hostname or TLS certificates.
Shield Standard provides DDoS protection; CloudFront is a CDN—they do not replace API Gateway custom domains.
ACM certificate + API Gateway custom domain is correct. ACM provisions and renews the TLS cert; the custom domain maps the hostname to the API stage.
Study ACM certificate management and API Gateway custom domains.

2. A Lambda function in a private VPC subnet must call S3 without using a NAT Gateway or internet gateway. Which VPC component enables this?

S3 gateway endpoint adds an S3 prefix to the VPC route table, routing S3 traffic privately without internet access.
Interface VPC endpoints provide private access to services with ENIs—S3 uses gateway endpoints, not interface endpoints.
NAT Gateway requires an internet gateway—it is the opposite of private VPC access.
Route 53 inbound resolver handles DNS queries—it does not provide S3 private access.
S3 gateway VPC endpoint is correct. It routes S3 traffic through a VPC endpoint prefix in the route table with no internet required.
Review VPC gateway endpoints and Gateway VPC endpoints.

3. An application in private subnets must call Secrets Manager APIs privately. There is no internet gateway. Which VPC component is needed?

Secrets Manager Interface VPC endpoint (PrivateLink) creates a private ENI in the subnet for Secrets Manager API calls.
S3 gateway endpoint only routes S3 traffic—it does not provide access to Secrets Manager APIs.
NAT Gateway requires an internet gateway and cannot be placed directly in a private subnet without a public NAT.
Edge-optimized API Gateway is publicly accessible—it does not provide private VPC access to Secrets Manager.
Secrets Manager Interface VPC endpoint is correct. AWS PrivateLink creates private ENIs for Secrets Manager with no internet traffic.
Study Secrets Manager VPC access and VPC endpoints.

4. A Lambda function behind an ALB in a VPC subnet needs an explicit deny for inbound traffic from a specific CIDR range while allowing all other traffic. Which network control is appropriate?

NACLs are stateless and can both allow and deny rules, enabling explicit subnet-level denies for specific CIDRs.
Security groups are stateful and only allow traffic—they cannot explicitly deny by CIDR.
ACM manages TLS certificates—it does not control subnet-level network traffic.
Lambda function URL auth controls access to Lambda—not subnet-level traffic filtering.
Network ACL is correct. NACLs are stateless and support explicit deny rules at the subnet level.
Review NACL vs security groups and Network ACLs.

5. A Lambda function in a VPC subnet experiences NACL rule failures after a rule change causes outbound API calls to fail. The NACL was changed to block ephemeral ports. What is the root cause?

NACLs evaluate both directions of traffic. Ephemeral return traffic (typically ports 32768–65535) must be explicitly allowed in stateless NACL rules.
Security groups are stateful and automatically allow return traffic—stateless NACLs do not.
Cold starts do not cause NACL-level ephemeral port failures.
KMS key policy revocation would cause decrypt failures, not NACL ephemeral port failures.
NACL statelessness is correct. Ephemeral port ranges must be explicitly allowed on both directions of a NACL rule.
Study NACL ephemeral port rules and Network ACLs.

6. An API Gateway REST API must block SQL injection and XSS patterns in incoming requests. Which AWS service provides this filtering?

AWS WAF inspects HTTP(S) requests and blocks SQL injection, XSS, IP sets, and other layer 7 attack patterns using managed or custom rules.
ACM manages TLS certificates—it does not filter HTTP requests or block attack patterns.
STS provides temporary credentials—it does not inspect or filter HTTP requests.
KMS manages encryption keys—it does not filter HTTP requests or block SQL injection.
AWS WAF is correct. It is the layer 7 web application firewall that blocks SQL injection and XSS patterns at API Gateway.
Review WAF web filtering rules and AWS WAF.
