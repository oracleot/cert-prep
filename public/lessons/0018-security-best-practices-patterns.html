Lesson 0018

Pick the best-practice security pattern.

Task Statement 2.5: eliminate distractors by matching the risk to the smallest AWS control that fixes it.

The win

You will choose fast among S3 encryption/access, private service access, network controls, and edge protection.

Best-practice selector

Encrypt objects with AWS-managed key → SSE-S3 | Audit/control customer-managed key use → SSE-KMS | Temporary object download/upload → S3 presigned URL Keep bucket private behind CDN → CloudFront OAC | Detect sensitive data in S3 → Amazon Macie | Block layer 7 attack patterns → AWS WAF Baseline DDoS protection → Shield Standard

1. S3 access traps

  • Block Public Access is the first cut when a bucket should not be public.
  • Bucket policies enforce conditions such as required encryption or access only through a VPC endpoint.
  • Presigned URLs grant temporary object access without making the bucket public.
  • CloudFront OAC lets CloudFront access a private S3 origin; users should not access the bucket directly.

2. S3 encryption traps

  • Default encryption handles new objects; a bucket policy can deny uploads that omit the required encryption header.
  • SSE-C means the customer supplies keys for S3 to use; AWS does not store those keys.
  • KMS key policy controls key use. An S3 bucket policy alone cannot grant KMS decrypt permission.
  • Exam shortcut: prefer SSE-KMS for compliance/audit; SSE-S3 for transparent encrypt at rest.

3. Edge and detection controls

  • AWS WAF: choose for API Gateway, CloudFront, and ALB web request filtering.
  • Shield Standard: automatic DDoS protection for common infrastructure attacks.
  • Shield Advanced: choose when the scenario asks for enhanced DDoS response, cost protection, or DRT access.
  • Macie: choose for discovering sensitive data in S3, not for blocking live HTTP requests.

Mixed drill: choose the best answer

1. A private S3 bucket needs temporary download links for authenticated users without making the bucket public. Which S3 feature grants time-limited access?

S3 presigned URLs grant temporary access to specific objects using the owner's credentials without changing bucket access.
Public read ACL makes the bucket publicly accessible—violating the requirement for a private bucket.
NACLs filter subnet traffic—they do not grant S3 object-level temporary download access.
Shield Standard protects against DDoS attacks—it does not provide temporary object download access.
S3 presigned URL is correct. It grants temporary object access using the bucket owner's credentials without exposing the bucket publicly.
Review S3 presigned URLs and S3 presigned URLs.

2. A compliance team requires customer managed KMS keys with key policies that audit key usage and control who can decrypt. Which S3 encryption mode fits this requirement?

SSE-KMS with a customer managed key provides key policies, CloudTrail audit logs of key usage, and rotation controls.
SSE-S3 uses AWS managed keys with no customer key policy or audit trail of key usage.
SSE-C stores no key metadata—AWS cannot audit or log key usage because the key is never stored.
TLS-1.2 encrypts data in transit, not at rest—it does not address SSE requirements.
SSE-KMS with customer managed key is correct. Key policies and CloudTrail logging of KMS API calls satisfy audit and control requirements.
Study S3 SSE-KMS audit trails and S3 encryption.

3. A bucket policy must deny any S3 PUT that does not include an encryption header. Which policy statement achieves this?

A bucket policy condition s3:x-amz-server-side-encryption:aws:kms enforces encryption on every PUT without encryption header.
Deny ListBucket does not address encryption headers on object uploads.
Allow from CloudFront controls source but does not enforce encryption headers on PUT.
aws:sourceVpce restricts access to a specific VPC endpoint—it does not enforce encryption headers.
Deny with s3:x-amz-server-side-encryption condition is correct. It blocks any PUT that omits the encryption header.
Review S3 bucket policy conditions and S3 bucket policies.

4. CloudFront must serve objects from a private S3 origin while preventing users from accessing the bucket directly. Which CloudFront and S3 feature pair is correct?

OAC grants CloudFront access to the private S3 bucket while S3 block public access prevents direct bucket access.
A public bucket policy contradicts the requirement to prevent direct access to the bucket.
Interface VPC link connects ALB to Lambda—it does not relate to CloudFront S3 origins.
NACL rules filter subnet traffic—they do not control CloudFront-to-S3 origin access patterns.
CloudFront OAC + S3 block public access is correct. OAC grants CloudFront access; block public access prevents direct bucket access.
Study CloudFront OAC configuration and CloudFront OAC.

5. A data governance team needs to discover and classify sensitive data already stored across multiple S3 buckets. Which AWS service scans S3 for PII?

Amazon Macie uses machine learning to automatically discover and classify sensitive data such as PII in S3 buckets.
Shield Advanced provides DDoS cost protection and DRT access—it does not scan S3 for sensitive data.
Route 53 is a DNS service—it does not scan S3 for PII or sensitive data.
ACM private certificates manage TLS—it does not discover or classify sensitive data in S3.
Amazon Macie is correct. It automatically discovers, classifies, and alerts on sensitive data in S3 using machine learning.
Review Macie data discovery and Amazon Macie.

6. An API Gateway REST API must block XSS attack patterns in incoming requests. Which AWS service provides layer 7 filtering?

AWS WAF inspects HTTP requests and blocks XSS patterns using managed rule groups or custom rules.
ACM manages TLS certificates—it does not inspect HTTP request bodies for XSS patterns.
KMS manages encryption keys—it does not filter HTTP requests or block XSS attacks.
STS issues temporary credentials—it does not inspect or filter HTTP requests.
AWS WAF with XSS rule is correct. WAF is the layer 7 web application firewall that blocks XSS, SQLi, and other attack patterns at API Gateway.
Study WAF rule configuration and AWS WAF.

7. A production workload requires DDoS response with cost protection and access to the DDoS Response Team (DRT). Which AWS Shield tier includes these features?

Shield Advanced includes DRT access, DDoS cost protection for EC2, ELB, CloudFront, and Route 53, and enhanced attack visibility.
Shield Standard provides automatic DDoS protection for common layer 3 and 4 attacks with no cost protection or DRT access.
WAF managed rule groups filter layer 7 traffic—they do not provide DDoS cost protection or DRT access.
ACM public certificates provide TLS—it has no DDoS protection, cost protection, or DRT features.
Shield Advanced is correct. DRT access and DDoS cost protection are exclusive to Shield Advanced, not Shield Standard.
Review Shield tier comparison and AWS Shield.

8. An S3 bucket must allow access only through one VPC endpoint. All other access paths, including direct internet access, must be denied. Which condition key enforces this?

aws:sourceVpce in a bucket policy restricts access to requests originating from the specified VPC endpoint ID, blocking all other access paths.
aws:SourceIp filters by IP address—it does not restrict access to a specific VPC endpoint.
aws:username identifies IAM users—it does not restrict S3 access to a specific VPC endpoint.
aws:userid in an SCP controls IAM user identifiers—it does not enforce VPC endpoint access restrictions.
aws:sourceVpce is correct. Bucket policy condition aws:sourceVpce restricts access to traffic from the specified VPC endpoint only.
Study S3 bucket policy VPC endpoint conditions and S3 access conditions.

9. A private S3 bucket must serve authenticated downloads without making the bucket public. Which S3 and CloudFront features are valid for this requirement? Choose TWO.

S3 presigned URLs grant time-limited object access to specific users without making the bucket public or changing bucket policy.
CloudFront with OAC can serve private S3 objects while preventing direct bucket access—the bucket stays private and the CloudFront distribution handles authentication.
Marking the bucket public exposes every object to the internet and defeats the private-bucket requirement; cron-based policing is not an access control mechanism.
Long-term access keys for each downloader violate least privilege and are not scalable; short-lived STS credentials or presigned URLs are the exam-accepted answer.
Presigned URLs and CloudFront with OAC are the two standard ways to grant temporary, controlled access to private S3 object content. Both keep the bucket private.
Review S3 access patterns and CloudFront OAC.

10. A compliance team needs to discover sensitive data already stored in S3 and filter layer 7 attack patterns in incoming web requests. Which AWS services are appropriate for these requirements? Choose TWO.

Macie uses machine learning to discover and classify sensitive data such as PII in S3 buckets, and findings can trigger EventBridge rules for remediation workflows.
AWS WAF filters layer 7 attack patterns like SQL injection and XSS on CloudFront, API Gateway, and ALB using managed or custom rule groups.
Shield Standard protects against layer 3 and 4 DDoS attacks; it does not scan S3 object contents for sensitive data.
CloudTrail records AWS API calls and resource changes; it does not scan S3 object contents for sensitive data or redact findings.
Macie finds sensitive data in S3 and AWS WAF filters layer 7 attacks. Shield Standard handles DDoS protection and CloudTrail records API calls; neither scans S3 contents.
Review Macie and WAF patterns and Amazon Macie.
