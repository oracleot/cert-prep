# AWS DVA-C02 Resources

## Knowledge

- [Local curriculum: `docs/aws dva-c02.md`](docs/aws%20dva-c02.md)
  Primary course map for domains, review lessons, walkthrough timing, SimuLearn labs, and Builder Labs.
- [AWS Certified Developer - Associate exam guide](https://aws.amazon.com/certification/certified-developer-associate/)
  Use for validating domain weights, task statements, and official exam scope.
- [AWS Lambda Developer Guide](https://docs.aws.amazon.com/lambda/latest/dg/welcome.html)
  Use for invocation modes, event source mappings, concurrency, destinations, DLQs, permissions, and performance tuning.
- [Amazon API Gateway Developer Guide](https://docs.aws.amazon.com/apigateway/latest/developerguide/welcome.html)
  Use for method/integration requests, mapping templates, stages, variables, throttling, caching, authorizers, and mock integrations.
- [Amazon DynamoDB Developer Guide](https://docs.aws.amazon.com/amazondynamodb/latest/developerguide/Introduction.html)
  Use for capacity math, consistency, indexes, hot partitions, streams, DAX, and partition-key design.
- [AWS IAM User Guide](https://docs.aws.amazon.com/IAM/latest/UserGuide/introduction.html)
  Use for roles, policies, permission boundaries, SCPs, federation, identity providers, and least privilege.
- [AWS KMS Developer Guide](https://docs.aws.amazon.com/kms/latest/developerguide/overview.html)
  Use for KMS keys, key policies, envelope encryption, GenerateDataKey, rotation, and integrations.
- [AWS SAM Developer Guide](https://docs.aws.amazon.com/serverless-application-model/latest/developerguide/what-is-sam.html)
  Use for serverless packaging, build, local testing, deployments, and CloudFormation transform behavior.
- [AWS CodePipeline User Guide](https://docs.aws.amazon.com/codepipeline/latest/userguide/welcome.html)
  Use for CI/CD orchestration, source-build-deploy stages, cross-account deployments, and rollback patterns.
- [Amazon CloudWatch User Guide](https://docs.aws.amazon.com/AmazonCloudWatch/latest/monitoring/WhatIsCloudWatch.html)
  Use for logs, metrics, alarms, dashboards, Logs Insights, embedded metric format, and observability.
- [AWS X-Ray Developer Guide](https://docs.aws.amazon.com/xray/latest/devguide/aws-xray.html)
  Use for traces, segments, annotations, filter expressions, service maps, and distributed troubleshooting.
