Lesson 0020

Prepare the artifact before you deploy.

Task Statement 3.1: choose the build output, package command, repository, or configuration source the scenario is really testing.

The win

You will separate dependency artifacts, serverless packages, container images, and runtime configuration without over-selecting CI/CD services.

Source map: Domain 3 Task 3.1 in docs/aws dva-c02.md.

Artifact selector

Package dependencies for teams → AWS CodeArtifact Store SAM/Lambda zip artifacts → Amazon S3 Build Lambda-native dependencies → sam build --use-container Zip and upload SAM artifacts → sam package Create or update the stack → sam deploy Declare SAM inside CloudFormation → Transform Store container deployment images → Amazon ECR Point traffic without changing ARNs → Lambda alias

Exam shortcut

If the template includes SAM resources, look for Transform: AWS::Serverless-2016-10-31. If the question says deploy code plus dependencies, decide between zip, layer, or container image.

Retrieval drill: pick the deployment clue

1. Several development teams publish internal npm and Python packages that must be shared across AWS accounts. The company wants a fully managed repository that works with common package managers and stores versioned artifacts. Which AWS service should the developer use?

Option A is the correct answer. CodeArtifact is the managed repository service for package formats such as npm, pip, Maven, and NuGet.
Option B is incorrect. CloudFormation provisions infrastructure, but it does not host dependency packages for developers.
Option C is incorrect. CodeDeploy handles deployments, not package repository management.
Option D is incorrect. AppConfig delivers runtime configuration, not versioned language packages.
Option A is the correct answer. Managed package hosting for development dependencies points to CodeArtifact.
Review CodeArtifact documentation and Lesson 0023.

2. A team maintains an AWS SAM application with several Lambda functions. Before the stack can be deployed, the build process must zip the local artifacts, upload them to Amazon S3, and generate a deployment-ready template that references those uploaded objects. Which SAM CLI command should the developer run?

Option A is the correct answer. sam package uploads local artifacts to S3 and rewrites the template so it can be deployed later.
Option B is incorrect. sam deploy creates or updates the stack after packaging is complete, but it is not the packaging step itself.
Option C is incorrect. sam validate checks template syntax and structure without uploading artifacts.
Option D is incorrect. sam publish targets the Serverless Application Repository and is not the standard answer for packaging deployment artifacts.
Option A is the correct answer. Zip plus S3 upload plus template rewrite is the exact job of sam package.
Study sam package and Lesson 0023.

3. A Python Lambda function depends on native libraries that must be compiled against the same Amazon Linux environment used by the Lambda runtime. The developer wants the SAM build step to run in a compatible container locally. Which command should the developer use?

Option A is the correct answer. The --use-container flag builds dependencies inside a Lambda-compatible container image.
Option B is incorrect. Guided deploy helps collect deployment parameters, but it does not compile native dependencies in a matching runtime environment.
Option C is incorrect. Deleting a stack has nothing to do with building deployment artifacts.
Option D is incorrect. Traces are for observability, not packaging native Lambda dependencies.
Option A is the correct answer. Native dependency compatibility is the clue for sam build --use-container.
Review sam build and Lambda deployment packages.

4. A developer receives a CloudFormation template that declares AWS::Serverless::Function and AWS::Serverless::Api resources. CloudFormation must preprocess those serverless resources into standard resources before creating the stack. Which top-level template key is required?

Option A is the correct answer. Transform: AWS::Serverless-2016-10-31 invokes the SAM transform that expands serverless resources for CloudFormation.
Option B is incorrect. Mappings provide lookup values and do not run the SAM macro.
Option C is incorrect. Outputs expose stack values after deployment and do not preprocess resources.
Option D is incorrect. Metadata stores supporting information and does not tell CloudFormation to transform SAM resources.
Option A is the correct answer. The required top-level SAM declaration is the Transform key.
Study the SAM transform and Lesson 0023.

5. A company has 25 Lambda functions, and each deployment package contains the same shared image-processing libraries. The total package size is growing, and the team wants to update the shared dependencies without copying them into every function zip. Which feature should the developer use?

Option A is the correct answer. Layers package shared libraries separately so multiple Lambda functions can reuse them without duplication.
Option B is incorrect. Aliases provide stable version pointers, not shared dependency packaging.
Option C is incorrect. Event source mappings connect streams or queues to Lambda and do not reduce package duplication.
Option D is incorrect. Resource policies control who can invoke a function and do not manage code dependencies.
Option A is the correct answer. Reusable shared libraries across many Lambda functions point to Lambda layers.
Review Lambda layers and Lesson 0021.

6. An application is moving to Amazon ECS, and the release process must store immutable container images in an AWS-managed registry before deployment. The same registry should also support Lambda container image functions. Which AWS service is the best fit?

Option A is the correct answer. Amazon ECR is the AWS-managed container registry for ECS images and Lambda container image packages.
Option B is incorrect. EFS is a network file system and does not act as a Docker image registry.
Option C is incorrect. SQS is a message queue, not an artifact store for container images.
Option D is incorrect. A VPC provides networking boundaries and does not store deployment images.
Option A is the correct answer. AWS-managed container image storage for ECS and Lambda means Amazon ECR.
Study Amazon ECR and Lesson 0023.

7. An EC2 instance is launched from a CloudFormation stack, and the template metadata must install packages, create configuration files, and start services during boot. Which CloudFormation helper script should the instance run to apply the metadata?

Option A is the correct answer. cfn-init reads AWS::CloudFormation::Init metadata and configures the instance during bootstrap.
Option B is incorrect. cfn-lint validates templates locally and does not configure running EC2 instances.
Option C is incorrect. cfn-flip converts CloudFormation template formats and is not a bootstrap helper.
Option D is incorrect. cfn-diff compares templates and does not execute instance setup tasks.
Option A is the correct answer. Metadata-driven EC2 bootstrapping is the cfn-init pattern.
Review CloudFormation helper scripts and Lesson 0021.

8. A retailer stores feature flags and throttle settings outside the application code. The operations team wants schema validation, gradual rollout, CloudWatch alarm monitoring, and automatic rollback if a bad configuration causes errors in production. Which AWS service should the developer use?

Option A is the correct answer. AppConfig validates configuration data, deploys it gradually, watches alarms, and rolls back automatically when needed.
Option B is incorrect. CodeBuild compiles and tests source code, but it does not manage runtime configuration rollouts.
Option C is incorrect. CodeStar organizes project resources and is not the best answer for safe runtime configuration deployment.
Option D is incorrect. CloudTrail records API calls and does not validate or distribute application configuration.
Option A is the correct answer. Validated runtime configuration with progressive rollout and rollback is the AppConfig use case.
Study AWS AppConfig and Lesson 0021.
