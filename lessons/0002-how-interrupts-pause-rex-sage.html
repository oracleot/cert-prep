Lesson 0002

Interrupts are where the graph waits for the human.

Your win: know exactly why the Rex/Sage graph pauses twice, and how the API resumes it.

The exact line

In agents/graphs/session.py, the graph is compiled like this:

raw = build_session_graph().compile(
    checkpointer=get_checkpointer(),
    interrupt_before=["evaluate_answer", "rex_rechallenge"]
)

That means LangGraph stops before those nodes run. It saves the current state under the session's thread_id. Official background: LangGraph persistence.

Interrupt 1: before evaluate_answer

/session/start → coach_open # picks concept, sets cycle = 1 → rex_challenge # writes current_challenge ✋ interrupt before evaluate_answer

This pause exists because evaluate_answer needs user_answer, and the user has not answered yet.

At this point, /session/start reads the checkpoint and returns the generated challenge to the frontend.

Resume 1: submit the answer

When the user answers, /session/submit does two things:

await graph.aupdate_state(config, {
    "user_answer": user_answer,
    "answer_intent": answer_intent,
})

async for event in graph.astream_events(None, config=config):
    ...

aupdate_state fills in the missing state. astream_events(None, ...) resumes from the bookmark, so evaluate_answer finally runs.

Then Sage runs

evaluate_answer → route_after_evaluation → sage_depth # if correct or sage_explain # if incorrect

The evaluator writes last_evaluation. The route function reads it. Sage streams tokens back to the client while it generates the teaching response.

Interrupt 2: before rex_rechallenge

sage_depth / sage_explain → route_after_sage ✋ interrupt before rex_rechallenge

This pause exists so the user can read Sage's response before Rex immediately throws the next challenge.

The snapshot now has snapshot.next = ["rex_rechallenge"]. The route uses that to report phase sage_done.

Resume 2: ask for the next challenge

When the user clicks next, /session/next calls:

await graph.ainvoke(None, config=config)

That resumes from rex_rechallenge. Rex generates a harder challenge, increments cycle, then the graph follows the edge back toward evaluate_answer and pauses there again.

What happens at the end?

After the final Sage response, route_after_sage returns coach_close instead of rex_rechallenge. There is no interrupt. The graph closes the session and reaches END.

Retrieval check

The graph is paused with snapshot.next = ["evaluate_answer"]. What is it waiting for?
