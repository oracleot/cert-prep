Lesson 0021

Test deployments without touching production.

Task Statement 3.2: pick the safest dev/test pattern when the scenario mentions versions, mock APIs, stages, templates, or EC2 bootstrapping.

The win

You will classify Domain 3 test-environment questions by the clue: deployment risk, Lambda promotion, API dependency, SAM template, or CloudFormation helper script.

Source review: Domain 3 Task 3.2 in docs/aws dva-c02.md.

Fast selector

Least time, highest blast radius → all-at-once Major runtime/platform change → blue/green Stop app, install version, validate → in-place EC2 package/files/services from metadata → cfn-init Wait for EC2 config before stack continues → cfn-signal Deploy one SAM template, not project root → sam deploy --template Backend not ready, API consumers must test → API Gateway mock integration Same API, different dev/test config → API Gateway stages + variables Stable event source while Lambda changes → Lambda alias ARN

Exam shortcut

Versions are immutable. Aliases are movable pointers. Point S3, API Gateway, or other event sources at the alias ARN so promotion only changes the alias target.

Mixed drill: choose the best answer

1. A company is building a new mobile backend with Amazon API Gateway. The frontend team must begin integration testing this week, but the Lambda function for the GET /profile method will not be finished for another sprint. The API team needs API Gateway to return a predefined JSON payload without calling any backend service. Which integration type should a developer configure?

Option A is correct because a MOCK integration lets API Gateway generate a configured response without invoking a backend.
Option B is a distractor because HTTP proxy integration still requires a live HTTP backend endpoint.
Option C is a distractor because Lambda proxy integration invokes a Lambda function, and the stem says the backend is not ready.
Option D is a distractor because a VPC link connects API Gateway to private integrations, not to static mock responses.
Option A is the correct answer. The requirement is to return test data before any backend exists, which is exactly what an API Gateway MOCK integration is for.
Review Task 3.1 artifact preparation and API Gateway mock integrations.

2. A media company publishes new versions of a Lambda function after each sprint. An Amazon SQS queue already invokes the function in the beta environment, and the operations team does not want to update the event source mapping every time a new function version is promoted. Which Lambda feature should the queue target?

Option A is correct because a Lambda alias is a stable pointer that can be moved to a new version without changing the invoking resource.
Option B is a distractor because a layer version packages shared dependencies and cannot serve as the queue's invocation target.
Option C is a distractor because a function URL exposes an HTTPS endpoint and does not replace an SQS event source mapping target.
Option D is a distractor because environment variables change function configuration, not which version an event source invokes.
Option A is the correct answer. The exam clue is stable trigger target during version promotion, which points to a Lambda alias.
Study Task 3.1 artifact preparation and Lambda aliases.

3. A retail application stores product images in Amazon S3. The bucket sends object-created events to a Lambda image-processing function. The team wants to test new Lambda versions in development and production without reconfiguring the S3 notification each time a version changes. Which target should the S3 event notification use?

Option A is correct because the alias ARN stays stable while the alias can be repointed to different published versions.
Option B is a distractor because a version ARN is immutable, so the S3 notification would need to be updated after every promotion.
Option C is a distractor because a stack ARN identifies infrastructure, not the Lambda invocation target for S3.
Option D is a distractor because API Gateway stage ARNs are unrelated to S3-to-Lambda event notifications.
Option A is the correct answer. Event sources should invoke the alias ARN when the goal is to keep the trigger stable across version promotions.
Review Task 3.1 artifact preparation and Lambda aliases.

4. A company is upgrading a payment-processing Lambda function from one runtime family to another. The release team wants to validate the new environment separately and be able to switch traffic back immediately if an issue appears after cutover. Which deployment approach best meets these requirements?

Option A is correct because blue/green creates a parallel environment, supports validation before cutover, and enables fast rollback.
Option B is a distractor because all-at-once changes production immediately and carries the highest blast radius.
Option C is a distractor because in-place deployment updates the existing environment instead of validating a separate one first.
Option D is a distractor because rolling updates reduce risk on fleets, but the key exam clue here is a fully separate environment with quick cutback.
Option A is the correct answer. A major runtime change with rollback-sensitive cutover is the classic blue/green scenario.
Study Task 3.3 traffic shifting and CodeDeploy configurations.

5. A developer launches Amazon EC2 instances from an AWS CloudFormation template for a test environment. During provisioning, the instances must install OS packages, write configuration files, create directories, and start services based on metadata defined in the template. Which CloudFormation helper script should the developer use?

Option A is correct because cfn-init reads AWS::CloudFormation::Init metadata and applies package, file, service, and command configuration on the instance.
Option B is a distractor because cfn-hup watches for metadata changes after launch rather than performing the initial configuration steps.
Option C is a distractor because cfn-signal reports success or failure back to CloudFormation instead of applying metadata-driven configuration.
Option D is a distractor because sam init scaffolds a serverless project and is unrelated to EC2 instance bootstrap from template metadata.
Option A is the correct answer. The requirement is metadata-driven EC2 configuration during stack provisioning, which is exactly the job of cfn-init.
Review CloudFormation helper scripts and Task 3.1.

6. A company uses CloudFormation to create a development environment. The application server runs cfn-init at launch, and the stack must not continue creating dependent resources until that configuration finishes successfully. Which helper script should the instance call to notify CloudFormation that setup is complete?

Option A is correct because cfn-signal sends success or failure to CloudFormation so the stack can proceed or roll back.
Option B is a distractor because cfn-init performs configuration work, but it does not tell CloudFormation when the instance is ready.
Option C is a distractor because package prepares deployment artifacts and has nothing to do with EC2 bootstrap synchronization.
Option D is a distractor because sam build compiles serverless artifacts and does not coordinate CloudFormation wait behavior.
Option A is the correct answer. When the stack must wait for EC2 configuration to finish, the instance uses cfn-signal to report readiness.
Study Task 3.1 and CloudFormation helper scripts.
