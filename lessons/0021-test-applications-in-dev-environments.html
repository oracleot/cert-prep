Lesson 0021

Test deployments without touching production.

Task Statement 3.2: pick the safest dev/test pattern when the scenario mentions versions, mock APIs, stages, templates, or EC2 bootstrapping.

The win

You will classify Domain 3 test-environment questions by the clue: deployment risk, Lambda promotion, API dependency, SAM template, or CloudFormation helper script.

Source review: Domain 3 Task 3.2 in docs/aws dva-c02.md.

Fast selector

Least time, highest blast radius → all-at-once Major runtime/platform change → blue/green Stop app, install version, validate → in-place EC2 package/files/services from metadata → cfn-init Wait for EC2 config before stack continues → cfn-signal Deploy one SAM template, not project root → sam deploy --template Backend not ready, API consumers must test → API Gateway mock integration Same API, different dev/test config → API Gateway stages + variables Stable event source while Lambda changes → Lambda alias ARN

Exam shortcut

Versions are immutable. Aliases are movable pointers. Point S3, API Gateway, or other event sources at the alias ARN so promotion only changes the alias target.

Mixed drill: choose the best answer

1. A DVA-C02 scenario describes a development team that needs API responses before the Lambda backend is built. You must choose the API Gateway integration that returns test data. Which integration type returns responses without invoking any backend?

MOCK integration makes API Gateway return fixed, configured responses with no backend call—Task 3.2.
Lambda alias is a stable pointer to a version; it does not replace a missing backend.
cfn-init configures EC2 instances at launch; it is not an API Gateway integration type.
SAM package zips and uploads artifacts; it does not return API responses.
MOCK is correct. It returns test responses without invoking Lambda—used when the backend is not yet available.
Review Task 3.1 artifact preparation and API Gateway mock integrations.

2. A scenario involves a Lambda function whose versions are promoted through dev, beta, and prod environments. You must choose the feature that keeps event source ARNs unchanged during promotion. Which Lambda construct provides a stable trigger target that can be moved between versions?

A Lambda alias is a movable pointer to a specific version—event sources targeting the alias ARN do not need updates after promotion—Task 3.2.
Lambda layer versions share dependencies; they do not serve as stable trigger targets.
Wait conditions synchronize CloudFormation stack creation; they do not provide stable Lambda trigger targets.
Public mappings route API requests; they do not point to Lambda versions.
Lambda alias is correct. Aliases (dev, beta, prod) allow stable event source ARNs while versions are promoted.
Study Task 3.1 artifact preparation and Lambda aliases.

3. A DVA-C02 scenario describes an S3 event trigger that must keep invoking the same Lambda function even after new versions are published. You must choose the stable pointer. Which Lambda construct should the S3 trigger target instead of a version ARN?

An alias ARN is the stable trigger target—S3 and other event sources pointing to the alias do not need updates after version promotion—Task 3.2.
Lambda layer ARNs reference shared dependencies, not the function itself; they cannot serve as event source targets.
CloudFormation stack ARN identifies the stack, not a specific Lambda function version.
API Gateway stage ARN identifies a deployed API, not a Lambda version pointer.
Lambda alias ARN is correct. Event sources should target the alias, not the version ARN, so promotions do not require trigger updates.
Review Task 3.1 artifact preparation and Lambda aliases.

4. A DVA-C02 scenario involves a major runtime version change for a Lambda function where cutover risk must be minimized. You must choose the deployment strategy. Which deployment approach provisions a new environment in parallel before switching traffic?

Blue/green provisions a new environment in parallel and shifts traffic—minimizing cutover risk for major changes—Task 3.2.
All-at-once shifts all traffic immediately; it has the highest cutover risk.
In-place replaces code on the same instances; it does not provision a parallel environment.
API Gateway mock returns test responses; it is not a Lambda deployment strategy.
Blue/green is correct. It creates a parallel environment and shifts traffic, providing instant rollback capability.
Study Task 3.3 traffic shifting and CodeDeploy configurations.

5. A scenario describes an EC2 instance that must install packages, create directories, and start services automatically from CloudFormation template metadata during provisioning. You must name the bootstrap script. Which CloudFormation helper script applies metadata-driven configuration to an EC2 instance?

cfn-init reads AWS::CloudFormation::Init metadata and installs packages, creates files, and starts services—Task 3.2.
cfn-hup detects metadata changes and re-runs cfn-init; it does not perform the initial configuration.
sam init creates a new SAM project; it does not configure EC2 from CloudFormation metadata.
API Gateway mock returns test responses; it does not configure EC2 instances.
cfn-init is correct. It applies CloudFormation metadata to configure EC2 at launch.
Review CloudFormation helper scripts and Task 3.1.

6. A DVA-C02 scenario involves a CloudFormation stack that must pause until an EC2 instance finishes its cfn-init configuration. You must choose the synchronization mechanism. Which CloudFormation feature makes the stack wait for a successful EC2 configuration signal before proceeding?

cfn-signal sends a success or failure signal to CloudFormation so the stack can proceed or roll back—Task 3.2.
cfn-init configures the instance; it does not signal CloudFormation to continue.
sam build compiles Lambda artifacts; it does not synchronize CloudFormation.
API Gateway stage is a deployed API environment; it does not synchronize CloudFormation.
cfn-signal is correct. Combined with a CreationPolicy or wait condition, it tells CloudFormation the instance is ready.
Study Task 3.1 and CloudFormation helper scripts.
