Lesson 0017

Secure traffic before you secure code.

Task Statement 2.4: choose the AWS feature that protects traffic paths: public HTTPS, private AWS API calls, or subnet-level filtering.

The win

You will classify security clues into four exam moves: TLS with ACM, API Gateway custom domains, VPC endpoints, and security group vs NACL.

Use this after the Domain 2 source notes in docs/aws dva-c02.md.

1. Public HTTPS: ACM + the front door

Need HTTPS certificate → AWS Certificate Manager Need API vanity hostname → API Gateway custom domain Need DNS to the API → Route 53 alias / CNAME Need minimum TLS version → API Gateway security policy
  • ACM issues, imports, stores, and renews public/private TLS certificates for integrated AWS services.
  • API Gateway custom domains map names such as api.example.com to API stages; the custom domain uses an ACM certificate.
  • Regional API Gateway custom domains use an ACM certificate in the same Region; edge-optimized custom domains use certificates in us-east-1.
  • WAF is not TLS. Choose it for layer 7 filtering: SQL injection, XSS, IP sets, rate-based rules, and managed rule groups.

2. Private AWS traffic: endpoint first, NAT second

VPC endpoint clue

If private subnets must reach AWS services without internet gateways, NAT gateways, public IPs, or VPN paths, choose a VPC endpoint.

Gateway endpoint S3 and DynamoDB, route table target, no ENI Interface endpoint PrivateLink ENIs, security groups, most AWS APIs Endpoint policy Restrict which actions/resources use the endpoint Bucket policy Restrict S3 access with aws:sourceVpce

3. Network filters: stateful vs stateless

  • Security groups attach to ENIs, are stateful, and only allow traffic. Return traffic is automatically allowed.
  • NACLs attach to subnets, are stateless, can allow or deny, and evaluate rules by number.
  • Ephemeral port trap: NACLs need rules for both request and return traffic, including ephemeral ports. Security groups do not.
  • DVA clue: app-to-database access usually means security groups; subnet-wide explicit deny usually means NACL.

Retrieval drill: classify the control

1. An API needs api.example.com with HTTPS.

2. Private Lambda traffic must call S3 without NAT.

3. Private subnets must call Secrets Manager APIs.

4. A subnet needs an explicit deny for one CIDR.

5. Return traffic fails after a subnet rule change.

6. API requests must block SQL injection.
