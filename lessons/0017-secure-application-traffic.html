Lesson 0017

Secure traffic before you secure code.

Task Statement 2.4: choose the AWS feature that protects traffic paths: public HTTPS, private AWS API calls, or subnet-level filtering.

The win

You will classify security clues into four exam moves: TLS with ACM, API Gateway custom domains, VPC endpoints, and security group vs NACL.

Use this after the Domain 2 source notes in docs/aws dva-c02.md.

1. Public HTTPS: ACM + the front door

Need HTTPS certificate → AWS Certificate Manager Need API vanity hostname → API Gateway custom domain Need DNS to the API → Route 53 alias / CNAME Need minimum TLS version → API Gateway security policy
  • ACM issues, imports, stores, and renews public/private TLS certificates for integrated AWS services.
  • API Gateway custom domains map names such as api.example.com to API stages; the custom domain uses an ACM certificate.
  • Regional API Gateway custom domains use an ACM certificate in the same Region; edge-optimized custom domains use certificates in us-east-1.
  • WAF is not TLS. Choose it for layer 7 filtering: SQL injection, XSS, IP sets, rate-based rules, and managed rule groups.

2. Private AWS traffic: endpoint first, NAT second

VPC endpoint clue

If private subnets must reach AWS services without internet gateways, NAT gateways, public IPs, or VPN paths, choose a VPC endpoint.

Gateway endpoint S3 and DynamoDB, route table target, no ENI Interface endpoint PrivateLink ENIs, security groups, most AWS APIs Endpoint policy Restrict which actions/resources use the endpoint Bucket policy Restrict S3 access with aws:sourceVpce

3. Network filters: stateful vs stateless

  • Security groups attach to ENIs, are stateful, and only allow traffic. Return traffic is automatically allowed.
  • NACLs attach to subnets, are stateless, can allow or deny, and evaluate rules by number.
  • Ephemeral port trap: NACLs need rules for both request and return traffic, including ephemeral ports. Security groups do not.
  • DVA clue: app-to-database access usually means security groups; subnet-wide explicit deny usually means NACL.

Retrieval drill: classify the control

1. A company exposes a Regional Amazon API Gateway REST API for its mobile checkout app. The product team wants customers to use api.example.com over HTTPS, and the certificate should be managed by AWS. Which solution best meets these requirements?

Option A is the correct answer. ACM provides the managed TLS certificate, and the API Gateway custom domain maps api.example.com to the API stage.
Option B is incorrect. DNS routing alone does not provision the TLS certificate or create the custom domain mapping in API Gateway.
Option C is incorrect. Network ACLs filter subnet traffic, but API Gateway custom domains and certificates are not created with NACL rules.
Option D is incorrect. Shield Standard helps with baseline DDoS protection, but it does not provide a vanity hostname or managed HTTPS certificate for the API.
Option A is the correct answer. The exam clue is a public HTTPS hostname for API Gateway, which points to ACM plus an API Gateway custom domain.
Study Lesson 0016 and API Gateway custom domains.

2. A Lambda function in private subnets generates monthly invoices and stores PDF files in Amazon S3. The security team does not allow internet gateways, NAT gateways, or public IP addresses in this VPC. Which change should a developer make so the function can reach S3 privately?

Option A is the correct answer. Amazon S3 uses a gateway endpoint, which adds private route table entries so instances and Lambda ENIs can reach S3 without internet access.
Option B is incorrect. Interface endpoints are used for many AWS APIs, but the standard DVA answer for private S3 access is a gateway endpoint.
Option C is incorrect. A NAT gateway still depends on internet egress and does not satisfy the requirement to keep traffic private inside AWS networking.
Option D is incorrect. Route 53 Resolver helps with DNS resolution, not with establishing private network access to Amazon S3.
Option A is the correct answer. When the stem says private subnet access to S3 without NAT or internet, choose an S3 gateway VPC endpoint.
Review Lesson 0016 and gateway VPC endpoints.

3. A payments application runs on Amazon EC2 instances in private subnets and retrieves database credentials from AWS Secrets Manager. The instances must call the Secrets Manager API without traversing the public internet. Which solution best meets this requirement?

Option A is the correct answer. Secrets Manager is accessed privately through an interface VPC endpoint that creates PrivateLink ENIs in the subnets.
Option B is incorrect. Gateway endpoints are only for Amazon S3 and DynamoDB, not for Secrets Manager.
Option C is incorrect. Developers cannot solve AWS service access by peering to an internal AWS-managed service VPC.
Option D is incorrect. API Gateway is unrelated to private connectivity for the native Secrets Manager API.
Option A is the correct answer. For private access to most AWS APIs, including Secrets Manager, the exam pattern is an interface VPC endpoint.
Study Lesson 0016 and interface VPC endpoints.

4. A company hosts an internal application behind an Application Load Balancer in public subnets. The security team must block all inbound traffic from one partner CIDR range before the traffic reaches any targets, while allowing other source ranges. Which network control should the developer update?

Option A is the correct answer. Network ACLs are stateless and support explicit deny rules at the subnet level, which matches the requirement.
Option B is incorrect. Security groups only allow traffic and cannot create an explicit deny for a single CIDR range.
Option C is incorrect. ACM manages certificates and TLS, not IP-based network filtering.
Option D is incorrect. Lambda function URLs are unrelated to filtering ALB subnet traffic by source CIDR.
Option A is the correct answer. When the stem says explicit deny by CIDR at subnet scope, the best answer is a network ACL.
Review Lesson 0016 and network ACLs.

5. After a subnet network ACL change, an application on EC2 instances can still send outbound HTTPS requests, but responses from an external payment provider never return. The developer discovers that the new ACL removed the ephemeral port range. What is the most likely cause?

Option A is the correct answer. Network ACLs evaluate inbound and outbound traffic separately, so return traffic on ephemeral ports must be opened explicitly.
Option B is incorrect. Security groups are stateful and automatically allow return traffic for established connections.
Option C is incorrect. ENI quotas can cause other networking failures, but the clue here is the missing ephemeral port range in the NACL.
Option D is incorrect. KMS permissions affect encryption operations, not stateless subnet return traffic behavior.
Option A is the correct answer. The exam trap is ephemeral ports plus NACLs, which means stateless return traffic was blocked.
Study Lesson 0016 and NACL rule behavior.

6. A retailer exposes checkout APIs through Amazon API Gateway. Security testing shows that attackers are sending requests with common SQL injection and cross-site scripting payloads. The company wants a managed control that filters these patterns before the requests reach the backend. Which AWS service should the developer use?

Option A is the correct answer. AWS WAF performs layer 7 inspection and can block managed or custom SQL injection and XSS patterns on API Gateway.
Option B is incorrect. ACM secures transport with certificates, but it does not inspect request payloads for attack signatures.
Option C is incorrect. STS manages temporary credentials, not HTTP request filtering at the API edge.
Option D is incorrect. KMS key rotation affects encryption keys, not malicious web request filtering.
Option A is the correct answer. The phrases SQL injection and XSS are direct AWS WAF clues on DVA-C02.
Review Lesson 0018 and AWS WAF.
