Lesson 0015

Handle failures without leaking security context.

DVA-C02 Domain 2 scenarios test whether you can classify failures, preserve evidence, and avoid exposing secrets while fixing the error path.

The win

When a question says error handling, do not only think retry. Ask whether the app must recover, isolate, redact, or prove who was denied.

Related review: retries, idempotency, and DLQs.

The exam decision path

Transient AWS API failure? → SDK retry mode, exponential backoff, jitter Poison async event or repeated message failure? → Lambda destination / DLQ / SQS redrive policy Multi-step workflow needs alternate branch? → Step Functions Retry and Catch AccessDenied or KMS Decrypt denied? → check explicit deny, SCP, boundary, session, resource policy, key policy Security incident or audit trail needed? → CloudTrail for API calls, CloudWatch Logs for app logs, X-Ray for traces

1. Return safe errors; log useful facts

Client-facing errors should be useful but not revealing. Do not return stack traces, policy names, secret names, access keys, tokens, or raw exception dumps. Log request IDs, principal context, resource IDs, and sanitized failure codes for investigation.

Source: AWS Lambda logging best practices.

2. Pick the failure mechanism by shape

  • SDK call failed temporarily: retry with backoff and jitter.
  • Lambda async event failed: use destinations or a DLQ.
  • SQS message keeps failing: configure a redrive policy.
  • Workflow task fails: use Step Functions Retry and Catch.

Sources: SDK retries, Lambda async errors, and Step Functions error handling.

3. Troubleshoot denial in evaluation order

For IAM, an explicit deny wins. If there is no explicit deny, the request still needs an allow from the right policy path. Check identity policies, resource policies, permission boundaries, session policies, SCPs, and service-specific policies.

KMS has an extra trap: the caller usually needs permission through IAM and the KMS key policy or grant.

Source: IAM policy evaluation logic.

4. Use logs and traces without creating evidence leaks

Use CloudWatch Logs for application security events and CloudTrail for AWS API activity. Use X-Ray to trace requests across services, but keep secrets and sensitive data out of annotations, metadata, and log fields.

Sources: CloudWatch Logs, CloudTrail, and X-Ray concepts.

Exam traps

  • Retry is not authorization. AccessDenied is usually policy evaluation, not transient failure.
  • Logging is not dumping. Never log credentials, bearer tokens, API keys, or decrypted secrets.
  • CloudWatch is not CloudTrail. App logs go to CloudWatch; AWS API history goes to CloudTrail.
  • KMS deny may be key policy. Identity allow alone can still fail.

Practice: classify the safe response

1. A company exposes a payments API through API Gateway and Lambda. During testing, the Lambda function returns stack traces, IAM policy names, and internal exception messages in the HTTP response body. The security team says customers must not see this data, but developers still need enough detail to investigate failures. What is the best response?

Returning the full exception leaks internal details and increases the attack surface.
A generic 500 protects users, but skipping server-side evidence makes incident investigation harder.
Correct. The safe pattern is a sanitized client response plus structured server-side logging for operators.
Suppressing all errors does not help customers or developers understand what failed.
Option C is the correct answer. The application should hide sensitive internals from the client while logging request IDs and sanitized diagnostics for investigation.
Review Lesson 0009 and Lambda logging best practices.

2. A marketing application invokes a Lambda function asynchronously whenever users upload new campaign assets. Some events still fail after all retries, and the business requires the failed payloads to be captured for later analysis and reprocessing. Which solution should the developer choose?

Correct. Lambda Destinations and DLQs are built to capture asynchronous failures after retries are exhausted.
S3 lifecycle rules expire or transition objects and do not receive failed Lambda events.
CloudFront cache invalidation affects cached content, not asynchronous Lambda failure handling.
DynamoDB TTL removes old items and is unrelated to retaining failed async events.
Option A is the correct answer. Failed asynchronous Lambda events should be routed to a destination or DLQ so they can be inspected and replayed later.
Study Lesson 0009 and Lambda async invocation errors.

3. An insurance company uses AWS Step Functions to coordinate claim intake, document validation, and payment approval. If a validation Lambda returns a known business error, the workflow must move to a compensation branch instead of failing the entire execution immediately. Which Step Functions feature should the developer use?

Correct. Retry can handle transient errors, and Catch can route known failures to an alternate branch.
An EventBridge schedule triggers events by time and does not control Step Functions error branches.
SQS visibility timeout affects queue consumers and does not redirect Step Functions states.
CloudWatch alarms notify on metrics but do not change workflow execution flow.
Option A is the correct answer. Step Functions uses Retry and Catch to recover from or branch around task failures inside a workflow.
Study Lesson 0009 and Step Functions error handling.

4. A Lambda execution role has an IAM policy that allows access to a DynamoDB table, but the function still receives AccessDenied errors. The developer has already confirmed the table ARN is correct. What should the developer check next?

Memory size affects performance, not IAM policy evaluation.
Metric filters help monitoring, but they do not explain whether another policy layer denied the call.
Correct. An explicit deny in an SCP, permission boundary, or session policy overrides an allow.
Redeploying the function does not change the result of IAM evaluation logic.
Option C is the correct answer. When an allow exists but access is still denied, the next check is whether another policy type is explicitly denying the request.
Review Lesson 0014 and IAM policy evaluation logic.

5. An auditor asks which IAM principal called kms:Decrypt on a customer managed KMS key during a suspected incident. The team needs the AWS service that records the API caller, time, and source information. Which service should the developer use?

Correct. CloudTrail records AWS API activity, including the principal and request details for kms:Decrypt.
CloudWatch Metrics measure service behavior and do not provide API-call identity history.
X-Ray traces application requests, not AWS control-plane API audit records.
AWS Config tracks resource configuration state, not who invoked a specific KMS API call.
Option A is the correct answer. CloudTrail is the audit service for AWS API calls such as kms:Decrypt.
Study Lesson 0009 and CloudTrail user guide.
