Lesson 0011

Choose storage by access pattern, not by habit.

DVA-C02 Domain 1 tests whether you can separate object, file, block, relational, key-value, ephemeral, and lifecycle clues quickly.

The win

When a scenario asks for a data store, first classify the data shape: object, shared file, single-instance block, managed database, or temporary scratch.

Related: Domain triage and event service selection.

The exam map

S3 object storage: static assets, backups, data lakes, lifecycle tiers EFS shared Linux file system: many EC2/Lambda/ECS clients, NFS EBS block volume: one EC2 instance, boot/data disk, snapshots FSx managed file systems: Windows, Lustre, NetApp ONTAP, OpenZFS RDS/Aurora relational database: SQL, joins, transactions, managed engines DynamoDB key-value/document: low-latency, serverless, partition-key access /tmp ephemeral scratch: Lambda or EC2 instance-local temporary data

Step 1: Is it storage or a database?

  • S3/EFS/EBS/FSx store files, objects, or blocks. Choose them when the app already knows how to read/write those forms.
  • RDS/Aurora stores relational records when the scenario says SQL, schema, joins, transactions, or existing MySQL/PostgreSQL/Oracle/SQL Server apps.
  • DynamoDB stores key-value/document items when the scenario says serverless scale, single-digit millisecond latency, partition key, or massive throughput.

Step 2: Match the access method

  • Object: S3. You retrieve whole objects by key; great for static content, logs, artifacts, and backups.
  • Shared POSIX files: EFS. Multiple compute clients mount the same elastic Linux file system.
  • Block disk: EBS. One EC2 instance needs a low-latency disk volume or boot volume.
  • Specialized managed file system: FSx. Windows file shares, Lustre HPC, ONTAP, or OpenZFS clues point here.

Step 3: Persistent or ephemeral?

Persistent means the data must survive restarts, instance replacement, or function completion. Think S3, EFS, EBS snapshots/volumes, FSx, RDS/Aurora, and DynamoDB.

Ephemeral means scratch space. Lambda /tmp and EC2 instance store can disappear; use them for caches, decompression, image processing, ETL staging, and temporary downloads.

Lifecycle clues

  • S3 Lifecycle: transition objects to cheaper storage classes or expire/delete objects by age, prefix, tag, or version status.
  • EFS Lifecycle Management: move cold files to lower-cost EFS storage classes based on last access.
  • Amazon Data Lifecycle Manager: automate EBS snapshot and EBS-backed AMI creation, retention, copy, and deletion.
  • Backups are not lifecycle policies. If the clue is database backup/point-in-time recovery, think RDS/Aurora/DynamoDB backup features first.

Fast distractor eliminations

  • If many instances must mount the same Linux directory, eliminate EBS; choose EFS.
  • If the answer needs SQL joins or transactions, eliminate S3 and DynamoDB; choose RDS/Aurora.
  • If the clue is static website hosting or object expiration, eliminate EFS/EBS; choose S3.
  • If the clue is Windows SMB file shares, eliminate EFS; choose FSx for Windows File Server.
  • If the clue is temporary Lambda scratch, eliminate persistent stores unless results must survive.

Practice: choose the store

1. A media company stores raw video uploads as objects in AWS. To reduce cost, the company wants AWS to delete the raw files automatically 30 days after upload without running any cleanup code or cron jobs. Which storage service should the developers use for this requirement?

Option A is correct because S3 stores objects and supports lifecycle rules that can automatically expire them after a set number of days.
Option B is a distractor because EFS is a shared file system and is not the standard AWS object store with built-in object expiration.
Option C is a distractor because EBS provides block storage for EC2 instances, not managed object storage with lifecycle expiration.
Option D is a distractor because RDS stores relational data, not uploaded object files.
Option A is the correct answer. Raw uploads plus automatic expiration is a direct S3 lifecycle scenario.
Review Lesson 0011 on storage selection. Amazon S3 lifecycle management.

2. A machine-learning inference workflow uses several Lambda functions that must share the same Linux directory for model files and intermediate outputs across concurrent executions. The team wants a fully managed, elastic file system that Lambda can mount. Which AWS storage service is the best fit?

Option A is a distractor because S3 is object storage and cannot be mounted by Lambda as a shared POSIX file system.
Option B is correct because EFS provides shared NFS-based file storage that multiple Lambda executions can mount at the same time.
Option C is a distractor because EBS volumes are primarily attached to EC2 instances and do not provide Lambda-accessible shared file storage.
Option D is a distractor because FSx for Windows is for SMB/Windows workloads, while the stem points to shared Linux file access for Lambda.
Option B is the correct answer. Shared Linux file access for multiple Lambda functions is a standard EFS pattern.
Review Lesson 0011 on EFS. Amazon EFS.

3. A retailer is migrating an inventory application to AWS. The application needs SQL queries, multi-row transactions, foreign-key relationships, and JOIN operations across product and supplier tables, but the team wants AWS to manage backups and patching. Which data store should the developers choose?

Option A is a distractor because S3 stores objects and does not provide relational SQL queries or transactions.
Option B is a distractor because DynamoDB is a NoSQL service and does not support relational joins or foreign keys.
Option C is a distractor because EFS provides file storage, not a managed relational database engine.
Option D is correct because RDS and Aurora provide managed relational databases with SQL, ACID transactions, and foreign-key support.
Option D is the correct answer. SQL joins, foreign keys, and transactions are the classic RDS or Aurora exam clues.
Review Lesson 0011 on RDS. Amazon RDS.

4. An operations team must keep daily backups of a fleet of EC2 data volumes. The solution must automatically create EBS snapshots and delete older snapshots based on a retention policy. Which AWS feature should the team use?

Option A is a distractor because S3 Lifecycle manages S3 objects, not EBS snapshot creation and retention.
Option B is a distractor because EFS Lifecycle Management moves cold files to lower-cost EFS storage classes and does not manage EBS backups.
Option C is correct because Amazon Data Lifecycle Manager automates EBS snapshot schedules, retention, and cleanup.
Option D is a distractor because DynamoDB TTL expires table items, not EBS snapshots.
Option C is the correct answer. When the requirement is scheduled EBS snapshots with retention, Data Lifecycle Manager is the best-fit AWS service.
Review Lesson 0011 on EBS lifecycle. Amazon Data Lifecycle Manager.

5. A Lambda function downloads a PDF from Amazon S3, converts it to images, and uploads the results back to S3. The intermediate files are needed only during the invocation and should not persist afterward. Where should the function store the temporary files?

Option A is correct because Lambda provides ephemeral /tmp storage for temporary files that do not need to survive after execution.
Option B is a distractor because RDS is a database service and is not intended as scratch space for transient file processing.
Option C is a distractor because Lambda cannot attach an EC2 EBS volume for temporary local processing.
Option D is a distractor because FSx for Lustre is a persistent managed file system and is unnecessary for temporary, invocation-scoped scratch files.
Option A is the correct answer. Temporary files used only during a Lambda invocation belong in the function's ephemeral /tmp storage.
Review Lesson 0011 on ephemeral storage. Lambda ephemeral storage.
