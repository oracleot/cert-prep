Lesson 0002

Lambda: know how the event enters, then how failure is handled.

DVA-C02 Lambda questions usually hide the answer in the invocation path: synchronous, asynchronous, or event source mapping.

The exam move

Before picking a Lambda feature, ask: who invokes the function? The caller determines retry behavior, error handling, and whether Lambda reads from a source in batches.

Keep the Lambda quick sheet open while you practice.

Three invocation patterns

1. Synchronous

The caller waits. If Lambda succeeds, the caller gets the response. If it fails, the caller sees the error and must decide whether to retry. The AWS CLI command is aws lambda invoke.

2. Asynchronous

The caller hands off the event and does not wait. Lambda queues the event, runs the function, retries function errors twice, and can send invocation records to destinations or a dead-letter queue.

3. Event source mapping

Lambda polls SQS, Kinesis, or DynamoDB Streams and sends records to your function in batches. With Kinesis and DynamoDB Streams, shards are the concurrency unit because each shard is processed in order.

Failure routing mental model

Caller waits? → synchronous → caller handles retry Caller drops event? → asynchronous → Lambda retries, destinations or DLQ Lambda polls source? → event source mapping → source rules, batch behavior, redrive policy

Common DVA-C02 traps

  • Retries require idempotency. Your function may process the same event more than once.
  • Event source mapping is a Lambda resource. Use it for SQS, Kinesis, and DynamoDB Streams.
  • Memory affects CPU. Increasing memory can improve CPU-bound performance.
  • Reserved concurrency is a control. It can cap a function so it does not consume too much account concurrency.
  • Provisioned concurrency targets cold starts. It keeps environments initialized for low latency.

Practice: classify the Lambda path

1. A health-tech company uses Amazon API Gateway to front a Lambda function that validates insurance claims and must return a decision to the mobile client in the same request. Which invocation model is API Gateway using to call the function?

Option A is correct because API Gateway waits for the Lambda response before it can return data to the mobile client.
Option B is incorrect because asynchronous invocation means the caller hands off the event and does not wait for the function result.
Option C is incorrect because event source mappings are used when Lambda polls services such as SQS, Kinesis, or DynamoDB Streams.
Option D is incorrect because a destination is a failure-routing feature for asynchronous invocations, not an invocation model.
Option A is the correct answer. API Gateway uses synchronous invocation for request-response patterns where the client must receive the Lambda result immediately.
Review Lesson 0001 for domain triage. Lambda synchronous invocation.

2. A media company stores uploaded photos in Amazon S3. Each new object should trigger a Lambda function that generates thumbnails, and the user must not wait for processing to finish. How does Lambda receive the S3 event?

Option A is incorrect because S3 does not wait for the function response before completing the upload workflow.
Option B is correct because S3 event notifications invoke Lambda asynchronously and Lambda handles retry behavior.
Option C is incorrect because event source mappings apply when Lambda polls queues or streams, not S3 event notifications.
Option D is incorrect because provisioned concurrency reduces cold starts but does not describe how S3 delivers events.
Option B is the correct answer. S3 sends event notifications to Lambda asynchronously, so the uploader does not wait for the thumbnail job to finish.
Lesson 0001 covers Lambda invocation patterns. See Lambda asynchronous invocation.

3. An ecommerce platform places order-fulfillment jobs on an Amazon SQS queue. A Lambda function must automatically poll the queue and process up to 10 messages at a time. Which mechanism should the developer configure?

Option A is incorrect because there is no direct caller waiting for a response when Lambda processes SQS messages.
Option B is incorrect because SQS integration is poll-based rather than a service directly invoking Lambda asynchronously.
Option C is correct because Lambda uses an event source mapping to poll SQS and deliver batches to the function.
Option D is incorrect because a layer packages shared dependencies and does not connect Lambda to a queue.
Option C is the correct answer. SQS integrations use an event source mapping so Lambda can poll the queue and pass batches to the handler.
See Lesson 0004. Lambda event source mapping.

4. A ticketing API runs on API Gateway and Lambda. During concert drops, new execution environments create latency spikes, and the business requires consistently low response times for 200 concurrent requests. Which Lambda feature best meets this requirement?

Option A is incorrect because reserved concurrency limits or guarantees capacity, but it does not keep execution environments pre-initialized.
Option B is correct because provisioned concurrency keeps execution environments warm for a target concurrency level.
Option C is incorrect because event source mappings are for queue and stream polling, not API latency control.
Option D is incorrect because a dead-letter queue stores failed asynchronous events and has no effect on cold starts.
Option B is the correct answer. Provisioned concurrency pre-warms Lambda environments so the API avoids cold-start latency for the configured traffic level.
Review Lesson 0003. Lambda concurrency documentation.

5. A batch image-processing function can suddenly scale to thousands of concurrent invocations and starve more critical authentication and checkout functions in the same account. Which setting should the developer configure to cap this function's scale?

Option A is correct because reserved concurrency sets the maximum concurrent executions for a function and protects the rest of the account.
Option B is incorrect because provisioned concurrency pre-warms capacity but does not cap the function's maximum concurrency.
Option C is incorrect because destinations route asynchronous invocation records and do not control concurrency.
Option D is incorrect because extensions add runtime side processes and do not limit function scaling.
Option A is the correct answer. Reserved concurrency prevents one noisy function from consuming the entire account concurrency pool.
Lesson 0003. Reserved concurrency.

6. An EventBridge rule invokes a Lambda function asynchronously to process partner order files. Some malformed events keep failing, and the team wants the original event payload sent to an Amazon SQS queue after Lambda exhausts its retries so it can be replayed later. Which feature should the developer configure?

Option A is correct because a dead-letter queue stores the original failed asynchronous event in SQS or SNS after Lambda exhausts retries.
Option B is incorrect because a destination sends an invocation record, not the original event payload the team wants to replay directly.
Option C is incorrect because reserved concurrency controls scale and does not route failed events anywhere.
Option D is incorrect because a layer packages shared code and is unrelated to asynchronous failure handling.
Option A is the correct answer. The requirement is to capture the original asynchronous event in SQS after retries are exhausted, which is the dead-letter queue use case.
