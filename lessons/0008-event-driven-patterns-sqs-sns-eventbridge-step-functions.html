Lesson 0008

Pick the event service by the coupling problem.

DVA-C02 Domain 1 expects fast recognition of SQS, SNS, EventBridge, and Step Functions in loose-coupling scenarios.

The win

When a scenario says "decouple," do not stop at "serverless." Classify the communication shape: buffer, broadcast, route, or orchestrate.

Related review: Lambda event source mappings.

The four shapes

SQS producer → queue → consumer SNS fanout publisher → topic → many subscribers EventBridge event bus → rules → selected targets Step Funcs state machine → ordered workflow steps

SQS: buffer and absorb spikes

Choose SQS when producers and consumers should not depend on each other's speed or availability. Keywords: queue, backlog, polling, message retention, visibility timeout, dead-letter queue.

SNS: push one message to many

Choose SNS for pub/sub fanout when one publication must notify multiple subscribers such as SQS queues, Lambda functions, HTTPS endpoints, email, or SMS.

EventBridge: route application events

Choose EventBridge when events need content-based rules, SaaS/AWS/custom event buses, schema discovery, or many services reacting without a central workflow owner.

Step Functions: own the workflow state

Choose Step Functions when the scenario needs branching, retries, wait states, error handling, human-readable state history, or long-running workflow orchestration.

Orchestration vs choreography

  • Orchestration: one coordinator decides the next step. Exam clue: state machine, workflow, branch, retry policy.
  • Choreography: services react to events independently. Exam clue: event bus, loose coupling, rules, subscribers.
  • Fanout: duplicate one event to multiple consumers. Classic answer: SNS topic with SQS subscriptions.
  • Back pressure: smooth traffic spikes between producer and worker. Classic answer: SQS queue.

Exam traps

  • SNS is not a durable work queue. If workers need to pull and retry, put SQS behind SNS.
  • EventBridge is not Step Functions. EventBridge routes events; Step Functions tracks workflow state.
  • Lambda is not the default orchestrator. Complex coordination belongs in Step Functions.
  • Loose coupling still needs idempotency. Retries and duplicate events can happen.

Practice: choose the pattern

1. A ticketing company publishes purchase events from its checkout API, but flash-sale traffic arrives faster than the fulfillment workers can process it. The API must keep accepting orders even when workers are slow or temporarily unavailable. Which AWS service is the best fit to absorb the burst and decouple producers from consumers?

Option A is correct because SQS provides a durable queue that buffers messages so producers and consumers do not need to operate at the same speed.
Option B is a distractor because SNS fans out notifications but does not act as a durable pull-based work queue for slower workers.
Option C is a distractor because EventBridge routes events to targets but is not the primary service for backlog buffering and worker decoupling.
Option D is a distractor because Step Functions orchestrates workflow steps; it does not absorb queue backlogs between independent producers and consumers.
Option A is the correct answer. SQS is the classic DVA-C02 answer when the requirement is to buffer spikes and let consumers process messages later.

2. A retail application emits one OrderPlaced event after checkout. The same event must immediately notify the billing service, a fraud-detection Lambda function, and an analytics pipeline without the publisher managing each destination separately. Which AWS service should the developer choose?

Option A is a distractor because an SQS queue delivers each message to consumers for work processing, not simple one-to-many push fanout.
Option B is correct because SNS topics publish one message to many subscribers such as Lambda, SQS, HTTPS endpoints, email, or SMS.
Option C is a distractor because an EventBridge archive stores past events for replay; it does not perform the primary live fanout role in this scenario.
Option D is a distractor because Step Functions coordinates workflow state, not simple broadcast delivery to independent subscribers.
Option B is the correct answer. The exam clue is one event that must notify multiple independent subscribers, which points to SNS fanout.
Review Lesson 0008 for the broadcast pattern. Amazon SNS documentation.

3. An ISV publishes custom events from its SaaS platform. Events with source=orders must go to one Lambda function, events with detail-type=invoice.created must go to an SQS queue, and other events should be ignored. Which AWS service provides the best content-based routing for this requirement?

Option A is a distractor because SQS stores messages for consumers but does not inspect event fields and route them to different targets.
Option B is a distractor because SNS broadcasts to subscribers rather than selectively matching JSON event content.
Option C is correct because EventBridge rules evaluate event patterns such as source and detail-type, then route matching events to selected targets.
Option D is a distractor because Step Functions executes a workflow definition; it is not the main event-routing service for loosely coupled subscribers.
Option C is the correct answer. EventBridge is the AWS service built for content-based routing on an event bus.
Review Lesson 0008 on EventBridge. Amazon EventBridge rules.

4. An online pharmacy needs to coordinate prescription fulfillment across several services. The workflow must branch based on insurance results, wait for a pharmacist approval step, retry selected tasks, and provide an execution history for audits. Which AWS service should the developers use?

Option A is a distractor because SQS provides buffering, not branching logic, wait states, or workflow history.
Option B is a distractor because SNS broadcasts messages but does not maintain workflow state or retry rules per step.
Option C is a distractor because EventBridge routes events to targets but does not act as a stateful workflow engine.
Option D is correct because Step Functions state machines support branching, wait states, retries, and a visual execution history.
Option D is the correct answer. When the scenario needs a stateful, auditable workflow with branching and retries, Step Functions is the exam answer.
Review Lesson 0008 on Step Functions. AWS Step Functions documentation.

5. A marketplace publishes OrderPlaced, PaymentCaptured, and ShipmentCreated events to an Amazon EventBridge bus. Inventory, loyalty, and analytics services subscribe and react independently, and no central service decides the next step in the business flow. Which design best describes this architecture?

Option A is correct because independent services reacting to shared events on a bus is the definition of choreography.
Option B is a distractor because orchestration means one coordinator, such as Step Functions, determines the next step explicitly.
Option C is a distractor because request/response patterns are synchronous and tightly coupled compared with event-driven subscribers.
Option D is a distractor because nothing in the stem points to offline batch processing or object-created notifications.
Option A is the correct answer. The absence of a central coordinator and the use of an event bus signal a choreography pattern.
