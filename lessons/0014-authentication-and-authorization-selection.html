Lesson 0014

Pick the auth control from the exam clue.

Task Statement 2.1 is service selection: IAM, STS, Cognito, API Gateway authorizers, Lambda policies, or least privilege.

Related review: domain triage and the curriculum map.

The selection map

  • AWS code needs APIs → IAM role | Another account needs access → STS AssumeRole
  • App users need sign-up/sign-in → Cognito user pool | App users need AWS credentials → Cognito identity pool
  • API callers sign SigV4 → IAM authorization | Bearer-token custom logic → TOKEN Lambda authorizer
  • API needs headers/query/context → REQUEST Lambda authorizer | Service invokes Lambda → Lambda resource policy

Key concepts at a glance

1. IAM policy vs role: policy grants/denies actions; role is an assumable identity with a trust policy. Choose roles for Lambda, ECS tasks, EC2, cross-account, and temporary credentials.

2. STS AssumeRole: temporary credentials for cross-account, federation, or delegated access. Avoid long-term keys unless there is no federation path.

3. Cognito: user pools authenticate app users and issue tokens; identity pools exchange identities for temporary AWS credentials mapped to IAM roles.

4. API Gateway authorizers: Cognito/JWT authorizer for managed token validation; Lambda TOKEN authorizer for bearer-token custom logic; Lambda REQUEST authorizer for headers, query strings, stage variables, or context.

5. Lambda policies: execution role controls what Lambda can call; resource-based policy controls who can invoke Lambda. Restrict principals, actions, resources, and source ARNs.

Fast distractor cuts

  • SCPs, permission boundaries, and session policies limit permissions—they do not grant access.
  • API keys are for usage plans and throttling, not user authentication.
  • ECS task access uses task roles, not IAM groups or service-linked roles.
  • Cross-account access needs both trust and permissions.
  • Wildcard * is wrong when the question says most secure or least privilege.

Retrieval drill: choose the best answer

1. A company runs containers on Amazon ECS. Each task must call DynamoDB and Amazon S3 without storing long-term credentials in the image or on the host. Which IAM identity should the developer attach to the task?

Correct. An ECS task role supplies temporary credentials to the running task.
IAM groups organize users and cannot be assumed by ECS tasks.
Root credentials violate least privilege and should never be used by applications.
IAM user keys are long-term credentials and are not the best practice for ECS tasks.
Option A is the correct answer. ECS tasks should assume an IAM role so the application receives temporary AWS credentials automatically.
Review Lesson 0001. ECS task roles.

2. A company shares an S3 bucket with a partner that runs workloads in a separate AWS account. The partner needs time-limited access to specific objects, and the security team will not allow long-term access keys. Which solution should the developer implement?

Correct. STS AssumeRole returns temporary cross-account credentials with a defined session duration.
Static IAM keys are long-lived and do not satisfy the temporary-credentials requirement.
Root access is overly broad and should never be used for delegated application access.
A shared IAM user is hard to audit and violates least-privilege and credential-isolation practices.
Option A is the correct answer. Cross-account access with temporary credentials is the classic STS AssumeRole scenario.
Review Lesson 0014. STS AssumeRole.

3. A consumer mobile app needs built-in user sign-up, sign-in, and token issuance before the app can call backend APIs. Which Amazon Cognito feature should the developer choose?

Correct. Cognito user pools handle user registration, authentication, and token issuance.
Identity pools exchange identities for AWS credentials, but they do not provide primary sign-up and sign-in flows.
IAM roles grant AWS permissions to workloads, not app-user authentication.
API keys are for usage plans and throttling, not user sign-in.
Option A is the correct answer. App-user authentication with sign-up and sign-in is the main purpose of a Cognito user pool.
Compare Lesson 0016 and Cognito user pools.

4. After users sign in, a mobile app must let them upload photos directly to Amazon S3 without embedding AWS keys in the app. The users need temporary AWS credentials mapped to least-privilege IAM roles. Which Cognito feature should the developer use?

Correct. Identity pools exchange authenticated identities for temporary AWS credentials tied to IAM roles.
User pools authenticate users, but they do not directly hand out AWS credentials for S3 access.
API keys do not grant temporary AWS credentials to mobile users.
Stage caching stores API responses and is unrelated to identity federation.
Option A is the correct answer. Temporary AWS credentials for signed-in app users is the defining Cognito identity pool use case.
Study Lesson 0016 and Amazon Cognito.

5. Several internal AWS workloads call an API Gateway REST API by signing requests with Signature Version 4. The company wants API Gateway to authorize only those IAM principals. Which authorization method should the developer configure?

Correct. API Gateway IAM authorization validates SigV4-signed requests from AWS principals.
API keys help with quotas and throttling, not SigV4 identity-based authorization.
A user pool authorizer validates JWTs, not SigV4 request signatures.
Stage variables pass configuration values and do not authorize requests.
Option A is the correct answer. SigV4-signed API calls are the clue that API Gateway IAM authorization is required.
Review Lesson 0017 and API Gateway IAM authorization.

6. A company exposes an API through API Gateway and uses a custom bearer token issued by an external identity provider. The token validation logic must run in Lambda and use only the incoming token value. Which authorizer type should the developer choose?

Correct. A TOKEN authorizer passes the bearer token to Lambda for custom validation.
A REQUEST authorizer is for cases that need headers, query strings, or request context, not just a token.
Usage plans apply throttling and quotas and do not perform authentication.
Stage caching stores responses and does not validate tokens.
Option A is the correct answer. Custom bearer-token validation in Lambda maps directly to a TOKEN authorizer.
Review Lesson 0015 and Lambda authorizers.

7. An API Gateway authorizer must examine multiple request attributes, including headers, query string parameters, and stage variables, before allowing a call to continue. Which authorizer type is the best fit?

Correct. A REQUEST authorizer receives full request context such as headers and query strings.
A TOKEN authorizer is limited to the token value and does not receive broader request context.
Usage plans do not inspect request attributes for authorization decisions.
Stage caching is unrelated to request authorization logic.
Option A is the correct answer. When the decision depends on headers, query strings, and context, choose a REQUEST authorizer.
Study Lesson 0015 and Lambda authorizers.

8. An API Gateway REST API returns a permissions error when it tries to invoke a Lambda function. The Lambda execution role already allows the function to call downstream services. What should the developer add to let API Gateway invoke the function?

Correct. A Lambda resource-based policy grants another AWS service permission to invoke the function.
The execution role controls what Lambda can call, not who can invoke Lambda.
A user pool authenticates users and does not authorize API Gateway to invoke Lambda.
Access keys are not used to grant service-to-service invoke permission.
Option A is the correct answer. API Gateway needs permission in the Lambda resource policy because the question is about who may invoke the function.
Review Lesson 0005 and Lambda resource policies.

9. A Lambda function must write order records to a DynamoDB table. Which IAM identity should contain the permissions for dynamodb:PutItem on that table?

Correct. The Lambda execution role defines what AWS APIs the function code may call.
A resource policy controls who can invoke Lambda, not what Lambda can do in DynamoDB.
Usage plans throttle API clients and do not grant AWS service permissions.
An access token is used for user authentication, not for granting DynamoDB access to Lambda code.
Option A is the correct answer. When Lambda code needs DynamoDB access, the permission belongs on the Lambda execution role.
Study Lesson 0005 and Lambda execution role.

10. A developer must grant an application permission to read objects from only one S3 bucket. The security review says the policy must follow least privilege. Which resource scope is the best answer?

Correct. A policy scoped to the specific bucket or object ARN grants only the needed access.
A wildcard resource is broader than necessary and violates least privilege.
Root-level permissions are far too broad for a single-bucket read requirement.
Shared keys are a credential practice, not a least-privilege resource scope.
Option A is the correct answer. Least privilege means scoping access to the exact resource ARN the application needs.
Review Lesson 0001 and IAM least privilege.
