Lesson 0012

Choose the data pattern from the exam clue.

Domain 1 loves service-selection questions: cache the read path, stream events, transform data, or build the app flow.

The win

You will sort four common Domain 1 traps by keyword: lazy cache, write-through cache, streaming ingest, and ETL tool.

Related review: event-driven patterns and Domain 1 map.

1. Caching: pick the write/read behavior

Lazy loading app misses cache → reads database → fills cache Write-through app writes cache and database in the write path TTL cache entry expires after a chosen time window
  • Lazy loading: choose it when only requested data should be cached. First read is slower; later reads are faster.
  • Write-through: choose it when the cache must be warm immediately after writes. Write latency can increase.
  • TTL: choose it when stale data must expire automatically. TTL is a freshness control, not a database backup.
  • DAX / ElastiCache clue: DAX fits DynamoDB microsecond read caching; ElastiCache fits Redis/Memcached app caches.

2. Streaming: pick the ownership model

Kinesis Data Streams

Choose it for custom consumers, ordered records per shard, replay within retention, Lambda event source mappings, or multiple applications reading the same stream.

Amazon Data Firehose

Choose it when the clue says near-real-time delivery to S3, Redshift, OpenSearch, Splunk, or HTTP endpoints with minimal custom consumer code.

Amazon MSK

Choose it when the scenario already uses Apache Kafka, needs Kafka APIs, Kafka clients, topics, partitions, or wants AWS to manage Kafka brokers.

3. ETL: pick the transformation engine

Glue managed serverless ETL + Data Catalog + crawlers EMR managed Hadoop/Spark clusters for big data frameworks Lambda small event transform, short running, serverless glue code Kinesis streaming transform/analytics before storage or delivery
  • AWS Glue: default exam answer for managed ETL, schema discovery, crawlers, and the Glue Data Catalog.
  • Amazon EMR: choose for Spark/Hadoop ecosystem control, cluster tuning, or long/heavy big-data processing.
  • AWS Lambda: choose for lightweight transforms triggered by S3, streams, EventBridge, API Gateway, or queues.
  • Kinesis: choose when transformation must happen continuously on streaming records.

Fast distractor cuts

  • If the question says replay records, Firehose is usually the distractor; think Kinesis Data Streams.
  • If it says deliver to S3 with no servers, custom Kinesis consumers are the distractor; think Data Firehose.
  • If it says Kafka clients, do not translate to Kinesis; think Amazon MSK.
  • If it says schema catalog/crawler, Lambda is too small; think AWS Glue.

Retrieval drill: select the service or pattern

1. A retail company runs a product catalog on Amazon Aurora with Amazon ElastiCache in front of the database. Traffic is spiky, and the team wants only requested products to be cached so memory is not spent on rarely viewed items. Which caching strategy should the developer choose?

Correct. Lazy loading fills the cache only after a miss, so only requested items are stored.
Write-through keeps the cache warm on writes, but it does not limit caching to items users actually request.
Scheduled warming preloads data regardless of demand, which works against the memory-saving requirement.
TTL controls freshness, but by itself it does not define how entries first get into the cache.
Option A is the correct answer. Lazy loading matches on-demand caching because the application reads the database only after a cache miss and then stores the result.

2. A financial application writes stock prices to Amazon RDS and uses ElastiCache to reduce read traffic. Each new price must be available from the cache immediately after the write, even if no user has requested it yet. Increased write latency is acceptable. Which caching strategy meets this requirement?

Lazy loading populates the cache only after a read miss, so new writes are not visible in cache immediately.
Correct. Write-through updates the database and cache in the same write path, keeping the cache warm.
Periodic refresh can leave a gap between the write and the next refresh cycle.
TTL controls expiration, not immediate cache population after writes.
Option B is the correct answer. Write-through is designed for cases where the newest value must already be present in cache right after a write.
Review Lesson 0012. Amazon ElastiCache strategies.

3. A company collects order events from many microservices. Two internal applications must read the same ordered stream at different speeds, and developers need to replay the last 24 hours of records during incident analysis. Which AWS service is the best fit?

Correct. Kinesis Data Streams supports ordered records per shard, multiple consumers, and replay during the retention window.
Firehose delivers data to destinations with minimal code, but it is not the best choice when consumers need replay.
MSK is plausible only when the scenario says the company already needs native Kafka clients or APIs.
Glue Data Catalog stores metadata about datasets, not streaming records for consumers.
Option A is the correct answer. Replay plus multiple applications consuming the same stream is the classic Kinesis Data Streams pattern.
Lesson 0008. Kinesis Data Streams.

4. A media company captures clickstream events from web applications and must deliver the records to Amazon S3 in near real time. The team wants minimal operational overhead and does not want to build or run custom consumers. Which service should the developer choose?

Kinesis Data Streams would require the team to manage consumer logic that writes records to S3.
Correct. Data Firehose is built for near-real-time delivery to S3 and similar destinations with little custom code.
MSK manages Kafka brokers, but the stem does not say the company needs Kafka APIs or clients.
Glue Data Catalog catalogs schemas; it does not act as a streaming delivery service.
Option B is the correct answer. The requirement is managed delivery to S3 with minimal code, which is the core Data Firehose use case.
Review Lesson 0012. Amazon Data Firehose.

5. A company is migrating an existing analytics platform that already uses Apache Kafka clients, topics, and partitions. The developers want AWS to manage the brokers but do not want to rewrite producers and consumers for a non-Kafka API. Which AWS service should they use?

Kinesis Data Streams is AWS-native streaming, so it would require the team to move away from Kafka clients.
Data Firehose is a managed delivery service, not a managed Kafka broker platform.
Correct. Amazon MSK runs managed Apache Kafka so existing Kafka tools and clients still work.
Glue Data Catalog manages metadata for datasets, not Kafka brokers.
Option C is the correct answer. Native Kafka compatibility with AWS-managed brokers points directly to Amazon MSK.
Review Lesson 0012. Amazon MSK.

6. A data team stores CSV and JSON files in Amazon S3 and needs an AWS service to crawl the data, infer schemas, and maintain a central catalog for ETL jobs and analytics. Which service is the best answer?

Correct. AWS Glue crawlers discover schemas in S3 and store them in the Glue Data Catalog.
Lambda can run small transforms, but it is not the managed schema crawler and catalog service in the stem.
EMR can process big data, but the clue is automatic schema discovery and a managed catalog.
DAX accelerates DynamoDB reads and has nothing to do with S3 schema discovery.
Option A is the correct answer. The schema-crawler plus shared catalog requirement is the signature AWS Glue pattern.
Review Lesson 0012. AWS Glue.

7. An S3 ObjectCreated event should trigger a short image-metadata transform and write the result to DynamoDB. The company wants per-request billing and no cluster management. Which compute service should the developer choose?

Glue jobs are better for managed ETL pipelines than short event-driven transforms from S3.
Correct. Lambda is serverless, event-driven, and ideal for lightweight transforms triggered by S3.
EMR adds cluster management and is excessive for a small per-object transform.
Kinesis Data Analytics is for stream processing, not direct S3 object event transforms.
Option B is the correct answer. A short S3-triggered transform with no servers to manage is a standard Lambda scenario.
Review Lesson 0002. AWS Lambda use cases.

8. A startup is building a public orders API. HTTPS requests must invoke application code that creates, reads, updates, and deletes items in DynamoDB, and the company wants a fully serverless design. Which service combination should the developer use?

Correct. API Gateway handles the HTTPS API and Lambda runs the CRUD logic against DynamoDB.
SNS and SQS are messaging services, not the primary pattern for a public request-response API.
CodeDeploy and SAM help package and release applications, not serve runtime API traffic.
CloudWatch and X-Ray monitor applications, but they do not implement the API itself.
Option A is the correct answer. Public HTTPS plus serverless business logic over DynamoDB points to API Gateway and Lambda.
