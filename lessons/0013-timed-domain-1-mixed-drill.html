Timed drill

1. A startup is building a public HTTPS endpoint for a checkout service. The endpoint must invoke application code, return JSON responses, and scale without provisioning servers. Which architecture should the developer choose?

Correct. API Gateway accepts HTTPS requests and Lambda runs the serverless business logic.
S3 and Firehose are for object storage and streaming delivery, not request-response APIs.
DAX accelerates DynamoDB reads, but it does not expose a public HTTPS API.
SNS and EventBridge route events, but they are not the primary synchronous API pattern.
Option A is the correct answer. Public HTTPS requests with serverless compute and JSON responses are the classic API Gateway plus Lambda pattern.
Lesson 0007. API Gateway HTTP API.

2. A photo-sharing application stores uploads in Amazon S3. Every new image must automatically invoke code that creates a thumbnail, and the company does not want to manage any servers. Which event-driven design best meets the requirement?

DAX improves DynamoDB read performance and is unrelated to S3 image processing.
Correct. S3 ObjectCreated events can invoke Lambda to run image-processing code.
Glue crawlers discover data schemas; they are not used for per-image thumbnail generation.
MSK is a managed Kafka service and is unnecessary for a direct S3-triggered workflow.
Option B is the correct answer. The stem describes the native S3 event notification to Lambda pattern.
Lesson 0002. Lambda with S3.

3. An order API sends work to a fulfillment service that sometimes becomes unavailable. The company needs a durable buffer between the two components so messages are retained and retried until the consumer succeeds. Which AWS service should the developer use?

DAX is an in-memory cache for DynamoDB and does not buffer work between services.
EFS provides shared file storage, not a durable message queue with retries.
Correct. SQS is designed for durable decoupling and retry-driven message processing.
X-Ray traces request paths, but it does not hold messages for later processing.
Option C is the correct answer. Durable buffering with retries between components is the core SQS use case.
Lesson 0008. SQS.

4. A commerce platform publishes one event when a customer places an order. That single event must trigger email notifications, invoke a Lambda fraud check, and send copies to multiple queues. Which AWS service is the best fit?

SQS delivers a message to consumers, but it does not provide broadcast fanout to many targets by itself.
DAX is a DynamoDB cache and has nothing to do with pub/sub fanout.
EBS is block storage for EC2 instances, not an event-broadcast service.
Correct. SNS publishes one message to many subscribers such as email, Lambda, and SQS.
Option D is the correct answer. One message sent to many subscribers is the defining SNS pattern.
Lesson 0008. SNS.

5. A SaaS platform emits events from several subsystems. Different Lambda functions must receive events based on values in the source and detail-type fields, and the routing rules must be changeable without modifying application code. Which AWS service should the developer choose?

DynamoDB Streams publish table item changes, not arbitrary event-bus routing by content fields.
Correct. EventBridge rules match on event content such as source and detail-type.
Lambda layers share code and dependencies; they do not route events.
S3 lifecycle manages object transitions and expiration, not event routing.
Option B is the correct answer. Content-based routing without code changes is exactly what EventBridge rules provide.
Lesson 0008. EventBridge rules.

6. A claims-processing workflow needs conditional branching, wait states for approval deadlines, and per-step retry logic when a downstream Lambda call fails temporarily. The company also wants an execution history for audits. Which AWS service should the developer use?

API Gateway handles API requests, but it does not orchestrate long-running workflows with state.
Data Firehose delivers streaming data to destinations and is unrelated to workflow control flow.
Correct. Step Functions supports branches, waits, retries, and visible execution history.
Secrets Manager stores secrets and does not orchestrate workflows.
Option C is the correct answer. The stem asks for managed workflow orchestration with retries, branches, and history, which points to Step Functions.
Lesson 0008. Step Functions.

7. A gaming application uses DynamoDB for player profiles. Read traffic is extremely high, and the team needs microsecond response times for frequently accessed items without changing the application to another database. Which AWS service should the developer add?

A dead-letter queue stores failed messages; it does not accelerate DynamoDB reads.
EBS is block storage for EC2 and cannot cache DynamoDB data.
MSK is managed Kafka and has no role in reducing DynamoDB read latency.
Correct. DynamoDB Accelerator provides an in-memory cache for microsecond read performance.
Option D is the correct answer. Microsecond read latency for DynamoDB is the key clue for DAX.
Lesson 0010. DAX.

8. A multi-tenant DynamoDB table uses tenant ID as the partition key. One large tenant now generates most of the writes, causing throttling on a single hot partition. Which design change should the developer make?

A dead-letter queue handles failed messages and does not redistribute DynamoDB write traffic.
Correct. Adding a suffix or other sharding value spreads writes across more partitions.
Scan reads the whole table and does nothing to reduce hot-partition writes.
X-Ray helps observe behavior, but it does not change partition-key distribution.
Option B is the correct answer. Partition-key sharding is the standard fix when one key becomes a write hotspot.
Lesson 0010. Partition key design.

9. A Lambda function processes asynchronous events from an internal application. Some events fail even after all retry attempts, and the operations team needs a separate location to inspect and reprocess those poison events later. Which feature should the developer configure?

Provisioned concurrency reduces cold starts, but it does not capture failed asynchronous events.
Stage variables pass API Gateway values and do not store failed Lambda events.
Correct. A dead-letter queue keeps failed asynchronous events for later inspection and replay.
Mapping templates transform API payloads and are unrelated to async failure handling.
Option C is the correct answer. The clue is failed asynchronous events that must be isolated for later analysis, which points to a DLQ.
Lesson 0002. Lambda DLQ.

10. A company stores application backups and user-uploaded media in an AWS service that must provide durable object storage and lifecycle rules to transition old objects to cheaper storage classes automatically. Which AWS service should the developer choose?

EBS is block storage for EC2 volumes and does not provide object lifecycle transitions.
EFS is a shared file system, not the standard object store with lifecycle rules in this scenario.
DAX is an in-memory cache for DynamoDB and not a storage service for media objects.
Correct. Amazon S3 is durable object storage and supports lifecycle transitions and expiration.
Option D is the correct answer. Durable object storage with lifecycle policies is the textbook Amazon S3 requirement.
Lesson 0011. S3 lifecycle.
