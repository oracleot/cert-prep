Lesson 0019

Classify Domain 3 deployment clues.

Domain 3 is Deployment: about 20% of DVA-C02, or roughly 13 questions. Your job is to pick the right lifecycle service cluster before the answer choices blur together.

The win

You will recognize whether a scenario is asking about artifacts, development testing, automated testing, or CI/CD deployment.

Related review: Domain 2 security patterns and the Domain 3 intro in docs/aws dva-c02.md.

Domain 3 deployment overview

Deployment questions cover the full application lifecycle: provision resources, manage configuration, package and release code, update software, monitor rollout health, control access, and recover safely.

Exam scenarios often combine services. The clue is not just "deploy"; it is what part of deployment needs control: artifact packaging, environment testing, IaC automation, traffic shifting, rollback, or release orchestration.

The four task statements

  1. 3.1 Prepare application artifacts: dependencies, package structure, repositories, resource requirements, and configuration data.
  2. 3.2 Test in development environments: deployed-code testing, mock API integrations, development endpoints, and stack updates.
  3. 3.3 Automate deployment testing: test events, API environments, approved versions, IaC templates, and service-managed environments.
  4. 3.4 Deploy with CI/CD services: pipelines, deployment strategies, rollbacks, branches, labels, and dynamic runtime configuration.

Deployment service cluster selector

Artifact repository or package storage → CodeArtifact, S3, ECR Serverless packaging and local build → SAM, Lambda layers, container images Infrastructure as code and stack updates → CloudFormation, SAM, CDK Application configuration lifecycle → AppConfig, Parameter Store, Secrets Manager Development endpoints and mock testing → API Gateway stages/mocks, Step Functions Local Version promotion without changing triggers→ Lambda versions and aliases Release orchestration across environments → CodePipeline, CodeBuild, CodeDeploy Traffic shifting, rollback, safe rollout → CodeDeploy, Elastic Beanstalk, ECS, CloudFormation

Fast distractor cuts

  • If the clue says package, dependency, layer, image, or repository, start in Task 3.1.
  • If the clue says mock endpoint, stage variable, dev endpoint, or stack update, start in Task 3.2.
  • If the clue says automated test event, approved version, IaC test environment, or API environment, start in Task 3.3.
  • If the clue says commit, branch, build, deploy stage, rollback, canary, linear, or blue/green, start in Task 3.4.

Retrieval drill: choose the best answer

1. A team is preparing a serverless release and must build Lambda dependencies, store versioned artifacts, and prepare configuration values before anything is deployed. Which Domain 3 task statement is primarily being tested in this scenario?

Option A is the correct answer. The clues are build dependencies, artifact storage, and configuration preparation, which all belong to Task 3.1.
Option B is incorrect. Development-environment testing focuses on mock endpoints, stack updates, and validating behavior after code is deployed.
Option C is incorrect. Automated deployment testing is about repeatable tests or temporary environments inside the pipeline, not artifact preparation itself.
Option D is incorrect. CI/CD services handle orchestration and promotion, but the stem stops earlier at preparing the release inputs.
Option A is the correct answer. Artifact preparation is the first Domain 3 task statement and a common exam starting point.

2. A frontend team needs an Amazon API Gateway endpoint that returns sample order responses while the real backend is still under development. The API should not invoke any Lambda function or HTTP service. Which integration type should the developer choose?

Option A is the correct answer. MOCK integrations let API Gateway return configured responses without any backend call, which is ideal for development testing.
Option B is incorrect. Lambda proxy integration requires a live Lambda backend and does not satisfy the no-backend requirement.
Option C is incorrect. Step Functions can be integrated with API Gateway, but it is still a real backend service invocation.
Option D is incorrect. A DynamoDB service integration also invokes a real backend and is not the best answer for stubbed responses.
Option A is the correct answer. When API Gateway must return fixed test data with no backend, choose MOCK integration.
Study Lesson 0021 and API Gateway integration types.

3. A pipeline must create a temporary stack from infrastructure as code, run integration tests against that stack on every change, and fail the release automatically if the tests do not pass. Which Domain 3 task statement is primarily being tested?

Option A is incorrect. Task 3.1 is about building and packaging the release inputs, not automatically testing deployed stacks.
Option B is incorrect. Task 3.2 involves development testing patterns, but the key clue here is automation inside the release flow.
Option C is the correct answer. Temporary IaC test environments and automatic pass/fail gates are core Task 3.3 signals.
Option D is incorrect. CI/CD orchestration may be present, but the stem is specifically about the testing phase within deployment automation.
Option C is the correct answer. Automated deployment testing is the task statement behind IaC-created test environments and release gates.
Review Lesson 0022 and CodePipeline concepts.

4. A company wants every commit to its main branch to trigger a source stage, a build stage, an automated test stage, and then a deployment stage in a fixed order. Which AWS service should the developer use to orchestrate these stages?

Option A is the correct answer. CodePipeline is the managed service that organizes source, build, test, approval, and deploy actions into ordered stages.
Option B is incorrect. CloudTrail records API activity for auditing and does not run release stages.
Option C is incorrect. CloudFront is a content delivery network, not a deployment orchestrator.
Option D is incorrect. CodeGuru Reviewer analyzes code quality but does not manage end-to-end CI/CD stages.
Option A is the correct answer. Ordered source-to-deploy orchestration is a direct CodePipeline clue.
Study Lesson 0023 and AWS CodePipeline.

5. A team runs an SQS event source that invokes a Lambda function. The queue configuration should keep targeting the same Lambda ARN even when the team promotes new function versions from test to production. Which Lambda feature best meets this requirement?

Option A is the correct answer. A Lambda alias provides a stable ARN that can be repointed to different published versions during promotion.
Option B is incorrect. Layers share dependencies across functions, but they do not provide a stable version pointer for invocations.
Option C is incorrect. Container images package code, but event sources do not target image URIs instead of function aliases.
Option D is incorrect. Reserved concurrency controls scaling and does not keep the trigger target stable across version changes.
Option A is the correct answer. Stable trigger ARN while versions move is the Lambda alias pattern.
Review Lesson 0021 and Lambda aliases.

6. A serverless application uses AWS CodeDeploy for Lambda deployments. The release plan requires CodeDeploy to send 10% of production traffic to the new version every 15 minutes until all traffic has shifted. Which deployment configuration should the developer select?

Option A is the correct answer. Linear deployments move equal percentages of traffic at fixed intervals, exactly matching 10% every 15 minutes.
Option B is incorrect. Canary sends a small initial portion first and then shifts the remainder later, not equal repeated increments.
Option C is incorrect. In-place is not the Lambda traffic-shifting pattern being tested by this serverless scenario.
Option D is incorrect. All-at-once would shift 100% immediately, which violates the gradual rollout requirement.
Option A is the correct answer. Equal traffic increments plus fixed intervals always point to a linear CodeDeploy configuration.
Study Lesson 0022 and CodeDeploy deployment configurations.

7. A product team uses feature flags for a checkout workflow. They need validators before rollout, gradual configuration deployment, and automatic rollback if CloudWatch alarms fire, all without redeploying application code. Which AWS service should they use?

Option A is the correct answer. AppConfig validates configuration, rolls it out gradually, and can roll back automatically based on alarms.
Option B is incorrect. CodeBuild compiles and tests code, but it does not manage runtime feature-flag rollout.
Option C is incorrect. CodeArtifact stores packages and dependencies, not runtime configuration deployments.
Option D is incorrect. CloudTrail records API calls for auditing and does not deploy application configuration.
Option A is the correct answer. Safe runtime configuration rollout without code redeploy is the AppConfig use case.
Review Lesson 0020 and AWS AppConfig.
