Lesson 0022

Automate the test gate before production.

Task Statement 3.3: choose the AWS service or feature that makes deployment testing repeatable across environments.

The win

You will classify deployment-testing clues into pipeline stages, buildspec tests, traffic shifting, API integrations, and approved environment promotion.

Source map: Domain 3 Task 3.3 in docs/aws dva-c02.md.

Exam selector

Commit triggers release workflow → CodePipeline source stage Run unit tests from commands → CodeBuild buildspec Shift Lambda traffic every interval → CodeDeploy linear Shift a small slice, then the rest → CodeDeploy canary Route all traffic immediately → CodeDeploy all-at-once Promote only approved container image → ECR tags or digests

Buildspec clue

If the scenario says install dependencies, run unit tests, package artifacts, or publish test reports from commands, choose CodeBuild buildspec.yml.

Retrieval drill: pick the tested feature

1. A software company stores application code in a Git repository. After each commit to the main branch, the release process must automatically run build and test actions, deploy to a staging environment, pause for approval, and then deploy to production. Which AWS service should orchestrate this workflow?

Option A is correct because CodePipeline is the AWS service that chains source, build, test, approval, and deploy stages into one release workflow.
Option B is a distractor because CodeArtifact stores packages and dependencies, but it does not orchestrate pipeline stages.
Option C is a distractor because CodeCommit hosts source code, but it does not coordinate downstream build and deployment stages.
Option D is a distractor because CodeGuru Reviewer analyzes code quality and security issues rather than running CI/CD workflows.
Option A is the correct answer. The stem is asking for the service that automates an end-to-end release pipeline with ordered stages and approval gates.
Review Task 3.4 CI/CD services and CodePipeline.

2. A developer creates an AWS CodeBuild project for a Node.js application. The build must install dependencies, run unit tests, execute lint checks, and package artifacts using commands defined in a file that is stored with the application source. Which file should the developer create?

Option A is correct because buildspec.yml defines the install, pre_build, build, and post_build commands that CodeBuild runs.
Option B is a distractor because appspec.yml is used by CodeDeploy for deployment hooks, not by CodeBuild for build commands.
Option C is a distractor because a CloudFormation or SAM template defines infrastructure resources, not CodeBuild phases.
Option D is a distractor because settings.xml is associated with Maven configuration and is not the standard CodeBuild workflow file.
Option A is the correct answer. When the exam says CodeBuild must execute named phases from source-controlled YAML, that points to buildspec.yml.
Study Task 3.4 CI/CD services and CodeBuild buildspec reference.

3. A company deploys a new Lambda version with AWS CodeDeploy. The release policy requires 10% of production traffic to shift to the new version every 10 minutes until all traffic uses the new version. Which deployment configuration meets this requirement?

Option A is correct because a linear deployment shifts equal percentages of traffic at fixed intervals, such as 10% every 10 minutes.
Option B is a distractor because canary shifts a small initial percentage and then moves the remaining traffic in a later step, not equal increments throughout.
Option C is a distractor because blue/green describes parallel environments rather than the exact incremental traffic pattern in the stem.
Option D is a distractor because immutable deployments create a fresh fleet, but the requirement is specifically about timed traffic shifting percentages.
Option A is the correct answer. Equal traffic increments at a fixed interval are the defining clue for a CodeDeploy linear deployment.
Review CodeDeploy configurations and Task 3.4.

4. A team is contract-testing a REST API before the backend microservice is available. The developers need Amazon API Gateway to return a static success payload so the client application can continue development without a live integration target. Which API Gateway integration type should they use?

Option A is correct because MOCK integration returns a configured response directly from API Gateway without calling a backend.
Option B is a distractor because HTTP integration still needs a reachable HTTP backend endpoint.
Option C is a distractor because AWS integration invokes an AWS service such as Lambda, and the scenario says no backend is ready.
Option D is a distractor because VPC link is used for private integrations, not static mock responses.
Option A is the correct answer. The requirement is backend-free test responses, which is the textbook use case for API Gateway MOCK integration.
Study API Gateway integration types and Task 3.2.

5. A CodePipeline pipeline in a tooling account must deploy application resources into a separate production AWS account. The security team requires the pipeline to use temporary credentials instead of long-term access keys. Which approach should the developer use?

Option A is correct because STS AssumeRole provides temporary credentials for secure cross-account access from a pipeline.
Option B is a distractor because KMS protects data with encryption but does not issue credentials for deployment actions.
Option C is a distractor because service control policies restrict permissions in AWS Organizations and do not grant temporary access.
Option D is a distractor because network ACLs control subnet traffic and are unrelated to identity delegation for CI/CD.
Option A is the correct answer. Cross-account deployments that must avoid long-term keys use STS AssumeRole for temporary, role-based credentials.
Review Task 3.4 CI/CD and IAM AssumeRole.

6. A startup hosts a web application with AWS Amplify. The team wants the develop branch to deploy automatically to a test environment and the main branch to deploy automatically to production whenever code is pushed. Which AWS service best matches this branch-based deployment requirement?

Option A is correct because Amplify connects Git branches to app environments and automatically deploys on each commit.
Option B is a distractor because AppConfig manages runtime configuration and feature flags, not branch-based web app deployments.
Option C is a distractor because OpsWorks is for configuration management and is not the managed branch-to-environment workflow described here.
Option D is a distractor because DataSync transfers data between storage systems and has no role in CI/CD branch deployment.
Option A is the correct answer. Automatic deployments mapped directly from Git branches to frontend environments are an Amplify pattern.
Study AWS Amplify and Task 3.4 CI/CD.
