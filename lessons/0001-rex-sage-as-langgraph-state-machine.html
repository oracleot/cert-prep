Lesson 0001

Rex and Sage are nodes, not magic agents.

Your first win: trace the Rex/Sage loop as a LangGraph state machine with shared state and explicit routing.

The mental model

In this repo, Rex and Sage are built as LangGraph nodes. A node is just a function that receives AppState and returns a dictionary of updates. LangGraph merges those updates into state, then follows an edge to the next node.

Primary source: agents/graphs/session.py. Official background: LangGraph low-level concepts.

The whole loop

START → coach_open → rex_challenge # Rex writes current_challenge → evaluate_answer # Evaluator writes last_evaluation → sage_depth # if correct or sage_explain # if incorrect → rex_rechallenge # if another cycle remains → coach_close → END

The important architectural move: the graph controls order. Rex does not call Sage. Sage does not call Rex. The graph routes between them.

How Rex is built

rex_challenge reads exam context from state, builds a prompt with build_rex_challenge_prompt, calls a LangChain chat model, parses JSON, then returns:

return {
    "current_challenge": {
        "domain": state["current_domain"],
        "scenario": challenge["scenario"],
        "question": challenge["question"],
        ...
    }
}

That returned dictionary is a state patch. It is not the whole app state.

How Sage is built

sage_respond.py exposes two nodes:

  • sage_depth — used when the answer is correct.
  • sage_explain — used when the answer is incorrect.

Both nodes call the same helper, generate a response, build an exchange record, optionally persist it, and return:

return {"session_history": [exchange]}

Because session_history uses an additive reducer, LangGraph appends the exchange instead of replacing the whole list.

The routing trick

The evaluator writes last_evaluation.outcome. Then the graph asks a small Python function which Sage node should run:

def route_after_evaluation(state):
    if state["last_evaluation"]["outcome"] == "correct":
        return "sage_depth"
    return "sage_explain"

This is the heart of many LangGraph apps: model calls produce state; plain code routes on state.
