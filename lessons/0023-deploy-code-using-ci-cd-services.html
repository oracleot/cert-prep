Lesson 0023

Deploy code by matching the workload to the CI/CD service.

Task Statement 3.4: select the pipeline, deployment configuration, IaC tool, or rollback control the scenario is really testing.

The win

You will map exam clues to CodePipeline, CodeBuild, CodeDeploy, CloudFormation StackSets, Elastic Beanstalk, SAM/CDK, and rollback triggers.

Source map: Domain 3 Task 3.4 in docs/aws dva-c02.md.

Selector map

Commit starts ordered release stages → CodePipeline Run build commands and test reports → CodeBuild buildspec Traffic shift for Lambda or ECS → CodeDeploy configuration EC2 app update with agent and AppSpec → CodeDeploy EC2/on-prem Same stack across accounts or Regions → CloudFormation StackSets Managed app platform with deployment modes → Elastic Beanstalk Serverless local build and deploy → AWS SAM Rollback on alarm during stack operation → CloudFormation rollback trigger

Traffic-shift shortcut

Lambda and ECS blue/green deployments use canary, linear, or all-at-once traffic shifting. EC2/on-premises deployments use in-place or blue/green, commonly with OneAtATime, HalfAtATime, or AllAtOnce.

Retrieval drill: pick the deployment control

1. A company wants every merge to the main branch to trigger a release workflow that pulls source code, runs a build, waits for a security approval, and then deploys to production. The team wants one managed AWS service to orchestrate the ordered stages. Which service should they use?

Option A is correct because CodePipeline is designed to orchestrate source, build, approval, and deployment stages in a release workflow.
Option B is a distractor because CodeArtifact stores software packages and dependencies rather than controlling CI/CD stage flow.
Option C is a distractor because CodeCommit is a source repository service, not the workflow engine for multiple deployment stages.
Option D is a distractor because CodeGuru Profiler analyzes application performance and does not manage release orchestration.
Option A is the correct answer. The stem is testing whether you recognize CodePipeline as the service that chains release stages and approval gates together.
Review Task 3.3 testing automation and CodePipeline.

2. A developer is creating a CodeBuild project for a containerized application. The project must run npm install, unit tests, and a Docker image build in separate phases defined in a YAML file stored with the source code. Which file should the developer configure?

Option A is correct because buildspec.yml defines the phases and commands that CodeBuild executes during a build.
Option B is a distractor because appspec.yml is used by CodeDeploy for deployment lifecycle hooks, not by CodeBuild for build instructions.
Option C is a distractor because taskdef.json defines Amazon ECS task settings and does not tell CodeBuild which commands to run.
Option D is a distractor because parameters.json can be used with infrastructure tools but is not the standard CodeBuild phase definition file.
Option A is the correct answer. When the question asks which YAML file controls CodeBuild install, test, and package phases, the answer is buildspec.yml.
Study Task 3.3 testing automation and CodeBuild buildspec.

3. A serverless application uses CodeDeploy to release a new Lambda version. The change management policy requires 10% of user traffic to shift to the new version every 10 minutes until the deployment is complete. Which CodeDeploy deployment configuration should the developer choose?

Option A is correct because linear deployment shifts equal percentages of traffic at fixed intervals.
Option B is a distractor because canary sends a small slice first and then shifts the remaining traffic later rather than using equal repeated increments.
Option C is a distractor because blue/green is an environment strategy, but the clue here is the exact timed traffic-shift pattern.
Option D is a distractor because immutable deployment replaces infrastructure with a new fleet and is not the Lambda traffic-shifting mode described.
Option A is the correct answer. The phrase 10% every 10 minutes is the classic exam signal for a linear CodeDeploy deployment.
Review CodeDeploy configurations and Task 3.3.

4. An ecommerce company runs a production service on Amazon ECS behind an Application Load Balancer. During deployments, the company wants traffic to move from the current task set to a replacement task set with automatic rollback support if health checks fail. Which AWS service should manage this deployment?

Option A is correct because CodeDeploy manages ECS blue/green deployments and shifts the load balancer from the original task set to the replacement task set.
Option B is a distractor because CodeBuild compiles and tests code but does not control ECS production traffic shifting.
Option C is a distractor because CodeArtifact stores packages and has no role in task set routing or rollback.
Option D is a distractor because CodeCommit stores source code and does not manage ECS deployment behavior.
Option A is the correct answer. ECS traffic shifting between task sets with rollback support is a CodeDeploy responsibility.
Study Task 3.3 traffic shifting and CodeDeploy configurations.

5. A company deploys a web application to Amazon EC2 instances by using AWS CodeDeploy. The deployment must copy revised files to the instances and run lifecycle hooks such as BeforeInstall and AfterInstall on each host. Which file must be included with the application revision?

Option A is correct because the AppSpec file tells the CodeDeploy agent which files to copy and which lifecycle hooks to run on EC2 instances.
Option B is a distractor because buildspec files are for CodeBuild phases, not CodeDeploy agent behavior on target hosts.
Option C is a distractor because an ECS task definition applies to containers, not to EC2 instance file-copy and lifecycle-hook deployments.
Option D is a distractor because CloudFormation parameter files provide stack inputs and do not control CodeDeploy host actions.
Option A is the correct answer. If CodeDeploy on EC2 must copy files and run hooks, the AppSpec file is required.
Review Task 3.3 testing automation and AppSpec file.

6. An enterprise team maintains a standard network baseline in AWS CloudFormation. The same template must create and update identical stacks in multiple AWS accounts and multiple Regions from a single administration point. Which CloudFormation feature should the team use?

Option A is correct because StackSets deploy and manage the same CloudFormation stack across multiple accounts and Regions.
Option B is a distractor because ChangeSets preview proposed changes but do not distribute templates across accounts or Regions.
Option C is a distractor because nested stacks organize templates inside one stack hierarchy, not across many accounts and Regions.
Option D is a distractor because wait conditions synchronize stack operations and are unrelated to multi-account rollout.
Option A is the correct answer. A single template managed across many accounts and Regions is the defining use case for StackSets.
Study CloudFormation StackSets and Task 3.3 cross-account.

7. A developer is building a serverless application that uses AWS Lambda, Amazon API Gateway, and Amazon DynamoDB. The developer wants to build the app locally, invoke functions for testing, and then deploy the stack with a tool designed specifically for serverless workloads. Which AWS tool best fits this requirement?

Option A is correct because AWS SAM provides local build, invoke, test, and deploy workflows tailored to Lambda, API Gateway, and related serverless services.
Option B is a distractor because Fault Injection Service runs resilience experiments and does not build or deploy serverless applications.
Option C is a distractor because Database Migration Service moves database data and is unrelated to serverless development tooling.
Option D is a distractor because AWS WAF provides application-layer protection and does not package or deploy serverless code.
Option A is the correct answer. Local build and deploy for a Lambda and API Gateway application is a direct signal for AWS SAM.
Review Task 3.1 artifact preparation and AWS SAM.

8. A team is updating a production CloudFormation stack for an internal payments service. The operations manager wants the stack update to roll back automatically if a specified Amazon CloudWatch alarm enters the ALARM state during the deployment window. Which CloudFormation feature should the team configure?

Option A is correct because rollback triggers monitor CloudWatch alarms during stack operations and automatically roll back if an alarm breaches.
Option B is a distractor because stage variables pass environment-specific values to APIs and do not affect CloudFormation rollback behavior.
Option C is a distractor because parameter labels help version Systems Manager parameters but do not monitor alarms during stack updates.
Option D is a distractor because notification rules send alerts about events and do not initiate CloudFormation rollback decisions.
Option A is the correct answer. The moment the stem says roll back a stack update when a CloudWatch alarm fires, think CloudFormation rollback triggers.
Review CloudFormation rollback triggers and Task 3.3.
