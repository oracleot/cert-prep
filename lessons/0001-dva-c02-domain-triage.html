Lesson 0001

DVA-C02 domain triage

Your first exam skill: read a scenario and identify what AWS is really testing before looking at the answer choices.

The win

In the local curriculum, DVA-C02 is split into four domains: Development, Security, Deployment, and Troubleshooting/Optimization. The fastest first move is to classify the question by intent.

Use the DVA-C02 curriculum map while reviewing the source notes in docs/aws dva-c02.md.

The triage algorithm

1. Ask: what is the scenario trying to change? 2. Match the verb to a domain. 3. Then choose the service family. Build / integrate / store data → Domain 1 Authenticate / authorize / encrypt → Domain 2 Package / test / release / rollback → Domain 3 Debug / trace / measure / optimize → Domain 4

Keyword map

  • Domain 1: Lambda invocation, API Gateway mapping, DynamoDB capacity, SQS/SNS/EventBridge, Step Functions, SDK retries.
  • Domain 2: IAM roles, STS, Cognito, permission boundaries, KMS, Secrets Manager, Parameter Store, WAF, Macie.
  • Domain 3: SAM package/deploy, Lambda layers, aliases, CodePipeline, CodeDeploy, CloudFormation, ECR/ECS, AppConfig.
  • Domain 4: CloudWatch metrics/logs, X-Ray traces, Logs Insights, CloudTrail, CodeGuru, Compute Optimizer, cache hits/misses.

Practice: classify first

Read the stem first. Identify the key words, then classify the domain before judging the options.

1. A company runs a serverless order API with Amazon API Gateway, AWS Lambda, and Amazon DynamoDB. A developer must update the Lambda code so that dev, test, and prod requests use separate DynamoDB tables based on the stage name. Which DVA-C02 domain is primarily being tested?

Correct. The key words are API Gateway, Lambda code, and DynamoDB tables. This is application code integrating with AWS services.
Security would focus on auth, authorization, encryption, or secrets. The stem is not asking for access control.
Deployment would focus on packaging, pipelines, aliases, or rollback. The stem is about runtime service integration.
Troubleshooting would focus on errors, metrics, traces, or performance. No failure or optimization target is given.
Option A is the correct answer. The stem is testing Domain 1 because the developer is changing application behavior that connects Lambda to DynamoDB.
Review the Lambda invocation lesson for the Domain 1 service cluster. See also Lambda supported services.

2. A mobile application allows users to upload images to an S3 bucket. The company does not want to embed long-term AWS access keys in the app. Users must receive temporary, least-privilege credentials after they authenticate. Which DVA-C02 domain is primarily being tested?

Domain 1 includes S3 integration, but the decisive words are temporary credentials and least privilege.
Correct. Temporary credentials, authentication, authorization, IAM roles, and STS point to Domain 2.
Domain 3 would mention build artifacts, CI/CD, deployment strategies, or rollback.
Domain 4 would mention debugging, logs, traces, or performance metrics.
Option B is the correct answer. The question is testing identity delegation and least-privilege access, which are Security topics.
Study Lesson 0014 for IAM role patterns. IAM documentation.

3. A team maintains an AWS SAM application. Before releasing a new version, the build process must create ZIP files for Lambda functions, upload the artifacts to Amazon S3, and generate a CloudFormation template that can be deployed later. Which DVA-C02 domain is primarily being tested?

Domain 1 is about application logic and service integration, not preparing deployment artifacts.
Domain 2 would involve permissions, encryption, or secret handling. Those are not the focus here.
Correct. ZIP artifacts, S3 upload, SAM, and deployable templates are deployment preparation signals.
Domain 4 would involve diagnosing failed deployments or measuring application behavior.
Option C is the correct answer. The stem is testing Domain 3 artifact preparation, especially SAM package and CloudFormation deployment workflows.
Review Lesson 0020 for SAM commands. SAM CLI reference.

4. Customers report intermittent latency from an API Gateway endpoint backed by Lambda. A developer must compare API latency, Lambda duration, and downstream call timing to identify where requests are slow. Which DVA-C02 domain is primarily being tested?

Domain 1 may include API Gateway and Lambda, but the task is not to build the integration.
Security would focus on access, encryption, or data protection. This stem is about latency.
Deployment would focus on release mechanics. No package, pipeline, or rollback problem is described.
Correct. Latency, duration, downstream timing, and identifying the slow component point to troubleshooting and optimization.
Option D is the correct answer. The key requirement is to measure and diagnose performance, which is Domain 4.
Study CloudWatch metrics and AWS X-Ray.

Practice: service family

5. A company has 30 Lambda functions that use the same Python utility library. The deployment packages are becoming large, and the team wants to update the shared library independently without copying it into every function package. What should the developer use?

Aliases route traffic to function versions. They do not share library code across functions.
Correct. Layers package shared libraries and make them available to one or more Lambda functions.
Event source mappings poll streams or queues and invoke Lambda. They do not solve shared code packaging.
Reserved concurrency controls scaling capacity for a function. It does not manage dependencies.
Option B is the correct answer. The key phrase is shared library across multiple Lambda functions, which points to Lambda layers.
See Lesson 0005 for more Lambda configuration patterns. Lambda layers documentation.

6. A developer is configuring traffic shifting for a new Lambda function version. The release must move the same percentage of traffic to the new version every 10 minutes until all traffic has shifted. Which deployment configuration meets this requirement?

Canary sends a small percentage first, waits, and then shifts the remainder. It is not equal increments throughout.
Correct. Linear deployment shifts equal percentages at fixed intervals.
All-at-once shifts all traffic immediately, which does not match the gradual requirement.
Blue/green describes separate environments. The stem asks for Lambda traffic-shifting increments.
Option B is the correct answer. Equal traffic increments at a fixed interval are the defining signal for a linear CodeDeploy configuration.
Review Lesson 0023 for deployment strategies. CodeDeploy configurations.

7. A serverless checkout workflow uses API Gateway, Lambda, DynamoDB, and an external payment API. The operations team must visualize the path of a single request across services and identify which downstream call adds latency. Which AWS service should the team use?

Correct. X-Ray provides distributed traces, service maps, and timing details across supported services.
AWS Config records resource configuration history. It does not trace individual requests.
Macie discovers sensitive data in S3. It is unrelated to request tracing.
KMS manages cryptographic keys. It does not provide service maps or traces.
Option A is the correct answer. The key words are visualize the path of a single request and downstream latency, which point to X-Ray.
See X-Ray documentation and using X-Ray with Lambda.

8. A retail application stores the password for an Amazon RDS database. The security team requires the password to be encrypted at rest and rotated automatically every 30 days with minimal custom code. Which AWS service should the developer choose?

Correct. Secrets Manager stores secrets securely and supports native rotation for database credentials.
Parameter Store can store secure strings, but native automatic credential rotation is the differentiator in the stem.
Certificate Manager manages TLS certificates, not database passwords.
IAM Identity Center manages workforce federation and SSO. It does not rotate application database credentials.
Option A is the correct answer. Automatic rotation of database credentials with minimal custom code points to AWS Secrets Manager.
Review Lesson 0016. Secrets Manager documentation.
