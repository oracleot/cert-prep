Lesson 0018

Pick the best-practice security pattern.

Task Statement 2.5: eliminate distractors by matching the risk to the smallest AWS control that fixes it.

The win

You will choose fast among S3 encryption/access, private service access, network controls, and edge protection.

Related review: secure application traffic and the Domain 2 section in docs/aws dva-c02.md.

Best-practice selector

Encrypt objects with AWS-managed key → SSE-S3 Audit/control customer-managed key use → SSE-KMS Temporary object download/upload → S3 presigned URL Keep bucket private behind CDN → CloudFront OAC Detect sensitive data in S3 → Amazon Macie Block layer 7 attack patterns → AWS WAF Baseline DDoS protection → Shield Standard

1. S3 access traps

  • Block Public Access is the first cut when a bucket should not be public.
  • Bucket policies enforce conditions such as required encryption or access only through a VPC endpoint.
  • Presigned URLs grant temporary object access without making the bucket public.
  • CloudFront OAC lets CloudFront access a private S3 origin; users should not access the bucket directly.

2. S3 encryption traps

Exam shortcut

When the clue says compliance, audit key usage, customer control, or key policies, prefer SSE-KMS. When it only says encrypt at rest with minimal management, SSE-S3 is enough.

  • Default encryption handles new objects; a bucket policy can deny uploads that omit the required encryption header.
  • SSE-C means the customer supplies keys for S3 to use; AWS does not store those keys.
  • KMS key policy controls key use. An S3 bucket policy alone cannot grant KMS decrypt permission.

3. Edge and detection controls

  • AWS WAF: choose for API Gateway, CloudFront, and ALB web request filtering.
  • Shield Standard: automatic DDoS protection for common infrastructure attacks.
  • Shield Advanced: choose when the scenario asks for enhanced DDoS response, cost protection, or DRT access.
  • Macie: choose for discovering sensitive data in S3, not for blocking live HTTP requests.

Mixed drill: choose the best answer

1. A training platform stores course-completion certificates in a private S3 bucket. After users sign in, the application must let each user download only one certificate for a short time without making the bucket public. Which solution best meets this requirement?

Option A is the correct answer. A presigned URL gives temporary access to one object by using the signer’s credentials without exposing the bucket publicly.
Option B is incorrect. Public-read ACLs make the object broadly accessible and violate the requirement to keep the bucket private.
Option C is incorrect. Network ACLs control subnet traffic, not temporary object-level authorization for S3 downloads.
Option D is incorrect. Shield Standard provides baseline DDoS protection, not time-limited access to private S3 objects.
Option A is the correct answer. The phrase temporary access to a private object is the classic S3 presigned URL pattern.
Review Lesson 0017 and S3 presigned URLs.

2. A healthcare company stores patient exports in Amazon S3. Compliance requires customer-managed encryption keys, key policies that limit decrypt access, and an audit trail of key usage. Which S3 encryption option should the developer choose?

Option A is the correct answer. SSE-KMS with a customer managed key gives the company key policies, KMS audit logs, and customer control over decrypt permissions.
Option B is incorrect. SSE-S3 encrypts data at rest, but it does not provide customer-managed key policies or KMS usage auditing.
Option C is incorrect. SSE-C uses customer-provided keys, but it does not match the common AWS-managed audit-and-policy pattern tested on DVA-C02.
Option D is incorrect. TLS protects data in transit, not the at-rest encryption and key governance requirements in the stem.
Option A is the correct answer. Auditability plus customer-managed key control is a direct SSE-KMS clue.
Study Lesson 0017 and SSE-KMS for Amazon S3.

3. A media company allows internal tools to upload video files to an S3 bucket. Security requires every new object upload to use server-side encryption, and any upload that omits the encryption header must be rejected automatically. Which approach should the developer use?

Option A is the correct answer. A deny on s3:PutObject with an encryption-header condition enforces that every upload declares the required server-side encryption setting.
Option B is incorrect. Listing the bucket is unrelated to enforcing encryption at upload time.
Option C is incorrect. A CloudFront origin policy controls who can read objects, not whether uploads include the encryption header.
Option D is incorrect. aws:SourceVpce restricts the network path to the bucket, but it does not require encryption on each PUT request.
Option A is the correct answer. The DVA clue is deny the PUT unless the encryption header is present.
Review Lesson 0017 and S3 policy condition keys.

4. A news site serves premium images through Amazon CloudFront. The S3 origin bucket must remain private so that users cannot bypass CloudFront and request objects directly from S3. Which configuration best meets this requirement?

Option A is the correct answer. CloudFront OAC lets CloudFront read from a private S3 origin while S3 Block Public Access prevents direct public access to the bucket.
Option B is incorrect. A public bucket policy directly violates the requirement to prevent users from reaching S3 without CloudFront.
Option C is incorrect. VPC endpoints do not solve CloudFront’s origin authorization pattern for a private S3 bucket.
Option D is incorrect. Network ACLs do not control how CloudFront is authorized to access S3 objects.
Option A is the correct answer. Private S3 behind CloudFront is the standard OAC pattern.
Study Lesson 0017 and CloudFront OAC.

5. A governance team has inherited dozens of S3 buckets and needs to identify where customer names, emails, and national ID numbers are already stored. The team wants a managed service that scans S3 and classifies sensitive data. Which AWS service should the developer recommend?

Option A is the correct answer. Amazon Macie discovers and classifies sensitive data such as PII across S3 buckets.
Option B is incorrect. Shield Advanced focuses on DDoS response and cost protection, not S3 data discovery.
Option C is incorrect. Route 53 Resolver handles DNS resolution and does not inspect S3 object contents.
Option D is incorrect. ACM manages certificates for TLS, not sensitive-data discovery in S3.
Option A is the correct answer. When the question asks to discover or classify PII already in S3, choose Macie.
Review Lesson 0015 and Amazon Macie.

6. A company runs a public REST API on Amazon API Gateway. A penetration test shows reflected XSS payloads in request parameters, and the security team wants a managed service to block those patterns before they hit the backend. Which option is the best answer?

Option A is the correct answer. AWS WAF provides layer 7 inspection and can block XSS patterns on requests to API Gateway.
Option B is incorrect. ACM secures the TLS connection but does not inspect or block malicious application-layer content.
Option C is incorrect. KMS protects stored secrets, not live HTTP requests containing XSS payloads.
Option D is incorrect. STS handles temporary credentials and is unrelated to request-body or query-string attack filtering.
Option A is the correct answer. XSS and SQL injection clues should immediately move you to AWS WAF.
Study Lesson 0017 and AWS WAF XSS match rules.

7. A gaming company is launching a major event and wants enhanced DDoS response, cost protection for scaling during attacks, and access to the AWS DDoS Response Team. Which service tier should the company choose?

Option A is the correct answer. Shield Advanced adds DRT access, enhanced protections, and cost-protection benefits that Shield Standard does not include.
Option B is incorrect. Shield Standard is included automatically, but it does not provide DRT access or DDoS cost protection.
Option C is incorrect. WAF rule groups help with layer 7 filtering, but they are not a substitute for Shield Advanced benefits.
Option D is incorrect. Origin failover improves availability design, but it does not provide DDoS response services or cost protection.
Option A is the correct answer. DRT access and DDoS cost protection are the signature clues for Shield Advanced.
Review Lesson 0017 and AWS Shield.

8. An analytics application writes reports to an S3 bucket through a gateway VPC endpoint. The security team wants the bucket to reject all requests that do not come through that specific endpoint, including any direct internet access. Which bucket policy condition key should the developer use?

Option A is the correct answer. aws:sourceVpce is the S3 bucket policy condition that restricts access to requests coming through a specific VPC endpoint.
Option B is incorrect. Source IP restrictions do not reliably express the endpoint-only requirement and are not the exam’s best answer here.
Option C is incorrect. IAM identity conditions can restrict principals, but they do not enforce that traffic must traverse a particular VPC endpoint.
Option D is incorrect. An SCP is the wrong control plane for a bucket-level network-path restriction.
Option A is the correct answer. Endpoint-only S3 access is the classic aws:sourceVpce bucket-policy pattern.
Study Lesson 0017 and S3 VPC endpoint bucket policies.
