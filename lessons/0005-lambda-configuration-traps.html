Lesson 0005

Lambda config traps are mostly "which knob owns this?"

DVA-C02 scenarios often hide the answer in one word: stage, runtime, handler, layer, extension, or package.

The win

You will classify Lambda configuration questions by ownership: Lambda function config, API Gateway stage config, deployment package, layer, or extension.

Review with the Lambda quick sheet after the drill.

The ownership map

Function code + dependencies → deployment package Shared libraries / custom runtimes → Lambda layer Runtime process add-on → Lambda extension Language execution environment → runtime Entry point Lambda calls → handler Per-function values → environment variables Per-API-stage values → API Gateway stage variables

Environment variables vs stage variables

Use Lambda environment variables for values attached to a function version, such as log level or table name for that function.

Use API Gateway stage variables when one API deployment stage must pass different values to the same integration, such as dev vs prod table names.

Runtime, handler, package

  • Runtime: the language environment Lambda uses to run code.
  • Handler: the method Lambda calls when the function is invoked.
  • Deployment package: the code and dependencies uploaded as a zip file or container image.

Layers vs extensions

A layer is reusable content mounted into the execution environment: libraries, dependencies, custom runtimes, or config files.

An extension is a companion process that integrates with the Lambda lifecycle, often for monitoring, security, or configuration tools.

Exam traps

  • Stage clue means API Gateway. "Different table per API stage" points to stage variables plus mapping/integration config.
  • Environment variables are not secret storage. For sensitive values, prefer Secrets Manager or Parameter Store.
  • Handler is not runtime. Runtime selects language; handler selects the entry point.
  • Layer is not invocation logic. Layers package shared dependencies; they do not trigger functions.
  • Container images do not use layers. Package dependencies inside the image instead.

Practice: pick the owner

1. A company uses one Lambda function behind an API Gateway REST API for both dev and prod. Each stage must send requests to a different DynamoDB table without deploying separate functions. Which configuration should the developer use to pass the table name per stage?

Option A is correct because API Gateway stage variables let the same deployed API pass different values for dev and prod.
Option B is incorrect because Lambda environment variables belong to the function configuration, not to individual API stages.
Option C is incorrect because layers package reusable code and do not provide per-stage configuration values.
Option D is incorrect because the handler identifies the entry point in the code and does not vary request data by API stage.
Option A is the correct answer. The clue is one Lambda function serving multiple API Gateway stages with different values, which points to stage variables.

2. A team maintains five Python Lambda functions that all use the same internal SDK helper library. The developers want to update that shared code once and avoid packaging it into every deployment zip. Which Lambda feature is the best fit?

Option A is incorrect because stage variables pass configuration values to API Gateway integrations and do not package libraries.
Option B is correct because Lambda layers let multiple functions share the same dependencies without duplicating them in each package.
Option C is incorrect because asynchronous destinations route invocation records and do not distribute shared code.
Option D is incorrect because reserved concurrency controls scaling and has nothing to do with packaging dependencies.
Option B is the correct answer. Shared libraries reused by multiple Lambda functions are the classic use case for Lambda layers.
Lesson 0003. Lambda layers.

3. A Node.js Lambda deployment succeeds, but every invocation fails because Lambda is looking for the wrong exported method in the code package. Which configuration setting should the developer update?

Option A is incorrect because the runtime chooses the language environment and does not identify the specific method Lambda should call.
Option B is correct because the handler setting tells Lambda which module and function to invoke, such as index.handler.
Option C is incorrect because a layer packages extra content and does not set the function entry point.
Option D is incorrect because a trigger defines how the function is invoked, not which method inside the code package runs.
Option B is the correct answer. When Lambda is calling the wrong method, the setting to fix is the handler configuration.
Lesson 0004 covers Lambda configuration. See Lambda handler.

4. A security team wants a monitoring agent to run alongside a Lambda function during initialization and invocation so it can collect telemetry without changing business logic. Which Lambda feature should the developer use?

Option A is correct because Lambda extensions are companion processes that integrate with the Lambda execution lifecycle.
Option B is incorrect because stage variables carry API Gateway configuration values and do not run code beside the function.
Option C is incorrect because the handler points to the business-logic entry point and is not a sidecar process.
Option D is incorrect because timeout sets the maximum runtime and does not provide monitoring functionality.
Option A is the correct answer. A Lambda extension is the feature built to run monitoring or security tooling alongside the function lifecycle.
Lesson 0004 covers Lambda configuration. See Lambda extensions.

5. A Lambda function connects to an Amazon RDS database. The security team requires the database password to be encrypted, retrieved securely at runtime, and rotated automatically with minimal custom code. Which AWS service should the developer use?

Option A is correct because Secrets Manager stores credentials securely and supports native rotation workflows for database secrets.
Option B is incorrect because Parameter Store can store secure strings, but automatic secret rotation is the decisive requirement in the stem.
Option C is incorrect because environment variables are not the preferred store for rotating database passwords.
Option D is incorrect because a layer version packages code and cannot securely manage or rotate credentials.
Option A is the correct answer. The key requirement is automatic rotation of a database password with minimal custom code, which points to AWS Secrets Manager.
