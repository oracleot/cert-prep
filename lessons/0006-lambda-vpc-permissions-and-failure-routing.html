Lesson 0006

Lambda in a VPC needs network config first, permissions second.

DVA-C02 scenarios often mix VPC access, execution-role permissions, and failure-routing distractors. Classify the problem before choosing a feature.

The win

When a Lambda question mentions private resources, ask three questions in order: Where is it attached? What can its role do? Where do failed async events go?

Use this with the Lambda quick sheet and Lesson 0002.

The decision path

Needs RDS/ElastiCache/private subnet resource? → configure Lambda VPC subnets + security groups → execution role needs EC2 network-interface permissions → outbound internet needs NAT, not just a public subnet Async failure routing? → destinations for success/failure records → DLQ only for failed async events

VPC access keyword pattern

Choose Lambda VPC configuration when the function must reach resources inside a VPC, such as private RDS, ElastiCache, or internal services. You select subnets and security groups; Lambda creates and manages elastic network interfaces.

ENI permission pattern

The function execution role must allow Lambda to manage network interfaces. The exam-friendly answer is usually the AWS managed policy AWSLambdaVPCAccessExecutionRole, which includes EC2 network-interface permissions plus CloudWatch Logs permissions.

Subnet and security group pattern

For private database access, put Lambda in subnets that can route to the target and assign a security group that the target allows inbound. If the function also needs public internet access from private subnets, look for a NAT gateway.

Destinations vs DLQ pattern

For asynchronous invocation, destinations can route success or failure records to SQS, SNS, Lambda, or EventBridge. A DLQ captures failed async events only, usually to SQS or SNS, and gives less context than destinations.

Exam traps

  • Resource policy is not execution role. Invocation permission lets a service call Lambda; execution role lets Lambda call AWS APIs.
  • VPC attachment is not internet access. A Lambda in private subnets needs NAT for outbound public internet.
  • Security groups are still enforced. The database must allow the Lambda security group inbound.
  • DLQ is not for success. Choose destinations when the scenario needs success and failure routing.
  • Console test events are payload tests. They do not prove private subnet routing, NAT, or security group access.

Practice: classify the missing piece

1. A payroll application runs a Lambda function that must connect to an Amazon RDS for PostgreSQL database deployed in private subnets. Which Lambda configuration is required so the function can reach the database?

Option A is correct because Lambda must be attached to the VPC by selecting subnets and security groups to reach private RDS resources.
Option B is incorrect because a dead-letter queue handles failed asynchronous events and does not provide network connectivity.
Option C is incorrect because an alias swap changes version routing and does not affect VPC access.
Option D is incorrect because a layer packages files for the execution environment and does not connect Lambda to private subnets.
Option A is the correct answer. To access an RDS database in private subnets, the Lambda function must be configured with the correct VPC subnets and security groups.
Lesson 0005. Lambda VPC config.

2. A Lambda function is attached to private subnets, but invocations fail during initialization because Lambda cannot create elastic network interfaces. Which IAM change should the developer make?

Option A is incorrect because the basic execution role adds logging permissions but not the EC2 network-interface permissions Lambda needs for VPC access.
Option B is correct because AWSLambdaVPCAccessExecutionRole includes the ENI permissions required for Lambda to create and manage network interfaces.
Option C is incorrect because S3 read access is unrelated to creating ENIs in a VPC.
Option D is incorrect because SNS publish permissions do not help Lambda attach to VPC networking.
Option B is the correct answer. The symptom points to missing EC2 network-interface permissions, and the managed policy AWSLambdaVPCAccessExecutionRole is the standard fix.
Lesson 0005 covers Lambda VPC configuration. See Lambda VPC permissions.

3. A Lambda function runs in private subnets so it can access an internal database, but it also must call a public third-party payment API on the internet. Which AWS resource is required to provide outbound internet access from those private subnets?

Option A is correct because private subnets need a NAT gateway to reach the public internet without assigning public IP addresses.
Option B is incorrect because an internet gateway alone does not give resources in private subnets outbound internet access.
Option C is incorrect because a resource policy controls who can invoke Lambda and does not affect routing.
Option D is incorrect because a dead-letter queue stores failed events and has nothing to do with internet connectivity.
Option A is the correct answer. A Lambda function in private subnets needs a NAT gateway for outbound access to public endpoints.
Lesson 0005 covers VPC networking patterns. See NAT gateway.

4. A logistics company invokes a Lambda function asynchronously to enrich shipment events. The team wants successful invocations sent to EventBridge and failed invocations sent to an SQS queue. Which Lambda feature supports separate targets for both outcomes?

Option A is correct because Lambda destinations can route success and failure records to different supported targets.
Option B is incorrect because a dead-letter queue captures only failed asynchronous events and cannot route successful ones.
Option C is incorrect because stage variables are API Gateway configuration values and do not route Lambda invocation results.
Option D is incorrect because redrive policy is an SQS setting and does not handle Lambda success routing.
Option A is the correct answer. When the scenario requires separate success and failure targets for asynchronous invocations, Lambda destinations are the right feature.
Lesson 0002. Lambda destinations.

5. A company wants Amazon S3 to be allowed to invoke a Lambda function whenever a new object is created in a bucket. Which policy type controls which principals and services can invoke the function?

Option A is correct because Lambda resource-based policies grant invocation permission to AWS services, accounts, or principals.
Option B is incorrect because the execution role controls what the function can access after it starts running, not who can invoke it.
Option C is incorrect because subnet settings control networking and do not authorize invokers.
Option D is incorrect because memory tuning affects performance and has no effect on invocation permissions.
Option A is the correct answer. The question is about who may call the Lambda function, which is handled by a resource-based policy.
