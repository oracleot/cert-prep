Lesson 0003

Lambda performance starts with concurrency, memory, and cold starts.

DVA-C02 expects you to choose the right Lambda knob from a scenario: cap concurrency, reduce latency, or tune memory/CPU.

The win

After the last lesson, you can classify how Lambda is invoked. Now ask the next exam question: what resource pressure is the scenario describing?

Use the Lambda quick sheet as your reference.

Three knobs to separate

Reserved concurrency

Reserved concurrency sets the maximum concurrent executions for one function. It can protect the rest of the account by preventing one function from using all available concurrency.

Provisioned concurrency

Provisioned concurrency prepares execution environments before requests arrive. Choose it when the scenario demands consistently low latency and cold starts are the problem.

Memory setting

Lambda allocates CPU in proportion to memory. If a function is CPU-bound or memory-bound, increasing memory can reduce duration, but you should tune for cost and speed together.

Cold start mental model

First request or changed function config → Lambda creates execution environment → downloads code / image → initializes runtime and code outside handler → invokes handler Next request may reuse the environment → warm start

Exam traps

  • Do not rely on warmers. Scheduled invocations do not guarantee low latency during scaling or across Availability Zones.
  • Provisioned concurrency is the low-latency answer. Use it when cold starts must be minimized.
  • Reserved concurrency is the blast-radius answer. Use it to cap or guarantee concurrency for a function.
  • Memory tuning changes CPU too. More memory can make CPU-bound functions faster.
  • Lambda Power Tuning uses Step Functions. It tests multiple memory settings and compares speed/cost.

Practice: choose the knob

1. A retail company serves a checkout API from Lambda through API Gateway. During flash sales, latency spikes appear only when new execution environments are created, and the business requires consistently low latency for 150 concurrent requests. Which Lambda feature should the developer choose?

Option A is incorrect because reserved concurrency controls scale limits but does not keep execution environments initialized.
Option B is correct because provisioned concurrency pre-warms execution environments for the configured concurrency level.
Option C is incorrect because layers share dependencies and do not reduce cold-start latency by themselves.
Option D is incorrect because scheduled warmers are not a reliable way to eliminate cold starts under real scaling conditions.
Option B is the correct answer. Provisioned concurrency is the Lambda feature designed to deliver consistently low latency by keeping environments ready before requests arrive.

2. A video-processing Lambda can spike sharply during uploads and is exhausting the shared account concurrency pool, causing unrelated payment and login functions to throttle. Which configuration best protects the rest of the account?

Option A is correct because reserved concurrency sets a hard ceiling on this function's concurrent executions.
Option B is incorrect because provisioned concurrency adds warm capacity but does not stop the function from scaling further.
Option C is incorrect because destinations route asynchronous results and do not manage concurrency.
Option D is incorrect because increasing memory changes resources per invocation, not the maximum number of concurrent invocations.
Option A is the correct answer. Reserved concurrency contains the blast radius of a noisy Lambda function by capping its concurrency.
Lesson 0001 covers Lambda concurrency concepts. See Reserved concurrency.

3. A document-conversion Lambda consistently runs with high CPU utilization and long duration. The developer wants the fastest configuration change to test whether more compute will improve performance. What should the developer do first?

Option A is correct because Lambda allocates more CPU as memory increases, which often reduces duration for CPU-bound workloads.
Option B is incorrect because a dead-letter queue handles failed asynchronous events and does not improve runtime performance.
Option C is incorrect because an alias controls version routing and does not change compute resources.
Option D is incorrect because a longer timeout lets the function run longer but does not make it run faster.
Option A is the correct answer. For CPU-bound Lambda workloads, increasing memory is the simplest test because CPU allocation scales with memory.
Lesson 0002 covers Lambda configuration. See Lambda memory configuration.

4. A team wants to evaluate several Lambda memory sizes and automatically compare duration and cost before choosing a production setting. Which AWS tool is designed for this analysis?

Option A is incorrect because X-Ray traces requests but does not run controlled tests across multiple memory settings.
Option B is correct because Lambda Power Tuning runs the function at multiple memory sizes and visualizes the speed-versus-cost tradeoff.
Option C is incorrect because Compute Optimizer is not the standard exam answer for directly benchmarking Lambda memory settings in this way.
Option D is incorrect because CodeDeploy canary releases manage traffic shifting, not performance tuning experiments.
Option B is the correct answer. Lambda Power Tuning is purpose-built to test multiple memory configurations and compare their performance and cost.
Lesson 0002 covers Lambda invocation and configuration. See Lambda Power Tuning.

5. A streaming analytics application uses Lambda to process records from an Amazon Kinesis Data Stream with 100 active shards. If the function code is healthy, what is the main factor that limits how many records Lambda can process in parallel?

Option A is correct because Lambda processes Kinesis records in parallel by shard, with ordering preserved within each shard.
Option B is incorrect because queue delay is an SQS concept and does not determine Kinesis parallelism.
Option C is incorrect because layers package dependencies and do not affect stream concurrency.
Option D is incorrect because aliases route to versions and do not increase Kinesis processing parallelism.
Option A is the correct answer. For Lambda event source mappings on Kinesis, shard count is the core concurrency clue because each shard is processed independently.
