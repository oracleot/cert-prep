Lesson 0004

Lambda event source mappings mean Lambda polls and batches for you.

DVA-C02 likes to test whether you can distinguish direct invocation from Lambda reading SQS, Kinesis, or DynamoDB Streams.

The win

You already know Lambda invocation and concurrency. Now add this exam move: when the source is a queue or stream, look for an event source mapping.

Keep the Lambda quick sheet open while you drill.

The pattern

SQS / Kinesis / DynamoDB Streams → event source mapping polls records → Lambda receives a batch → failures are handled by source-specific rules

SQS keyword pattern

Choose event source mapping when Lambda must process queue messages in batches. For failed SQS processing, think visibility timeout and redrive policy on the queue.

Kinesis and DynamoDB Streams keyword pattern

Choose event source mapping when Lambda processes stream records. The exam clue is shards: Lambda processes records from each shard in order, so shard count shapes parallelism.

Async destination distractor

Destinations are for asynchronous Lambda invocations. If Lambda is polling SQS, Kinesis, or DynamoDB Streams, start with event source mapping behavior instead.

Exam traps

  • Poll-based is not async invocation. Lambda is reading from the source.
  • SQS failures belong to SQS controls. Visibility timeout and redrive policy are common answer keywords.
  • Stream ordering matters. Kinesis and DynamoDB Streams preserve order per shard.
  • Shards shape concurrency. More active shards can allow more parallel Lambda processing.
  • Idempotency still matters. Retries can cause duplicate processing.

Practice: spot the mapping

1. A travel company writes booking-confirmation jobs to Amazon SQS so requests can be processed in batches. A Lambda function must automatically poll the queue and receive messages without the application calling Invoke directly. Which Lambda feature should the developer configure?

Option A is incorrect because destinations route records after asynchronous invocation and do not poll SQS.
Option B is correct because Lambda uses an event source mapping to poll SQS and deliver message batches to the function.
Option C is incorrect because subscribing SQS to SNS does not create the Lambda polling relationship described in the stem.
Option D is incorrect because API Gateway invokes Lambda directly and is unrelated to queue polling.
Option B is the correct answer. When Lambda must read from SQS in batches, the developer configures an event source mapping.

2. A customer-support application uses SQS with a Lambda consumer. The team wants messages that fail processing too many times to move automatically to a dead-letter queue after a defined number of receives. Which SQS setting controls that behavior?

Option A is correct because the redrive policy defines maxReceiveCount and the dead-letter queue target.
Option B is incorrect because visibility timeout controls how long a received message stays hidden before reappearing, not when it moves to a DLQ.
Option C is incorrect because reserved concurrency caps Lambda scaling and does not change SQS dead-letter routing.
Option D is incorrect because aliases route traffic to Lambda versions and have no effect on SQS retry thresholds.
Option A is the correct answer. In SQS, the redrive policy is the setting that sends repeatedly failed messages to the dead-letter queue.

3. An online game writes player profile updates to DynamoDB, and a Lambda function consumes the DynamoDB stream to update a leaderboard cache. The company must preserve record order for related items. What is the ordering and processing unit Lambda follows for this stream?

Option A is correct because Lambda processes DynamoDB Streams in order within each shard.
Option B is incorrect because layers package shared code and have nothing to do with stream ordering.
Option C is incorrect because aliases point to Lambda versions and do not define the unit of stream processing.
Option D is incorrect because stages are an API Gateway concept and do not control DynamoDB Streams behavior.
Option A is the correct answer. For DynamoDB Streams event source mappings, Lambda preserves ordering within each shard, so shards are the key processing unit.
Lesson 0003 covers Lambda concurrency concepts. See DynamoDB Streams.

4. A clickstream analytics team uses Lambda to consume an Amazon Kinesis Data Stream. The stream currently has 25 active shards, and the developers want to know what most directly limits Lambda's parallel processing capacity. Which factor is the key concurrency clue?

Option A is correct because each active Kinesis shard can be processed independently, so shard count drives Lambda parallelism.
Option B is incorrect because Kinesis uses shards rather than queues.
Option C is incorrect because Lambda layers are packaging constructs and do not change stream throughput.
Option D is incorrect because API Gateway stage count has no relationship to Kinesis event source mappings.
Option A is the correct answer. In Kinesis-based Lambda questions, shard count is the exam keyword that signals the parallelism limit.

5. A content-management system stores uploaded videos in Amazon S3 and triggers a Lambda function to create metadata after each object is created. The upload request must finish immediately without waiting for the Lambda result. How does S3 invoke the function?

Option A is correct because S3 event notifications hand off the event to Lambda without waiting for a function response.
Option B is incorrect because event source mappings are used when Lambda polls services such as SQS, Kinesis, or DynamoDB Streams.
Option C is incorrect because poll response is not a Lambda invocation model.
Option D is incorrect because stream mappings apply to streaming services, not S3 object notifications.
Option A is the correct answer. S3 invokes Lambda asynchronously, which is why the upload workflow does not wait for the metadata function to finish.
