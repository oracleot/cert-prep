DVA-C02 curriculum map

A compact map from the local curriculum to exam domains, service clusters, and practice work.

Domain weights

  • Domain 1 — Development with AWS Services: 32%, about 20 questions.
  • Domain 2 — Security: 26%, about 17 questions.
  • Domain 3 — Deployment: 20%, about 13 questions.
  • Domain 4 — Troubleshooting and Optimization: 24%, about 15 questions.

Task-statement anchors

Domain 1: Development

  • 1.1 Develop code for applications hosted on AWS.
  • 1.2 Develop code for AWS Lambda.
  • 1.3 Use data stores in application development.

Service cluster: Lambda, API Gateway, DynamoDB, S3, SQS, SNS, EventBridge, Step Functions, Kinesis, SDKs, SAM.

Domain 2: Security

  • 2.1 Implement authentication and authorization.
  • 2.2 Implement encryption by using AWS services.
  • 2.3 Manage sensitive data in application code.

Service cluster: IAM, STS, Cognito, KMS, CloudHSM, Secrets Manager, Parameter Store, ACM, WAF, Macie.

Domain 3: Deployment

  • 3.1 Prepare application artifacts.
  • 3.2 Test applications in development environments.
  • 3.3 Automate deployment testing.
  • 3.4 Deploy code by using AWS CI/CD services.

Service cluster: CodePipeline, CodeBuild, CodeDeploy, CodeArtifact, CloudFormation, SAM, CDK, Elastic Beanstalk, ECS, ECR, AppConfig.

Domain 4: Troubleshooting and Optimization

  • 4.1 Assist in root cause analysis.
  • 4.2 Instrument code for observability.
  • 4.3 Optimize applications by using AWS services and features.

Service cluster: CloudWatch, Logs Insights, X-Ray, CloudTrail, CodeGuru, Compute Optimizer, SNS filtering, CloudFront caching, DynamoDB Streams.

Fast triage rule

If the scenario asks... Build or integrate app behavior → Domain 1 Control identity, encryption, secrets → Domain 2 Package, test, release, rollback → Domain 3 Debug, trace, measure, tune → Domain 4
