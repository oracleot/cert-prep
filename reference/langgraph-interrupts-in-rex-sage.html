LangGraph Interrupts in Rex/Sage

Where the session graph intentionally pauses, what state exists there, and which API route resumes it.

The compile line

raw = build_session_graph().compile(
    checkpointer=get_checkpointer(),
    interrupt_before=["evaluate_answer", "rex_rechallenge"]
)

interrupt_before means: when execution reaches one of those nodes, LangGraph saves the checkpoint and stops before running the node.

Pause points

  • Before evaluate_answer: Rex has produced a challenge; the graph waits for the user answer.
  • Before rex_rechallenge: Sage has responded; the graph waits for the user to request the next challenge.

Route responsibilities

  • /session/start: starts the graph; it pauses before evaluate_answer.
  • /session/submit: writes user_answer, then resumes from evaluate_answer.
  • /session/state: reads the checkpoint snapshot and reports UI phase.
  • /session/next: resumes from rex_rechallenge, then pauses before the next evaluate_answer.

Phase detector

def _snapshot_phase(snapshot):
    nodes = set(snapshot.next or [])
    if not nodes:
        return "summary"
    return "sage_done" if "rex_rechallenge" in nodes else "ready"

The UI phase is inferred from the next node LangGraph is waiting to run.

Memory hook

An interrupt is a bookmark before a node. The checkpointer remembers the page. The API route writes the missing state. Calling ainvoke(None, config) continues from the bookmark.
