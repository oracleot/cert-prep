Security quick sheet

Domain 2 exam selectors for IAM, STS, Cognito, KMS, secrets, TLS, VPC endpoints, S3 access, and edge protection.

Domain 2 shape

  • Weight: 26%, roughly 17 exam questions.
  • Official tasks: authentication/authorization, encryption with AWS services, and sensitive data in application code.
  • Fast triage: identity, encryption, secrets, certificates, private traffic, or sensitive data discovery usually means Security.

Identity selector

AWS compute needs AWS API access → IAM role Cross-account or delegated access → STS AssumeRole External workforce identity → IAM Identity Center / federation App users sign up and sign in → Cognito user pool App users need AWS credentials → Cognito identity pool API caller signs with SigV4 → IAM authorization Bearer token custom auth → TOKEN Lambda authorizer Headers/query/context custom auth → REQUEST Lambda authorizer

Policy cuts

  • Identity policy: attached to user, group, or role.
  • Resource policy: attached to resource; common for S3, Lambda, KMS, SQS, SNS.
  • Trust policy: says who can assume a role.
  • SCP, boundary, session policy: limits maximum access; does not grant access.
  • Explicit deny: wins over any allow.

Encryption and secrets selector

Audit/customer control of object keys → SSE-KMS Minimum managed S3 encryption → SSE-S3 Customer supplies S3 object keys → SSE-C Encrypt data inside the application → KMS GenerateDataKey Database password or rotation needed → Secrets Manager Hierarchy/simple encrypted config → SSM SecureString TLS certificate for AWS front door → ACM Dedicated HSM control required → CloudHSM

Secret-handling cuts

  • Do not hardcode access keys, tokens, passwords, or decrypted secrets.
  • Use roles for AWS-hosted workloads; use federation or temporary credentials where possible.
  • Lambda environment variables are encrypted at rest, but they are not shared secret storage.
  • Use an interface VPC endpoint for private calls to Secrets Manager, SSM, KMS, or most AWS APIs.

Traffic and edge selector

Public HTTPS certificate → ACM API vanity hostname → API Gateway custom domain Private S3 or DynamoDB access → Gateway VPC endpoint Private access to most AWS APIs → Interface VPC endpoint ENI-level stateful allow rules → Security group Subnet-level stateless allow/deny rules → Network ACL Layer 7 request filtering → AWS WAF Enhanced DDoS response/cost protection → Shield Advanced Find sensitive data in S3 → Amazon Macie

High-yield traps

  • API keys are for usage plans and throttling, not strong authentication.
  • KMS stores keys, not application passwords.
  • An S3 bucket policy cannot grant KMS decrypt by itself; the key policy/grant must allow use.
  • NACLs are stateless, so return traffic and ephemeral ports matter.
  • CloudWatch Logs are application logs; CloudTrail is AWS API history.
  • Macie discovers sensitive data in S3; WAF blocks web request patterns.
