Lambda quick sheet

Exam-focused Lambda facts from Domain 1, with links to the official AWS docs.

Invocation modes

  • Synchronous: caller waits for the response. CLI uses aws lambda invoke.
  • Asynchronous: caller hands off the event; Lambda queues and retries function errors twice.
  • Polling: event source mappings read batches from SQS, Kinesis, or DynamoDB Streams.

Configuration knobs

  • Memory: increasing memory also increases CPU.
  • Timeout: max runtime before Lambda stops the invocation.
  • Concurrency: unit of scale; reserved concurrency caps or guarantees capacity.
  • Provisioned concurrency: keeps execution environments ready for low-latency starts.
  • Environment variables: function-level settings; use Parameter Store or Secrets Manager for shared sensitive values.

Event source mappings

Use event source mappings when Lambda must read from SQS, Kinesis, or DynamoDB Streams in batches. For Kinesis and DynamoDB Streams, shards control concurrency because Lambda processes each shard in order.

  • SQS: think visibility timeout and redrive policy for failed batches.
  • Streams: think ordering per shard and shard-shaped parallelism.

Failure handling

  • Async destinations: send success or failure records to SQS, SNS, Lambda, or EventBridge.
  • DLQ: capture failed asynchronous events for later inspection.
  • SQS redrive policy: queue-level failure path for event source mappings from SQS.
  • Idempotency: required when retries can deliver the same event more than once.

Packaging and dependencies

  • Zip archive: package code and dependencies; layers can provide shared libraries.
  • Container image: package runtime, code, and dependencies together; Lambda layers are not used.
  • SAM build: useful for building native dependencies in a Lambda-compatible environment.
